통합 계획 의사결정 피드백 콘솔

6대 벤치마킹 레포지토리 코드 분석에 기반하여, 안정적이고 토큰 효율적인 1000+ 사이클 생존을 목표로 하는 차기 통합 계획에 대한 의사결정 수렴 페이지입니다.

현재 의사결정 영역

Option A 기반 통합 구현 계획 승인 및 복원력 파라미터 튜닝

AS-IS (기존의 물리적 한계)

  • 동기식 I/O 병목: writeFileSync로 인한 200~500ms 지연으로 Mineflayer 봇 소켓이 킵얼라이브를 놓쳐 튕김 현상 발생.
  • 무한 루프 정체: 특정 행동이 반복 실패할 때 프롬프트 갱신 없이 무한정 동일 명령을 에러 실행하여 토큰 요금 파산.
  • 생존 무방비: LLM 프롬프트 대기 시간 및 기동 중 적대적 몹에게 피격당하면 반응 없이 그대로 사망.

TO-BE (통합 계획 복원력 전략)

  • 완전 비동기 논블로킹 I/O: fs.promises 상태 기록 분산화 및 100턴 에포크 단위 롤링 디스크 저장.
  • Stall Detection Guard: 3회 이상 동일 패턴 실패 시 인라인 시스템 프롬프트 가이드를 강제 주입해 루프 탈출 유도.
  • Combat Pulse 백그라운드 스레드: 에이전트 바깥에서 0.75초 간격으로 스캔하여 자동전투 및 자가 생존 모드 인터럽트.
  • 엇박자(Staggered) 자동 재연결: 서버 킥 발생 시 30s/60s 간격을 두고 자동 접속 회복 및 메모리 재수화.
