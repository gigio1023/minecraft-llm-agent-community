# Absolute Zero: Reinforced Self-play Reasoning with Zero Data

- **title**: Absolute Zero: Reinforced Self-play Reasoning with Zero Data
- **authors**: Andrew Zhao, Yiran Wu, Yang Yue, Tong Wu, Quentin Xu, Yang Yue, Matthieu Lin, Shenzhi Wang, Qingyun Wu, Zilong Zheng, Gao Huang (Tsinghua University, BIGAI, Pennsylvania State University)
- **year**: 2025 (arXiv v2 Oct 2025)
- **venue/source**: arXiv 2505.03335
- **arxiv_id**: 2505.03335
- **urls**: paper https://arxiv.org/abs/2505.03335 ; code https://github.com/LeapLabTHU/Absolute-Zero-Reasoner
- **source availability**: latex (papers/latex/2505.03335/)

## Primary-source facts (what the paper states/shows)

- Proposes the **Absolute Zero** paradigm: one model is both **proposer** (invents tasks) and
  **solver** (solves them), trained jointly via RL with **zero external data** (no human queries,
  no gold answers, no distilled traces). Contrasts with SFT (needs gold task+CoT+answer) and RLVR
  (needs gold task+answer): both still rely on human-curated datasets.
- The instantiation, **Absolute Zero Reasoner (AZR)**, uses a **code executor** as the
  environment. The executor both validates a proposed task's integrity and provides verifiable
  correctness feedback. Tasks are program/input/output triplets in three reasoning modes:
  abduction, deduction, induction.
- Objective (paper's eq.): the proposer gets a **learnability reward** `r_propose` (a task is
  rewarded for being neither trivially solved nor unsolvable, estimated by Monte Carlo solver
  rollouts), the solver gets a **solution reward** `r_solve(y, y*)` with "the environment again
  serving as the verifier." The paper color-codes the environment as the trusted (green) source
  and human experts as the removed (red) source.
- **Result**: trained with zero in-domain data, AZR reaches state-of-the-art on coding reasoning
  and competitive math, and outperforms prior "zero-setting" models by +1.8 absolute average.
  Coder base amplifies reasoning; cross-domain transfer (code self-play -> math) is large (+10.9
  to +15.2 points); gains scale with model size (3B/7B/14B -> +5.7/+10.2/+13.2).
- **Design rationale stated explicitly**: prior self-play with "learned reward models that are
  prone to hacking" is rejected; AZR stays "grounded in a real environment ... feedback from the
  environment as a verifiable source of reward ... helps prevent issues such as hacking with neural
  reward models." Likened to AlphaZero self-play but with an environment verifier.
- **Honest caution they raise**: an "uh-oh moment" where AZR with Llama3.1-8b "occasionally
  produces concerning chains of thought," flagging a safety need.

## Interpretation (flagged as inference)

- This is the strongest published example of admissible self-improvement under the wave-4 bright
  line: it removes ALL human labels (the data-cost dream of the autoresearch thesis) yet KEEPS an
  external, deterministic verifier (the code executor). The authors explicitly chose the
  environment verifier over a learned reward model BECAUSE the learned one hacks. That is the
  repo's design rule restated in a frontier system.
- The proposer/solver split maps onto an autoresearch loop where one component generates tasks or
  scenarios and another attempts them, both scored by the environment. For this repo, the runtime
  (Mineflayer + verifiers) is the analog of the code executor: it is the trusted `e`.
- The honest bound for this repo: AZR's verifiability is total because code execution is exact and
  resettable. The repo's Physical/Material layers approximate that (the verifier checks inventory,
  block, durability deltas), but the Social/Institutional layers do not have an exact executor for
  "promise honored" or "norm maintained." AZR transfers to the layers where the runtime gives a
  clean pass/fail, not to contested social success.

## Mechanically useful vs research contribution

- **Mechanically useful**: the proposer-side **learnability reward** is a borrowable idea for
  curriculum self-improvement: prefer auto-generated scenarios that the current actor solves with
  intermediate success rate (not 0, not 1), measured by Monte Carlo rollouts scored by the runtime
  verifier. This is a concrete, label-free way to grow scenario difficulty for a Minecraft loop
  while keeping the verifier as scorer.
- **Research contribution it is NOT**: AZR's contribution is zero-data reasoning self-play in a
  code-executor environment. The repo must not reframe its social simulation as "Absolute Zero for
  Minecraft." The borrow is the principle (self-play + environment verifier, no learned RM) and the
  learnability-reward mechanic, applied advisorily.

## WAM layer(s) informed

Method-level. Most directly informs Physical/Material self-improvement (where the runtime verifier
is exact, like a code executor). The proposer/solver + learnability-reward pattern is a scenario
generator for an autoresearch loop. Social/Institutional transfer is bounded by the absence of an
exact verifier there.
