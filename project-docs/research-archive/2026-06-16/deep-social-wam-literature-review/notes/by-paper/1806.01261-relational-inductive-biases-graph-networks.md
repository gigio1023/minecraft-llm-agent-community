# Relational inductive biases, deep learning, and graph networks (the Graph Network position paper)

- title: Relational inductive biases, deep learning, and graph networks
- authors: Peter W. Battaglia et al. (27 authors; DeepMind, Google Brain, MIT, Edinburgh)
- year: 2018
- venue: arXiv preprint (widely cited position/review/unification paper)
- arxiv_id: 1806.01261
- urls: paper https://arxiv.org/abs/1806.01261 , code https://github.com/deepmind/graph_nets
- source_availability: latex (papers/latex/1806.01261/sections/content.tex)

## Primary-source facts (what the paper states)

- It is "part position paper, part review, part unification." Central claim: COMBINATORIAL
  GENERALIZATION (constructing new inferences from known building blocks) should be a top
  priority for AI, and STRUCTURED representations + computations are key to it.
- A RELATIONAL INDUCTIVE BIAS is an architectural assumption that biases learning toward
  entities, relations, and rules for composing them. The paper reviews how MLPs, CNNs, RNNs each
  carry (weaker) relational biases, and argues graphs carry the strongest.
- It introduces the GRAPH NETWORK (GN) block as a unifying building block that generalizes prior
  graph neural networks (message-passing NNs, interaction networks, relation networks, etc.). A
  GN operates on a graph `(V, E, u)` of node attributes, edge attributes, and a global attribute,
  and applies update functions (edge -> node -> global) with aggregation (permutation-invariant
  pooling) between them. It maps an input graph to an output graph of the same structure with
  updated attributes.
- It ships an open-source library ("Graph Nets") for composing GN blocks.

## Interpretation (my reading, labeled inference)

- This is the conceptual backbone behind C-SWM, GNS, Interaction Networks, and OP3: each is a
  specific instantiation of "entities + relations + message passing." For this lane it is the
  source that names WHY structured prediction generalizes (entities/relations are reusable across
  configurations) and WHAT the general computation is (typed update functions + aggregation).
- The GN's explicit global attribute `u` maps cleanly onto an institutional/settlement-level
  summary in this project (norms, conventions, settlement state), while node = actor/item and
  edge = relation. That gives a single formalism spanning the project's 4 layers.

## Mechanically useful vs research contribution

- Mechanically useful: the GN block formalism (node/edge/global update + aggregation) as the
  vocabulary and architecture for a structured social-material predictor, plus the open-source
  library as a reference implementation. The "combinatorial generalization" argument is the
  defensible justification for predicting typed deltas instead of pixels.
- Avoid / overclaim: it is a framework paper, not an empirical social result. Cite it for the
  representation argument, not as evidence that any social-material WM exists (none does here).

## WAM layer(s) informed

- Cross-cutting (all 4 layers). It is the most general statement of the representation this
  project would predict in: entities (nodes), relations (edges), institution-level state (global).
