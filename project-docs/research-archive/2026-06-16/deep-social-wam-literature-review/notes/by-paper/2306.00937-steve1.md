# STEVE-1 — A Generative Model for Text-to-Behavior in Minecraft

- **authors**: Lifshitz, Paster, Chan, Ba, McIlraith (UToronto / Vector)
- **year**: 2023 — **venue**: NeurIPS 2023
- **arxiv_id**: 2306.00937
- **urls**: https://arxiv.org/abs/2306.00937 ; code https://github.com/Shalev-Lifshitz/STEVE-1 ; weights https://huggingface.co/CraftJarvis/MineStudio_STEVE-1.official (12,822 downloads — the most-downloaded Minecraft policy on the Hub)

## Primary-source facts

- **Research question**: can VPT be cheaply turned into an *instructable*
  (text- and image-goal-conditioned) policy without expensive text-gameplay
  annotation?
- **Architecture (unCLIP-style, hierarchical)**:
  - **Prior** `p(z_goal | y)`: a small CVAE (2-layer MLP, 512 hidden) mapping a
    frozen MineCLIP **text** embedding `z_y` to a MineCLIP **visual** goal
    embedding `z_goal`.
  - **Policy** `p(τ | z_goal)`: VPT (ResNet on 128×128×3 frames + Transformer-XL)
    fine-tuned with an affine `z_goal` injection added to the ResNet output.
  - Combined: `p(τ | y) = p(z_goal | y) · p(τ | z_goal)`.
- **Training trick — packed hindsight relabeling**: sample future timesteps as
  goals, set goals for preceding segments to those future MineCLIP embeddings,
  pack multiple relabeled segments into one sequence. Self-supervised ⇒ teaches
  "which actions lead to which states."
- **Inference — classifier-free guidance**: compute conditional and
  unconditional logits with the same net (goal dropout p=0.1 in training), mix
  with `λ`. Higher `λ` ⇒ stronger instruction adherence (large empirical gain).
- **Datasets**: policy trained on **54M frames** (~1 month at 20 FPS): 39M
  OpenAI contractor frames + 15M VPT-generated frames. Prior trained on **2,000
  hand-labeled text↔16-frame-clip pairs** (~30 min of gameplay, a few hours to
  collect) + 8,000 GPT-3.5-augmented instructions.
- **Observation / action space**: identical to VPT (pixels in, camera/buttons
  out). Adds a goal-embedding input.
- **Evaluation**: 11-task early-game suite + 2 prompt-chaining tasks. Two metric
  families: **programmatic** (travel distance, max log/seed/dirt counts from
  MineRL state) and **MineCLIP** (min cosine distance between goal embedding and
  any episode frame embedding) for tasks where programmatic checks aren't
  practical.

## Interpretation (inference)

- STEVE-1 is a **language/visual-goal-conditioned VLA**: `p(a | o, z_goal)`. The
  goal is a *latent*, not a predicted future, so it is not a WAM. But packed
  hindsight relabeling is implicitly "what state does this action reach," which
  is the closest VPT-family idea to a transition model.
- The key economic lesson: instructability was bought for ~30 minutes of labels
  + a frozen CLIP. The expensive part (the 54M-frame policy) was reused from VPT.

## Mechanically useful vs research contribution

- **Mechanically useful**: (a) the hindsight-relabeling recipe is reusable if we
  ever train a goal-conditioned helper from our own transcripts; (b) STEVE-1 is
  the natural *low-level executor baseline* under a separate visual-policy lane;
  (c) MineCLIP-as-evaluator is a concrete instance of "VLM/embedding scoring" —
  which this repo treats as a **support audit**, not runtime truth.
- **What would be overclaim**: using MineCLIP cosine distance as success truth.
  The paper itself flags it as a proxy, and for social/material claims it is
  blind (it cannot see who owns an item or whether a promise was kept).

## WAM layer(s) informed

Physical-WAM (executor). Reproducible: weights + code public. Relevant to this
repo only as an instructable low-level body, not as the social/material layer.
