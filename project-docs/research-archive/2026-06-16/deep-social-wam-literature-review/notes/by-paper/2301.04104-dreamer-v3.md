One algorithm with fixed hyperparameters across 150+ tasks, and the first to collect
diamonds in Minecraft from scratch with no human data. Source: LaTeX deep-read at

## Plain-language: what it introduced and why it mattered

DreamerV1 and V2 worked but still needed per-domain hyperparameter tuning, the brittleness
that keeps RL from being plug-and-play. DreamerV3's contribution is **robustness**: a single
configuration that works across more than 150 tasks spanning continuous and discrete actions,
visual and low-dimensional inputs, 2D and 3D worlds, and very different reward scales, with
**fixed hyperparameters and no tuning**. It reaches this through a set of normalization,
balancing, and transformation tricks.

The headline empirical result: **applied out of the box, DreamerV3 is the first algorithm to
collect diamonds in Minecraft from scratch, with no human demonstration data and no
hand-designed curriculum**. Getting diamonds in Minecraft from scratch had been a recognized
grand challenge (sparse rewards, hard exploration, long horizons, procedural world variety);
prior approaches all leaned on human expert data and curricula (e.g., VPT). Doing it from
scratch is why this is a landmark for the genealogy: it is the proof that a latent
world-model agent can do hard, long-horizon, open-world control.

A second important finding: **favorable scaling**. Larger DreamerV3 models not only score
higher but also need *less* environment interaction to solve a task, a predictable
"make it bigger to make it better" property that earlier RL lacked.

## Definitions (first use)

- **Fixed hyperparameters across domains**: the same numeric settings (learning rates, loss
  weights, etc.) are used for every task, so the algorithm does not have to be reconfigured
  per environment.
- **Symlog (symmetric log) transform / symlog squared loss**: a transformation that
  compresses the magnitude of targets so the network can predict quantities of very different
  orders of magnitude (e.g., tiny vs huge rewards) with one stable loss. DreamerV3 predicts
  rewards and values in this transformed space; TD-MPC2 later adopts a similar discrete-
  regression-in-log-space idea.
- **Free bits**: clipping the KL/dynamics loss below a small threshold (about 1 nat) so the
  model does not waste capacity over-optimizing an already-easy prediction, avoiding a
  degenerate solution where dynamics are trivially predictable but uninformative.
- **Continue predictor**: predicts a per-step continuation flag (1 = episode continues, 0 =
  ends), used to down-weight imagined steps past a likely episode end.

## Primary-source facts

- **Architecture**: same three-network structure as the lineage: world model (RSSM) predicts
  outcomes, critic judges value, actor chooses actions; all trained concurrently from replayed
  experience. The RSSM uses categorical latents with straight-through gradients (from V2). It
  reconstructs inputs (decoder) to keep representations informative; reward predictor uses the
  symlog squared loss; continue predictor uses logistic regression.
- **World-model loss**: prediction loss + dynamics loss + representation loss, with fixed
  weights (1, 1, 0.1); free bits clip the dynamics/representation losses; KL balancing carried
  over from V2.
- **Robustness toolkit**: normalization (symlog, return normalization), balancing (KL
  balancing, fixed loss weights), and transformations are what let one config span domains.
- **Result 1**: outperforms specialized expert algorithms across 150+ tasks with one
  hyperparameter set.
- **Result 2**: first to collect diamonds in Minecraft from scratch without human data or
  curricula (the long-standing MineRL Diamond challenge), applied out of the box.
- **Result 3**: favorable scaling, larger models give higher scores and higher
  data-efficiency.

## Interpretation (labeled inference)

- DreamerV3 is the genealogy's "it finally just works, and it works in Minecraft" node. For a
  newcomer, the key takeaway is not a new architecture (it is still RSSM + categorical latents
  + actor-critic in imagination) but that *robustness engineering* (symlog, free bits,
  balancing) is what turned a domain-specific recipe into a general one. This is the
  feasibility proof that a latent/structured world-model agent does hard Minecraft control on
  modest hardware.
- Crucially for this review: DreamerV3 (and its successor Dreamer 4) use **structured
  inventory state alongside pixels** and a latent (not pixel-reconstruction-primary)
  representation. This is the central evidence that "latent + structured state" is the
  feasible Minecraft path (cross-ref `wam-foundations.md` section 7 and
  `2509.24527-dreamer4.md`). But its "inventory state" is the single actor's own inventory for
  crafting; there is no multi-actor possession, claim, obligation, or trust.

## Mechanically useful vs research contribution

- **Mechanically useful**: the robustness tricks (symlog/return normalization, free bits, KL
  balancing) are reusable stabilizers for any predictor that must handle widely varying signal
  magnitudes. The "feed typed/structured inventory state as input to a world model" design is
  a direct parallel to feeding typed Minecraft state to a structured social predictor.
- **Research contribution (not this repo's)**: the diamonds-from-scratch result and the
  one-config generality are research achievements via imagination-trained RL, which is out of
  scope for an advisory WAM. Borrow stabilization and the structured-state-input precedent;
  do not adopt the imagination-trained actor as runtime authority.

## Relevance to this repo (one line)

Strongest feasibility proof that a latent + structured-state world-model agent does hard
long-horizon Minecraft control on modest hardware; the structured-state-input design and
robustness tricks transfer, the imagination-RL policy and single-actor scope do not.

## Which WAM layer(s) it informs
