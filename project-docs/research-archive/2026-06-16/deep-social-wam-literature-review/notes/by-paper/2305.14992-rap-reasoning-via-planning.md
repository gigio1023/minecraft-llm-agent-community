## Primary-source facts (LaTeX-verified)

- **The single idea**: repurpose ONE LLM into two roles at once. As the reasoning agent it samples an action `a_t ~ p(a | s_t, c)`; as the world model it predicts the next state via a state-transition distribution `p(s_{t+1} | s_t, a_t, c')`, where `c` and `c'` are different prompts. The reasoning trace is then an interleaved sequence of states and actions `(s_0, a_0, s_1, ..., s_T)`, in contrast to Chain-of-Thought which is only a sequence of actions. Quote: "we repurpose the same LLM to obtain a state transition distribution."
- **State/action are instantiated per task**: Blocksworld state = the block configuration in natural language, action = a block move; math (GSM8k) state = values of intermediate variables, action = a subquestion; logic (PrOntoQA) state = a fact, action = choosing a rule. This is the explicit, structured-state framing (not pixels).
- **The trace is fully internal**: "the full reasoning trace is simulated by the LLM itself ... without interacting with the external real environment." This is the key property and the key risk for an advisory model: the predicted dynamics are never grounded mid-rollout.
- **Reward design (four options)**: (1) log-probability of the action (the LLM's "instinct" / prior); (2) confidence of the predicted state, estimated by sampling the world-model answer multiple times and taking the proportion of the most frequent answer (self-consistency as a confidence proxy); (3) self-evaluation, prompting "Is this reasoning step correct?" and using the next-token probability of "Yes"; (4) task-specific heuristics (Blocksworld: compare predicted state to the goal).
- **Planning = MCTS**: build a reasoning tree; each node is a state, each edge an action+transition. UCT for selection (Q value for exploitation, visit-count term for exploration), expansion samples `d` actions and predicts `d` next states, simulation rolls out under a lightweight reward, back-propagation updates Q. RAP-Aggregate ensembles answers across MCTS iterations for answer-only tasks.

## Numbers as stated

- Blocksworld (LLaMA-33B base): RAP nearly solves all 4-step cases and part of 6-step; average success 64%. On 6-step (search space up to 5^6) it finds a plan 42% of the time within 20 iterations. RAP on LLaMA-33B beats GPT-4-CoT by 33% relative gain.
- GSM8k (math): RAP 48.8% accuracy (single trace), +~3% with RAP-Aggregate, beating CoT and least-to-most + self-consistency.
- PrOntoQA (logic): 94.2% answer accuracy, 78.8% proof accuracy; +14% proof over CoT, +4.4% answer over self-consistency CoT.

## Reproducibility status

Self-reported, code released. Results are benchmark accuracies on standard datasets; no environment-verified deltas. The "world model" is an LLM prompted to emit the next state in text; its prediction quality is never measured against an external ground-truth transition, only indirectly through final task accuracy. So RAP demonstrates that internal-WM-guided search helps task accuracy, not that the internal state predictions are individually correct.

## Relevance to this repo

- **Mechanical import (high)**: RAP is the canonical "LLM as both proposer and transition model" pattern, and it is the cheapest-first predictor the repo names: an LLM prompted to emit the next structured state. The state/action-per-task instantiation (configuration-as-state, move-as-action) is a direct template for a prompt-based advisory WAM at the Physical and Material layers (block config -> possession/claim ledger). The self-consistency-as-confidence trick is a concrete, training-free calibration handle.
- **The load-bearing caution for the repo**: RAP's world model and reward are the SAME LLM, and the whole rollout is internal with no grounding. This is precisely the progress-laundering risk the directions report warns about (proposer must never be the scorer). RAP works because final task accuracy is checkable; the repo's analog is that the runtime VERIFIER, not the LLM's self-evaluation reward, must own the score. RAP-as-advisor is admissible only if (a) it never selects the executed action and (b) every predicted next-state is scored against the verifier's observed delta, never against the LLM's own "Yes"-probability.
- **Research contribution vs support**: re-deriving RAP is not novel. The contribution surface is making the state typed and social-material (not free-text), and replacing RAP's internal self-evaluation reward with an external verifier-scored reward, which RAP itself never does.

## WAM layer(s) informed

Physical and Material (clean state/action instantiation, internal look-ahead). Social only by extension and unproven: RAP never models another agent's response. The internal-only rollout makes it least safe at Social/Institutional where errors compound and there is no mid-rollout grounding.
