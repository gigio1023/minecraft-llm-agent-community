- **title**: RT-1: Robotics Transformer for Real-World Control at Scale

- **arxiv_id**: 2212.06817

## Primary-source facts

- **Research question**: can one high-capacity, language-conditioned Transformer
  policy absorb a large, diverse, task-agnostic robot dataset and generalize
  zero-shot to new tasks, objects, and environments, while running fast enough for
  real-time control?
- **Model**: 35M-parameter pipeline. (1) 6-frame image history through an
  ImageNet-pretrained **EfficientNet-B3**, **FiLM-conditioned** on a Universal
  Sentence Encoder embedding of the instruction (early language fusion; FiLM layers
  identity-initialized so they do not disrupt pretrained features). Output: 81
  vision-language tokens per image. (2) **TokenLearner** soft-selects 81 -> 8
  tokens per image (6 images -> 48 tokens). (3) **decoder-only Transformer** (8
  self-attention layers, 19M params) emits action tokens. ("FiLM" = Feature-wise
  Linear Modulation, a way to condition a vision network on text by scaling/shifting
  its feature maps.)
- **Action space**: 11 dimensions: 7 arm (x, y, z, roll, pitch, yaw, gripper), 3
  base (x, y, yaw), 1 discrete mode-switch (arm / base / terminate). **Each
  dimension discretized into 256 uniform bins.** This is the root "action
  tokenization" scheme.
- **Observation**: RGB images (300x300) + instruction text. The model does NOT
  receive structured inventory/state (pixels + language only).
- **Training objective**: **behavioral cloning** ("behavior cloning" = supervised
  imitation: minimize negative log-likelihood of the demonstrator's action given the
  observation). Standard categorical cross-entropy over the action-token bins with
  causal masking.
- **Dataset shape**: ~130k human teleoperated demonstrations, 700+ task
  instructions (744 in the skill table), collected with a fleet of **13 robots over
  17 months** in office-kitchen "robot classrooms." A "task/instruction" = verb +
  noun(s), e.g. "pick coke can," "move pepsi can near rxbar," "open the top drawer."
- **Evaluation**: success rate over 3000+ real-world trials. 97% on 700+ training
  instructions; generalization to new tasks / distractors / backgrounds 25% / 36% /
  18% better than the best baseline (Gato, BC-Z); chains into SayCan long-horizon
  plans (up to 50 stages). Absorbs simulation and other-robot data without losing
  performance on original tasks.
- **Inference**: 3 Hz closed-loop (the human-speed target was 2-4 s/instruction,
  i.e. <100 ms/step). TokenLearner + caching/reusing image tokens give 2.4x and 1.7x
  speedups. Deliberately small (35M) so it runs in real time; explicitly removes
  Gato's autoregressive action decoding for speed.

## Interpretation (inference)

- RT-1's lasting contribution to the VLA lineage is **action tokenization**: cast
  continuous motor commands into 256 discrete bins per dimension and predict them
  with a sequence model under behavioral cloning. RT-2, OpenVLA, and (for its
  high-frequency failure mode) FAST all inherit or react to this exact scheme.
- It is a **pure VLA** in the survey's sense (`p(a|o,l)`): reactive, no model of how
  the world will evolve. There is no `o'` prediction anywhere. So it is the "act"
  half a WAM fuses with a "predict" half, not a WAM.

## Mechanically useful vs research contribution

- **Useful for THIS repo**: the *principle* that a typed, bounded action vocabulary
  plus a sequence model is enough to drive a robot from instructions. The repo
  already works at a *higher* altitude (typed Action Cards with logical params), so
  it does not need 256-bin motor tokenization; the transferable idea is "bounded
  discrete action space + language conditioning," which the repo realizes via schema-
  bound tool calls rather than learned bins.
- **Avoid / overclaim**: RT-1's results are pixel-to-motor manipulation; the 97% /
  generalization numbers say nothing about social or material-economic state. Do not
  treat RT-1 as evidence a Minecraft *social* controller is solved. It establishes
  only the Physical-layer substrate.
