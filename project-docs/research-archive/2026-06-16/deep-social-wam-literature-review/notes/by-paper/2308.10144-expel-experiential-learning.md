## Primary-source facts (what the paper states/shows)

- Goal: enable an LLM agent to learn from its own experience across many tasks WITHOUT parameter updates (so it works with closed-source models like GPT-4 / Claude that expose only API calls). Analogy: a student studies practice problems (training stage), then takes the exam in a single attempt (evaluation) using only the insights and recalled memories.
- Three stages.
  1. Gather experiences: interact with training tasks by trial and error via Reflexion-style retries, store both successes and failures into an experience pool (collecting success/failure pairs for later comparison).
  2. Extract insights (cross-task knowledge): an instruction-following LLM (gpt-4-0613) is iteratively shown fail/success pairs or lists of L successes and edits a growing set of natural-language insights via operators ADD, EDIT, UPVOTE, DOWNVOTE. Each new insight starts with an importance count of 2; UPVOTE/EDIT increment it, DOWNVOTE decrements it, count 0 removes it. This robustifies against suboptimal-but-successful trajectories misleading the insights.
  3. Task inference (evaluation): augment the task specification with the full concatenated insight list, and retrieve top-k most task-similar successful trajectories from the pool as few-shot in-context examples. No retries at deployment.
- Transfer learning: extracted insights from a source distribution (HotpotQA) are "fine-tuned" (rewritten by the LLM, grounded with a few target few-shot examples) for a target distribution (FEVER).
- Numbers as stated (4 text benchmarks, four-fold validation, success rate):
  - Both learning modes are synergistic: insights-only vs retrieve-only give 36%/31% (HotpotQA) and 50%/55% (ALFWorld), and ExpeL (both) beats each restricted mode.
  - Cross-task vs Reflexion: ExpeL matches Reflexion on HotpotQA (40% at R3 vs 39%) and exceeds it on ALFWorld (54% at one attempt vs 59% at R3) WITHOUT repeated attempts. WebShop remains near the low end of Reflexion's range.
  - ExpeL+Reflexion is synergistic on ALFWorld (up to 59.0% / 60.4% / 63.4% / 64.2% across rounds).
  - Transfer HotpotQA -> FEVER: ExpeL Transfer 70%, beating Act/ReAct and an insight-transfer-without-demos baseline; the few-shot "fine-tuning" of insights helps more than transfer without demos.
- Qualitative emergent behaviors observed: the agent reassesses its whole trajectory and concludes rather than saying "Unknown"; updates "world-model beliefs" (in ALFWorld, shifts its prior on a pan's location from drawers/countertops to stoveburners after an insight about an item's typical usage); self-corrects (puts back a wrongly-taken object and resumes).

## Reproducibility status

- Self-reported, with public code. Four standard text benchmarks, four-fold validation with mean and standard error reported, GPT-4 / GPT-3.5 as the insight extractor. Not environment-verified by me; numbers quoted from the paper.

## Interpretation (my inference, labeled)

- ExpeL is the canonical gradient-free experiential-learning archetype: it operationalizes "the agent accumulates and reuses verified experience across episodes" (the lane's experiential/lifelong angle) entirely in natural language. The key distinction from Reflexion is cross-task (inter-trial) retention vs intra-task retries; this is precisely the "carry a lesson across cycles" property the repo's multi-cycle actor needs.
- The insight-extraction operator set (ADD/EDIT/UPVOTE/DOWNVOTE with an importance count that decays to removal) is a concrete, simple curation-and-forgetting mechanism for distilled cross-cycle knowledge. It is the same idea as MemoryBank's recall-strengthens-retention and TierMem's retrieve-and-edit, expressed for abstracted insights rather than raw memories.
- The repo-relevant caution: ExpeL's insights are free natural-language prose, verified only by whether the downstream task succeeded. There is no per-insight citation to a concrete artifact. For the repo this is the un-hardened version: an insight like "pans are on stoveburners" would, in the repo's regime, have to cite the verified inventory/world events that support it, not float as prose.

## Mechanically useful vs research contribution

- Mechanically useful: gradient-free cross-task insight accumulation (works with API-only models); the importance-count curation operators as a forgetting/consolidation primitive; the synergy result (abstraction + recall together beat either alone) as a design argument for keeping both a distilled-insight store and a raw-trajectory retrieval store; insight "fine-tuning" with target few-shots as a cheap transfer mechanism.
- Research contribution it is NOT for this repo: ExpeL's contribution is a learning paradigm; it has no typed state, no transition prediction, no verifier beyond task success, and no social/material layer. Its memory asserts (insights) without pointing at evidence. It is a support mechanism (how an actor reuses experience), not the predictive object, and it must be hardened with evidence-linking before it fits the repo's hard line.

## WAM layer(s) informed

- Cross-cycle continuity substrate (how distilled experience persists and is reused), relevant to all layers but demonstrated only on physical/text task success. Its "world-model belief update" is a belief-revision-from-experience pattern relevant to the Social layer's beliefs-about-others, but ExpeL has no grounding that would make such an update admissible under the repo's rule.
