## Primary-source facts (LaTeX-verified)

- **The motivating puzzle.** Self-improvement (a model evaluating and refining its own generations with no external feedback) seems to contradict the data-processing inequality, which "implies that no form of self-training should be able to create information not already in the model." So why would supervision-free self-improvement add capability?
- **The answer: sharpening (a reframing).** Self-improvement does NOT create information; it is reframed as **sharpening**: using the model itself as a verifier during post-training to tilt the base policy toward responses with high self-reward (high model certainty), thereby AMORTIZING the expensive inference-time search for good sequences. The paper states the distinction sharply: self-improvement is "a computational phenomenon rather than a statistical one." It extracts "hidden knowledge" the model can verify but struggles to generate.
- **Why this is possible (the generation-verification gap).** Models are often better at VERIFYING response quality than GENERATING correct responses. This gap is grounded in complexity theory: computing the most likely response is NP-hard in the worst case (proved in the paper's hardness section), whereas evaluating a given response's likelihood is easy. Sharpening leverages cheap verification to improve costly generation.
- **Formal definition.** Sharpening tilts the base policy pi_ref toward `argmax_y r_self(y | x; pi_ref)`, where the self-reward r_self is derived purely from the base model (examples: sequence log-probability, model-as-judge, model confidence) with no external supervision. A concrete minimal self-reward studied is `r_self(y|x) = log pi_ref(y|x)` (maximum-likelihood sharpening).
- **Two algorithm families analyzed.** SFT-based (Best-of-N: filter high-self-reward samples, fine-tune on them) and RLHF-based (PPO/DPO directly on the self-reward).
- **Main theoretical results (the limits).** (i) The SFT/Best-of-N approach is **minimax optimal WHENEVER the base model has sufficient COVERAGE** (places enough probability mass on high-quality responses). (ii) The RLHF approach can IMPROVE over SFT by leveraging on-policy EXPLORATION, **bypassing the coverage requirement**. Lower bounds establish the fundamental role of coverage.
- **Empirical validation.** Maximum-likelihood sharpening via Best-of-N (N=50) gives non-trivial accuracy gains over greedy decoding across 7 models and 6 tasks (e.g. MATH, GSM, Game-of-24). Correct completions sit at noticeably higher sequence log-probability than incorrect ones, demonstrating the model's own certainty correlates with quality.

## Interpretation (flagged as inference)

- This is the theoretical heart of the "self-improvement only SHARPENS, it does not add new capability" bound. The data-processing-inequality framing makes it formal: a self-improvement loop with NO external signal cannot exceed what the base model already covers; it can only make accessible (amortize) knowledge already latent in the model. Capability the model has zero coverage of cannot be sharpened into existence (SFT case); exploration (RLHF) widens reach but still cannot manufacture a capability with no path.
- The coverage condition is the precise, citable statement of the repo bound: a verifier-grounded loop will make an advisory WAM BETTER-CALIBRATED where the verifier checks it and where the base model already has coverage; it will NOT make the WAM "socially smart" about phenomena outside the base model's coverage and outside the verifier's reach.
- Crucial distinction from H5: sharpening uses the MODEL ITSELF as verifier (self-reward) and is therefore the SELF-EVALUATED regime. It works for capability that the model can already verify (the generation-verification gap). It is NOT a license for the repo to let the actor score its own social success: sharpening's gains are bounded by the self-verifier's accuracy, and where the model is a poor self-verifier (contested social outcomes), sharpening adds nothing and can entrench error. The repo should sharpen against the EXTERNAL runtime verifier, not the actor's self-judgment, wherever an external check exists.

## Mechanically useful vs research contribution

- **Mechanically useful**: the coverage diagnostic (before running a self-improvement loop, check whether the base model places any mass on good outcomes for the target distribution; if coverage is zero, the loop cannot help and needs new data or exploration); the inference-time-vs-amortized framing (Best-of-N against the runtime verifier is a cheap, theoretically-clean way to improve outputs without weight updates); the generation-verification-gap idea (use the cheaper verify step to drive the costly generate step).
- **Not a contribution to copy**: sharpening is a lens/analysis, not a method to claim. The repo borrows the BOUND (no new capability without external information) as a guardrail on overclaiming.
- **Honest bound it supplies**: self-improvement amortizes existing latent ability up to the base model's coverage; it cannot create capability the model lacks. Where the model is a good self-verifier it sharpens; where it is a poor self-verifier (contested social judgments) it does not. Therefore do not claim an autoresearch loop will make the WAM socially intelligent; claim only better calibration where an external verifier (or the model's own reliable verification) checks it.
