- **title**: Language Agent Tree Search Unifies Reasoning Acting and Planning in Language Models
- **authors**: Andy Zhou, Kai Yan, Michal Shlapentokh-Rothman, Haohan Wang, Yu-Xiong Wang (UIUC, x.ai)
- **year**: 2023 (arXiv 2023-10-06; v2 2024-06)
- **venue/source**: ICML 2024
- **arxiv_id**: 2310.04406
- **urls**: paper https://arxiv.org/abs/2310.04406 ; code https://github.com/lapisrocks/LanguageAgentTreeSearch
- **source availability**: latex (tarball extracted; main.tex deep-read)

## Primary-source facts (LaTeX-verified)

- **What LATS is**: MCTS over a tree of ReAct-style reasoning+acting steps, where the SAME LLM is repurposed as agent, value function (state evaluator), and self-reflection generator. Six operations: selection (UCT), expansion, evaluation, simulation, backpropagation, and reflection.
- **The defining design choice for this lane**: LATS deliberately does NOT use an internal world model. Quote: "While standard MCTS and RAP rely on internal dynamics models to facilitate simulation, LATS uses environment interaction and does not require a world model." It grounds each step in real environment feedback instead of an LLM-predicted next state.
- **Why it drops the internal WM (its critique of RAP/ToT)**: it lists three shortcomings of reasoning-only methods: Flexibility (autoregressive, no alternatives), Sensibility ("Reasoning-based methods (CoT, RAP, or ToT) rely solely on the internal representations of the LM and cannot consider external observations. This dependency risks fact hallucination and error propagation while setting a performance ceiling"), and Adaptability. Critically: "While RAP also adopts MCTS, it is constrained to tasks where the LM can become a world model and accurately predict states."
- **Value function**: repurpose the LLM into a scorer, but obtain the score AFTER environment feedback (its key distinction from ToT, which scores before feedback), plus a self-consistency heuristic. Self-reflection on failed trajectories is stored in memory and fed back as a "semantic gradient."
- **Tree search is cheap for LLMs because state reset is free**: "we can conveniently reset to any step by simply copy-pasting historical text input," so the usual MCTS requirement of an environment model to undo steps does not bind. (Note the asymmetry: this holds for reversible text tasks, not for irreversible real-world actions, which is why WebDreamer keeps the WM internal.)

## Numbers as stated

- HumanEval programming: 92.7% pass@1 (abstract/v2) / 94.4% (v1 body) with GPT-4, state of the art.
- WebShop web navigation: average score 75.9 with GPT-3.5, gradient-free, comparable to gradient-based fine-tuning; +22.1 average over ReAct.
- HotPotQA: roughly doubles ReAct performance (GPT-3.5), with an oracle correctness signal from the environment.

## Reproducibility status

Self-reported, code released. Uses an oracle correctness signal on HotPotQA (acknowledged), so some results assume high-quality external feedback. The point relevant here is structural, not the exact numbers.

## Relevance to this repo

- **This is the lane's strongest external evidence for the repo's hard rule**: LATS is a frontier-quality system that explicitly chose to NOT trust the LLM's internal dynamics model, citing hallucination, error propagation, and a "performance ceiling." It uses external feedback as the grounding signal. The repo's deterministic verifier IS that external feedback, available at near-$0 and per-step. So LATS is independent confirmation that an LLM internal world model alone has a ceiling and that grounding fixes it.
- **The honest tension to record**: LATS grounds by ACTING in the environment and resetting via text replay. The repo cannot reset social state by "copy-pasting text"; physical/material state can be reset by clean scenario reload (the directions report's reset-granularity question), but social history is not text-revertible. So LATS's free-reset assumption holds for the repo only at Physical/Material with clean scenario resets, not at Social/Institutional.
- **Mechanical import (medium)**: the six-operation loop and the "score AFTER feedback, not before" rule are directly portable. The repo's version: the advisory WAM proposes a predicted delta, the runtime executes and verifies, the verifier's observed delta (not the LLM's pre-action guess) is the score, and a Reflexion-style memory of mispredictions feeds back.
- **Research contribution vs support**: LATS is a method to borrow, not a result to claim. The contribution is using a deterministic structured-state verifier (cheaper and more reliable than LATS's environment-interaction-plus-oracle) as the external feedback.

## WAM layer(s) informed

Method-level. Strongest argument that internal LLM dynamics need external grounding (all layers). Its free-reset assumption ties it to Physical/Material with clean resets; it does not solve the non-revertible Social/Institutional reset problem.
