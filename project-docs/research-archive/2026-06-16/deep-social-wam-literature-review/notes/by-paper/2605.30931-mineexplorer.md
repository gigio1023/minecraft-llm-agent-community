# MineExplorer - Evaluating Open-World Exploration of MLLM Agents in Minecraft

- **authors**: see paper; code Jometeorie/MineExplorer
- **year**: 2026 - **venue**: arXiv (ACL style)
- **arxiv_id**: 2605.30931
- **urls**: https://arxiv.org/abs/2605.30931 ; code https://github.com/Jometeorie/MineExplorer

## Primary-source facts

- **Research question**: can MLLM agents *sustain* exploration in dynamic open
  worlds, separating general open-world reasoning from Minecraft-specific priors?
- **Knowledge decoupling**: from 3,382 Minecraft tasks they **filter out atomic
  tasks whose success depends mainly on Minecraft-specific conventions**, keeping
  **1,497 knowledge-controlled atomic tasks**. The aim is to test reasoning, not
  memorized game lore.
- **Composite tasks**: atomic tasks are composed into **implicit multi-hop tasks**
  with a **latent task dependency graph**; difficulty is defined by aggregating
  capability load over **prerequisite paths**. **813 human-validated composite
  instances**, one-hop to four-hop.
- **Construction - multi-agent synthesis workflow**: a task selector, scene
  designer, **milestone agent**, Minecraft expert, and validator jointly produce
  the problem statement, the environment, the dependency graph, and a **rule-based
  milestone evaluator**. After a candidate graph is proposed, the scene is
  rendered in Minecraft and fed back so the agents adjust design + milestone
  rules to what actually appears. Human eval confirms the synthesis workflow
  yields significantly more reliable instances than a single-agent baseline.
- **Findings**: strong models handle many single-hop tasks but **degrade sharply
  when hidden prerequisites must be coordinated over longer trajectories**. Larger
  models / thinking modes do not consistently help. Milestone evaluator is
  reliable under human eval.

## Interpretation (inference)

- MineExplorer is a **task benchmark with rule-based milestone verifiers** built
  around **hidden multi-hop dependency graphs**. Its central empirical result -
  competence collapses on *latent prerequisite coordination* - is exactly the
  "physical predictions must be reliable before social claims" dependency this
  repo names: an agent that cannot infer "I need a pickaxe before I can mine"
  will never sustain a social claim that depends on that fact.
- The **rule-based milestone evaluator** (not VLM scoring) is the right kind of
  truth source; the **dependency graph** is the right kind of structure. Both are
  borrowable as engineering.

## Mechanically useful vs research contribution

- **Mechanically useful**: (a) the **latent dependency graph + milestone
  evaluator** pattern maps directly onto this repo's PlanBeadGraph + verifier
  evidence; (b) the **knowledge-decoupling** filter is a useful way to keep
  scenarios about *reasoning/social pressure* rather than Minecraft trivia; (c)
  the multi-agent synthesis-then-render-then-adjust loop is a reusable way to
  author scenario fixtures that actually instantiate in-world.
- **What would be overclaim**: treating multi-hop *task* completion as a social
  result. MineExplorer measures prerequisite coordination by one agent, not
  obligations/claims between actors. It is a competence-gate benchmark.
