# AVID - Adapting Video Diffusion Models to World Models

- **title**: AVID: Adapting Pretrained Video Diffusion Models to World Models
- **year**: 2024 (ICLR 2025)
- **venue/source**: ICLR 2025
- **arxiv_id**: 2410.12822
- **urls**: code https://github.com/microsoft/causica/tree/main/research_experiments/avid ; project https://sites.google.com/view/avid-world-model-adapters ; LaTeX at `papers/latex/2410.12822/`
- **source availability**: latex (full)

## Primary-source facts

- **Problem**: action-labelled data is scarce; the strongest video diffusion models are closed-source (no parameter access), so you cannot finetune or add ControlNet-style conditioning.
- **Method (AVID)**: train a domain-specific **adapter** on a small action-labelled video dataset that **modifies the noise predictions** of a frozen (or even API-only) pretrained image-to-video diffusion model, using a **learned mask** to combine pretrained outputs with the adapter's conditional outputs - producing an action-conditioned world model **without access to the base model's parameters**.
- **Eval**: video-game data + real-world robotics (uses a 1.4B internet-pretrained model, DynamiCrafter, as the base). AVID outperforms diffusion-adaptation baselines and "obtains a considerable benefit from using the pretrained model, even with limited access."
- Advocacy: providers of closed video models should expose intermediate outputs in APIs.

## Interpretation

- This is the canonical answer to the lane question "can existing weights be used without training from scratch?" - yes, via an **adapter over a frozen video model**, even with only noise-prediction access. It demonstrates weight reuse is feasible for *pixel/video* WAMs.
- But the output is still pixel video, and the demonstrated domains are robotics and video games (not structured state). So AVID's reuse path produces a *visual* world model. For a structured-state social WAM, the directly reusable substrate is not a video model's pixels; AVID would only help if the project wanted a pixel-imagination sidecar (e.g., a Minecraft visual rollout for human review), which the project explicitly does not need as runtime authority.

## Mechanically useful vs research contribution

- **Useful**: the adapter-over-frozen-backbone pattern as the general "reuse existing weights" template, and the explicit constraint that closed weights can still be adapted via noise predictions. Useful to cite when justifying that the project is NOT blocked on training a WAM from scratch - but also that the available reusable weights are pixel models, which is a poor fit for structured social state.
- **Avoid/overclaim**: do not claim AVID-style adaptation gives a social-material predictor; it gives a pixel video predictor.
