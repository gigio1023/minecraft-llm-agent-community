## Primary-source facts (LaTeX-verified)

- **The problem ADAS names**: "Automated Design of Agentic Systems" = use a search algorithm to discover agentic systems across a search space that optimize an evaluation function. Three components (their Fig 2): (1) Search Space (which agents are representable), (2) Search Algorithm (how the space is explored), (3) Evaluation Function (how a candidate is scored).
- **The key claim about the search space**: define the ENTIRE agentic system in code. Because most programming languages are Turing-complete, a code search space "theoretically enables ... any possible agentic system: including novel prompts, tool use, workflows, and combinations thereof." This is positioned as strictly more general than prior work that searches only prompts (PromptBreeder, OPRO) or only graph connectivity (GPTSwarm, DyLAN).
- **The algorithm = Meta Agent Search**: a meta agent (a foundation model, GPT-4 in their runs) iteratively programs new agents in code. Loop: (1) archive optionally seeded with baselines (CoT, Self-Refine, etc.); (2) conditioned on the archive, the meta agent writes a high-level idea then implements a `forward` function (the agent), with two self-reflection passes for novelty; (3) the new agent is evaluated on validation data, and on errors the meta agent self-reflects to fix, up to 5 times; (4) the agent plus its measured metric is added to the archive; repeat to a fixed iteration budget. The framework given to the meta agent is "within 100 lines of code" (FM-query and prompt-format helpers); the meta agent only writes the `forward` function (like FunSearch).
- **Evaluation function used**: numerical task performance (accuracy or F1) on a held-out validation set, "commonly adopted in existing works." The meta agent is told to maximize this number.
- **Domains and results (as stated)**: ARC (20-question validation, 60-question held-out test, GPT-3.5 evaluated, GPT-4 meta agent, 25 iterations); and four QA benchmarks (DROP reading comprehension, MGSM math, MMLU multi-task, GPQA science), 30 iterations each, searched independently per domain. Meta Agent Search beats the strongest hand-designed baseline in every domain: +13.6 F1 on DROP, +14.4% on MGSM math; it also beats the OPRO prompt-optimization baseline in all four domains (e.g. math 53.4 vs OPRO 30.6).
- **Transfer / generalization (their headline robustness claim)**: top-3 agents found on MGSM transfer to held-out math (GSM8K +25.9%, GSM-Hard +13.2% over baselines) AND to non-math domains (MMLU, DROP) where they still beat hand-designed baselines though not domain-specialized agents. Agents found with GPT-3.5 on ARC transfer to Claude-Haiku, GPT-4, Claude-Sonnet and keep their advantage.
- **Insight on FM-dependence (their own observation)**: the best GPT-3.5 ARC agent uses a complex feedback mechanism; transferred to stronger models a simpler agent with more refinement wins. They read this as GPT-3.5 being weaker at evaluating/refining its own answers, so it needs more scaffolding.
- **Stated limitations**: (a) only single-step QA, not multi-step interactive environments; (b) "we simply evaluate discovered agents on the evaluation set and use the numerical performance results. However, this approach is both expensive and misses a lot of information"; (c) single objective (performance) only, no cost/latency/robustness; (d) subjective tasks without ground truth need new evaluation functions. Future work explicitly raises "higher-order ADAS" (the meta agent improves itself, self-referential) and the connection to "the origins of complexity emerging from human organization and society."
- **Safety**: generated code runs containerized + manually inspected (cites SWE-Bench, Voyager as precedent).

## Interpretation (flagged as inference)

- ADAS is the canonical statement of "the agent improves the agent" as a search problem, and its three-component decomposition (space / algorithm / evaluation) is the cleanest framing to organize this whole lane. AFlow, GPTSwarm, OPRO, DSPy, PromptBreeder are all special cases differing in which of the three they fix.
- The most relevant fact for this repo's thesis is the evaluation function. ADAS optimizes a single number (validation accuracy) and the authors themselves call this "expensive and misses a lot of information." That is exactly the lever a runtime verifier provides: it scores transitions on physical truth, not a single aggregate.
- ADAS reports real held-out test sets and cross-domain/cross-model transfer, which is genuine evidence AGAINST a naive "all agent search overfits" story. The honest reading: code-space agent search CAN find transferable design patterns; whether the GAIN is robust depends entirely on the evaluation function being external and hard to game, which ADAS partly satisfies (held-out test) but does not stress (single aggregate accuracy, QA only).

## Mechanically useful vs research contribution

- **Mechanically useful for this repo**: (1) the archive-of-discoveries + interestingness loop as a record of what agent variants were tried (mirrors ENPIRE's idea-tree); (2) the meta agent writing a `forward` function against a tiny fixed helper API is directly analogous to this repo's `author_mineflayer_action` (the LLM authors code against a bounded helper API, runtime executes); (3) the "evaluate on held-out validation, self-reflect to fix runtime errors up to N times" loop maps onto the repo's retry/verifier gates.
- **What would be overclaim**: ADAS's research contribution is the code-space ADAS formulation + Meta Agent Search on QA. Do NOT claim "ADAS for Minecraft" or that auto-designed agents are validated for embodied multi-step social tasks; the paper explicitly scopes to single-step QA and lists multi-step environments as future work. Borrow the formulation and loop, not the result.
- **Honest bound for the thesis**: an auto-designed Actor Turn prompt or advisory-WAM prompt must be scored by the runtime verifier on HELD-OUT scenarios. ADAS shows transfer is achievable but its evaluation is a single aggregate number; a social Minecraft world has no such clean aggregate, so the verifier-scored, held-out discipline matters more here, not less.
