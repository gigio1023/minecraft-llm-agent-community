# Optimus-3 - Generalist Multimodal Minecraft Agents with Scalable Task Experts

- **authors**: Li et al. (JiuTian-VL)
- **year**: 2025 - **venue**: arXiv (NeurIPS 2025)
- **arxiv_id**: 2506.10357
- **urls**: https://arxiv.org/abs/2506.10357 ; code https://github.com/JiuTian-VL/Optimus-3 ; project https://cybertronagent.github.io/Optimus-3.github.io/

## Primary-source facts

- **Research question**: build a single generalist Minecraft agent with
  perception, planning, action, grounding, and reflection, despite scarce
  domain data, interference among heterogeneous tasks, and open-world visual
  diversity.
- **Three contributions**:
  1. a **knowledge-enhanced data generation pipeline** for scalable high-quality
     training data;
  2. a **Mixture-of-Experts (MoE) architecture with task-level routing** to reduce
     cross-task interference (the "scalable task experts");
  3. **Multimodal Reasoning-Augmented RL** to handle visual diversity.
- **Capabilities**: one model spanning perception, planning, action, grounding,
  reflection; reports surpassing generalist MLLMs and prior SOTA Minecraft agents
  across a wide task range.
- **Interface**: MLLM high-level + low-level control (Optimus-2 GOAP lineage);
  pixels + language in.

## Interpretation (inference)

- Optimus-3 is a **generalist VLA/agent** (planner + executor), not a WAM - no
  explicit predictive `o'` is co-generated for action selection. Its real lesson
  is **architectural**: MoE **task-level routing** to stop heterogeneous tasks
  from interfering. That interference problem is relevant if this repo ever asks
  one model to handle physical, material, and social reasoning at once - routing
  by concern is a plausible mitigation.

## Mechanically useful vs research contribution

- **Mechanically useful**: the MoE/task-routing idea as a *future* option if a
  single provider must juggle distinct concern types; the knowledge-enhanced data
  pipeline as a data-gen pattern (overlaps Lane 5's territory).
- **What would be overclaim**: treating Optimus-3's broad task competence as
  social capability; it is multi-task competence breadth, scored on task success.

## WAM layer(s) informed

Physical-WAM (generalist executor + planning). Reproducible: code public; full
training pipeline partial.
