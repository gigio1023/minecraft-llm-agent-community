## Primary-source facts

- **Research question**: how far can a VLA generalize "in the wild" - to homes never seen
  in training? Goal is open-world generalization for mobile manipulation.
- **Model**: built on pi_0 (flow-matching action expert). Two-stage recipe: (1)
  pre-training on a heterogeneous mixture with **discrete (FAST-tokenized) actions** plus
  web data and high-level semantic prediction; (2) post-training that adds a flow-matching
  **action expert** for fine continuous control. The model is one transformer that emits
  both text tokens (subtasks / VQA) and action chunks.
- **Hierarchical decomposition (verbatim form)**:
  `pi(a_{t:t+H}, hat{l} | o_t, l) = pi(a_{t:t+H} | o_t, hat{l}) * pi(hat{l} | o_t, l)`,
  where `hat{l}` is a predicted **high-level semantic subtask** (e.g. "pick up the plate")
  and the low-level action depends only on `hat{l}`, not the original prompt `l`. At
  inference: first infer the subtask (slow), then the action chunk (fast).
- **Data mixture**: ~400 hours of mobile-manipulator data, but **97.6% of first-phase
  training examples are NOT mobile-manipulation** - they are other robots, lab tasks,
  web image-caption/VQA/localization, verbal human-supervisor instructions, and
  high-level semantic-subtask prediction. Co-training across these heterogeneous sources
  is the central contribution.
- **Results**: first end-to-end learned system to do long-horizon (10-15 min) dexterous
  mobile manipulation (cleaning a kitchen/bedroom, making beds, hanging towels) in
  entirely new homes. Ablations show the heterogeneous transfer is what enables it.
- **Observation / action**: images + proprioception in; FAST-discrete (pretrain) or
  flow-matching continuous action chunks (post-train) out. Pixel/motor, gripper + mobile
  base. No structured social or possession state.

## Interpretation (clearly labeled, distinction-relevant)

- **pi_0.5 is a hierarchical VLA, not a WAM.** Its intermediate token `hat{l}` is a
  predicted *language subtask* (the VLA survey's "language description" action token), not
  a predicted future *observation* `o'`. There is no forward-predictive commitment to how
  the world will look, and no coupling of the action to a forecasted state - it couples
  the action to a *semantic plan*. So even the most generalization-focused recent VLA does
  not cross the WAM criteria. This is a precise counterexample to "predicting something
  intermediate makes it a WAM."
- It does show that the *VLA recipe itself* (heterogeneous co-training) buys a lot of
  open-world generalization without any world model - relevant to keeping the
  WAM-vs-VLA trade balanced: VLAs are not stuck, they generalize via data breadth.

## Mechanically useful vs research contribution

- **Useful**: the "predict a high-level semantic subtask, then act conditioned on it"
  structure rhymes with this repo's LifeGoal / PlanBead / CycleGoal layering - but in this
  repo the LLM proposes the goal and the runtime owns truth, which is the opposite of an
  end-to-end learned policy. Borrow the hierarchy idea conceptually only.
- **Avoid / overclaim**: pi_0.5's generalization claims are about gripper/mobile
  manipulation in homes; not transferable to social-material Minecraft claims. Not a
  source for any WAM layer; it is a VLA reference point for the distinction.
