## Primary-source facts

- **Research question**: build an open robot foundation model for *generalist humanoid*
  control that reasons about novel situations, handles real-world variability, and learns
  new tasks data-efficiently.
- **Architecture (dual-system VLA)**: System 2 = a slow (~10 Hz) autoregressive VLM
  (Eagle-2, ~1.34B params) for high-level contextual reasoning/planning from vision +
  language; System 1 = a fast (~120 Hz) diffusion transformer (~0.86B params) for
  real-time motor control. The two are "tightly coupled and jointly trained end-to-end."
- **Data pyramid** (the survey credits GR00T N1 with this framing): bottom = large-scale
  web + human video; middle = synthetic/simulation data (DexMimicGen); top = small-scale
  real robot data. GR00T N1 extracts **latent actions from human videos** and synthetic
  demos and combines them with real data for training.
- **Action / observation**: images + proprioception in; continuous motor actions out;
  cross-embodiment (deployed on Fourier GR-1 humanoid for bimanual manipulation). Pixel/
  motor; no structured social or material state.
- **Results (reported)**: outperforms state-of-the-art imitation-learning baselines on
  standard simulation benchmarks across multiple embodiments; strong real-robot bimanual
  manipulation with high data efficiency.

## Interpretation (clearly labeled, distinction-relevant)

- **GR00T N1 is a VLA, not a WAM**, even though it has a "System 2" that plans: System 2
  produces high-level *reasoning/plan* outputs (semantic, the survey's language/reasoning
  action tokens), and System 1 maps to motor actions. Neither system commits to predicting
  a future observation `o'` under a world-modeling objective, and the action is not
  coupled to a forecasted state. So "dual-system" / "System 1 + System 2" is a *planning*
  hierarchy, not the forward-prediction-then-couple structure of a WAM. This is a useful
  guard against conflating "has a slow deliberate module" with "is a WAM."
- The "latent actions from human video" trick is the VLA way to use action-free video
  (infer pseudo-actions); a WAM uses action-free video differently (to learn the
  forward dynamics `p(o'|o)` directly). Same data source, different objective.

## Mechanically useful vs research contribution

- **Useful**: the dual-system cadence (slow deliberation + fast execution) is a useful
  reference for how a slow advisory predictor could coexist with a faster Actor Turn, but
  in this repo the runtime (not a learned System 1) owns execution truth. The data-pyramid
  framing is a general lesson on mixing abundant-but-weak and scarce-but-precise data.
- **Avoid / overclaim**: humanoid manipulation results are not transferable to social
  Minecraft; GR00T is a VLA reference point, not a source for any WAM layer. Body-text
  details (exact params, Hz) are from the abstract/survey, not a full LaTeX read - treat
  precise numbers as reported, not independently verified from the paper body.
