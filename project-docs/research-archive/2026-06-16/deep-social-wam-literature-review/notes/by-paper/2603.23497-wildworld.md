# WildWorld — action-conditioned world modeling with explicit state annotations

- **title**: WildWorld: A Large-Scale Dataset for Dynamic World Modeling with Actions and Explicit State toward Generative ARPG
- **year**: 2026
- **venue/source**: arXiv
- **arxiv_id**: 2603.23497
- **urls**: project https://shandaai.github.io/wildworld-project/ ; LaTeX at `papers/latex/2603.23497/`
- **source availability**: latex (full)

## Primary-source facts

- **Core argument**: dynamical-systems/RL view world evolution as **latent-state dynamics driven by actions**, with visual observations as partial projections. Existing video-WM datasets "lack diverse and semantically meaningful action spaces, and actions are directly tied to visual observations rather than mediated by underlying states. As a result, actions are often entangled with pixel-level changes, making it difficult for models to learn structured world dynamics."
- **Key example (verbatim spirit)**: "the action 'shoot' implicitly affects internal state variables such as the 'remaining ammunition count'. This state cannot be reliably inferred from visual observations alone, yet it plays a crucial role in determining future visual outcomes." When ammo hits zero, "shoot" produces no projectile — a visual outcome that depends on hidden state.
- **Dataset**: **108M+ frames** auto-collected from the AAA ARPG *Monster Hunter: Wilds*; **450+ actions** (movement, attacks, skill casting); synchronized **per-frame ground-truth annotations of character skeletons, world states, camera poses, depth maps**.
- **Benchmark (WildBench)**: two metrics — **Action Following** (generated video matches ground-truth sub-actions) and **State Alignment** (track skeletal keypoints in generated video, compare to ground-truth annotations).
- **Findings**: persistent challenges in modeling semantically rich actions and maintaining long-horizon state consistency; "highlighting the need for state-aware video generation."

## Interpretation

- WildWorld is the strongest external validation of the project's structured-state thesis. Its central claim — that meaningful actions change **internal state not visible in pixels**, so models must be **state-aware** rather than pixel-only — is exactly the project's argument transposed from an ARPG to a social Minecraft world. The Minecraft analogue of "shoot decrements hidden ammo count" is "lend_item changes hidden possession + creates a return obligation" — neither is recoverable from pixels alone.
- It still keeps video as the output and adds *state annotations as supervision/eval*; the project goes further by making structured social-material state the *primary* `o` and `o'`. So WildWorld is a halfway precedent: state-aware video, not pure structured-state transition.

## Mechanically useful vs research contribution

- **Useful**: (1) the explicit "actions act through hidden state, not pixels" argument as a citable foundation; (2) the **State Alignment** evaluation concept — measuring whether predicted transitions match ground-truth state, which is directly the project's "social-material transition prediction accuracy"; (3) the per-frame structured-annotation data design as a template for logging typed social-material state alongside actions.
- **Avoid/overclaim**: WildWorld's state is character skeletons/world geometry for combat, not social/possession/obligation state; do not claim it models social transitions. It is single-agent combat, not multi-actor social interaction.
