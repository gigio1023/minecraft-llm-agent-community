# SAIL / SILVR - Self-Adapting Improvement Loops for Robotic Learning

- **title**: Self-Adapting Improvement Loops for Robotic Learning (HF/arXiv title "SAIL"; the LaTeX source uses the working name "SILVR", Self-Improving Loops for Visual Robotic Planning). Both names refer to the same system.
- **authors**: Calvin Luo, Zilai Zeng, Mingxi Jia, Yilun Du, Chen Sun (Brown University; Harvard).
- **year**: 2025 (submitted 2025-06-10).
- **venue/source**: arXiv preprint.
- **arxiv_id**: 2506.06658
- **urls**: paper https://arxiv.org/abs/2506.06658 ; project page https://diffusion-supervision.github.io/sail/
- **source availability**: latex (tarball, 9 .tex files). Deep-read.

## Primary-source facts (what the paper states/shows)

- **Research question**: can a video generative model used as a visual planner *continuously improve* on a novel task from its own self-collected online experience, rather than being fixed after offline training?
- **Loop (Algorithm 1)**: given an in-domain video model, an inverse dynamics model (IDM), and an optional frozen internet-pretrained video prior, for K iterations: (a) optionally compose the in-domain model with the internet prior via training-free Inverse Probabilistic Adaptation (IPA, score composition); (b) reset env, roll out N trajectories by generating a visual plan and converting frame pairs to actions via the IDM; (c) filter rollouts with a data filter `f_r`; (d) finetune the in-domain video model on accumulated filtered data. Optionally distill the final planner into a lightweight behavior-cloning policy.
- **Action space**: continuous robot-arm control (MetaWorld; Franka Panda), actions derived from consecutive generated frames via the IDM.
- **State representation**: pixels (video frames). This is a Video Action Model style instantiation, not structured state.
- **Dataset shape**: MetaWorld - 25 demos from 8 tasks for initial training, then self-improvement on 12 unseen tasks, 30 rollouts/iteration. Real arm - 120 teleoperated demos (10 each of 12 color pairs), self-improve on 2 novel colors.
- **Training objective**: iterative supervised finetuning of the diffusion video model on filtered self-collected successful trajectories. No new RL objective; the "loop" is the contribution.
- **Evaluation protocol + metrics**: task success rate across 5 self-improvement iterations. MetaWorld with ground-truth success; real arm judged by a human. Reported: MetaWorld success rises across iterations (the abstract/intro cite up to 285% relative improvement over 10 iterations on novel tasks); SAIL with VLM filter goes 17.0 -> 38.4 over 5 iterations (Table). Baselines DSRL "fails to improve" and BCIL "quickly saturates at a low success rate."
- **Filter-signal ablation (central to this lane)**: the ground-truth success filter can be replaced by a VLM-as-judge (GPT-5, Gemini-2.5-Pro) without stopping self-improvement; Gemini gave the best VLM-filter result. **No filtering** gives only "marginal" improvement in MetaWorld, but on the real arm with the internet video prior, even no-filter "still facilitates continuous improvement."
- **Failure mode they report**: on the *real* arm *without* the internet video prior, "the visual planner struggles to improve or even continuously deteriorates." This is an explicit self-improvement-collapse signal: removing the external prior turns the loop degenerative.

## Interpretation (flagged as inference)

- SAIL is a clean, minimal instance of the ENPIRE-class loop (reset, rollout, verify/filter, refine) at the Physical layer, but without a coding agent: the "refiner" is plain finetuning, not code generation. That makes it the most transferable analog for a loop that improves a *predictor* (here a video planner) rather than rewriting code.
- The filter is the success signal. The result that a VLM judge works *almost* as well as ground truth is encouraging for domains with no oracle, but the same result shows no-filter degrades. The signal quality, not the loop, is the binding constraint.
- The deteriorate-without-prior result is the manipulation-domain echo of model collapse (see 2510.16657): self-training on a model's own outputs without an external information source can spiral down.

## Mechanically useful vs research contribution

- **Mechanically useful for this repo**: the loop skeleton maps onto the repo cycle (scenario reset -> Actor Turn -> action -> verifier evidence -> next cycle). The filter == the repo verifier. The "distill the improved planner into a cheap policy" step is a concrete pattern: an expensive advisory WAM could be distilled into cheaper guidance after a loop converges. The robustness-to-suboptimal-data result suggests a social loop need not start from expert demos.
- **Research contribution NOT to copy**: SAIL's claim is sample-efficient self-improvement of a *video planner* for *robot manipulation*. This repo is neither video-based nor doing manipulation. Do not frame the repo as "SAIL for Minecraft."
- **Where it bounds the thesis**: SAIL has a near-binary, resettable success metric (task done or not) at the Physical layer. The deteriorate-without-prior finding is a direct warning that a verifier/filter alone is not sufficient if the only information source is the model's own rollouts; an external grounding signal (here the internet prior; in the repo, the runtime verifier on physical truth) is what keeps the loop honest.
