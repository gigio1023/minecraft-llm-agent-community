# WALL-E: World Alignment by Rule Learning Improves World Model-based LLM Agents

- title: WALL-E: World Alignment by Rule Learning Improves World Model-based LLM Agents
- authors: Siyu Zhou, Tianyi Zhou, Yijun Yang, Guodong Long, Deheng Ye, Jing Jiang, Chengqi Zhang
- year: 2024 (ICLR 2025)
- venue: ICLR 2025
- arxiv_id: 2410.07484
- urls: paper https://arxiv.org/abs/2410.07484 , code https://github.com/elated-sawyer/WALL-E ,
  hf https://huggingface.co/papers/2410.07484
- source_availability: latex (papers/latex/2410.07484/main_file.tex)

## Primary-source facts (what the paper states)

- Research question: can an LLM serve as a world model for a model-based agent, and can the gap
  between the LLM's prior and the actual environment dynamics be closed cheaply?
- Core idea: "world alignment" by RULE LEARNING. The world model is the pretrained LLM PLUS a
  small set of learned symbolic rules. Rules are induced, updated, and pruned GRADIENT-FREE by an
  LLM, by comparing agent-collected real trajectories against world-model predictions. Pruning
  solves a maximum-coverage problem to keep only rules complementary to the LLM's own predictions.
- The agent is built on MODEL-PREDICTIVE CONTROL (MPC): it optimizes look-ahead actions against
  the LLM+rules world model before executing.
- Input / output: text-based environment state and action descriptions; the world model predicts
  next state (a transition prediction), checked against real transitions to refine rules.
- Evaluation: open-world Minecraft and ALFWorld. Reported (unverified beyond the paper's own
  tables): in Minecraft, +15-30% success rate over baselines, 8-20 fewer replanning rounds, and
  60-80% of the tokens; in ALFWorld, 95% success after 6 iterations.

## Interpretation (my reading, labeled inference)

- This is the closest existing system to the project's "LLM proposes; structured layer checks
  consequences" idea, and it is in Minecraft. The world model is explicitly NEURO-SYMBOLIC: a
  neural LLM plus an inspectable, editable rule set that captures hard environment constraints.
- The key architectural CONTRAST to flag: WALL-E uses its world model as PLANNING AUTHORITY (MPC
  selects actions from world-model rollouts). The project's repo rule keeps the WAM ADVISORY (it
  predicts/evaluates deltas, never selects/gates actions). So WALL-E is a mechanism source for
  "learn checkable transition rules from trajectory mismatches", not a control architecture to copy.
- The rule-induction-from-mismatch loop is directly reusable: compare a predicted social-material
  delta to the verifier-logged actual delta, and induce a corrective typed rule when they diverge.

## Mechanically useful vs research contribution

- Mechanically useful: (1) representing a world model as LLM + a minimal learned symbolic rule
  set; (2) inducing/pruning rules from prediction-vs-reality mismatch with a maximum-coverage
  objective; (3) the token-cost argument (a few principal rules beat stuffing trajectories into
  the prompt). All fit a structured, auditable Minecraft predictor.
- Avoid / overclaim: do not adopt WALL-E's MPC-as-authority loop (violates advisory rule). Its
  Minecraft results are task-success (race-to-objective) framed, which the project explicitly
  rejects as its target. WALL-E does not model social-material state (possession, obligation,
  trust); borrow the rule-learning mechanism, not a social claim.

## WAM layer(s) informed

- Physical primarily (Minecraft task dynamics). The rule-learning-from-mismatch loop is the
  cross-cutting mechanism that could, by interpretation, refine Material/Social transition rules.
