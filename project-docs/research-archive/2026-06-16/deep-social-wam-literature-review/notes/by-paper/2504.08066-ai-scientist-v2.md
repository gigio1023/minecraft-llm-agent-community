# The AI Scientist-v2: Workshop-Level Automated Scientific Discovery via Agentic Tree Search

- **title**: The AI Scientist-v2: Workshop-Level Automated Scientific Discovery via Agentic Tree Search
- **authors**: Yutaro Yamada, Robert Tjarko Lange, Cong Lu, Shengran Hu, Chris Lu, Jakob Foerster, Jeff Clune, David Ha (Sakana AI et al.)
- **year**: 2025 (arXiv 2025-04-10)
- **venue/source**: arXiv preprint; reports the first fully AI-generated paper to pass a (workshop) peer review
- **arxiv_id**: 2504.08066
- **urls**: paper https://arxiv.org/abs/2504.08066 ; code https://github.com/SakanaAI/AI-Scientist-v2
- **source availability**: latex (extracted, `papers/latex/2504.08066/main_arxiv.tex`)

## Primary-source facts (from main_arxiv.tex)

- Changes vs v1 (sec:main_aisci_design, line 213): removes human-authored template-code dependency; more general (abstract-first) idea generation; adds an experiment progress manager + agentic tree search; integrates a VLM at experiment and review stages; single-pass write-up + reflection by a reasoning model (o1).
- **Experiment progress manager** runs four explicit stages with stopping criteria: (1) Preliminary Investigation, (2) Hyperparameter Tuning, (3) Research Agenda Execution, (4) Ablation Studies (lines 238-252). It launches replications at each stage end to produce mean/std statistics.
- **Parallelized agentic tree search** (lines 271-298): each node = experiment script + plan + error trace + runtime + metrics + LLM feedback + plotting script + figure paths + VLM feedback + buggy/non-buggy status. Nodes expand in parallel; best-first selection by an LLM evaluator; specialized node types: hyperparameter, ablation, replication, aggregation.
- **VLM reviewer** (sec:vlm_reviewer, line 304): VLM critiques generated figures during experimentation and checks figure-caption alignment during write-up.
- **Headline result** (sec:humaneval, lines 313-341): three fully autonomous manuscripts submitted to the ICLR 2025 ICBINB workshop (43 total submissions, blind review, reviewers told some might be AI-generated). One manuscript scored 6.33/10 (6,6,7), surpassing the workshop acceptance threshold; the other two were not accepted. Withdrawn after review by prior agreement; IRB approval H24-02652.
- **Heavy meta-selection caveat** (lines 359-368): for the accepted paper, humans (a) selected 3 of ~40 AI-generated ideas to run, (b) ran each through tree search multiple times with different seeds, (c) "selected the single best-resulting manuscript for submission based on a careful inspection." The authors frame this as "meta-selection from fully autonomous outputs," not in-content human intervention, and note their aim was "whether... at least one paper that survives peer review, and not what fraction of the time."
- **The authors' own audit of the accepted paper** (lines 378-382): "approximately 57% overlap between training and test sets," and a "reported 100% accuracy of the attention-augmented LSTM model" that their additional tests showed "was primarily due to task simplicity and significantly decreased when task complexity increased." They rate it borderline-accept for a workshop, not conference-grade.
- **Limitations** (sec:limitations, lines 399-413): acceptance was workshop-level (workshop accept rates 60-80% vs 20-30% main track); only 1 of 3 accepted; the system "does not yet consistently reach... top-tier conference publications, nor does it even reach workshop-level consistently"; citation hallucination persists; concern that systems could "evolve solely to game peer review."

## Interpretation (inference, flagged)

- The decisive honesty datapoint of the whole lane: a paper containing a 57% train/test overlap and an unverified 100% accuracy claim passed an AI-augmented blind peer review. Surface plausibility and reviewer scores did not catch a fabricated-strength result. The error was caught only by the human authors inspecting the code, not by the review.
- The "first AI peer-reviewed paper" claim is real but bounded: it is a best-of-many-seeds result with three human selection gates. It is evidence the loop CAN produce a passing artifact, not that it does so reliably or that the artifact is sound.
- Tree search with explicit buggy/non-buggy node status, error traces, and replication-for-statistics is the closest AI-scientist analog to ENPIRE's rollout+verify; the difference is that ENPIRE's verifier is the physical environment, whereas v2's node "success" is an LLM/VLM judgment of metrics and figures.

## Mechanically useful vs research contribution

- **Mechanically useful**: the staged experiment manager and the typed experiment-node record (plan, code, error trace, metrics, status, figures) are a clean schema for an autoresearch loop over the repo's WAM, and they resemble the repo's own typed artifacts. Replication nodes (rerun with seeds, compute mean/std) are a cheap, defensible reliability check that maps onto re-running Minecraft scenarios.
- **Research contribution to NOT copy**: "AI generates a paper that passes peer review" is not this repo's goal, and the VLM/LLM reviewer that grades the output is precisely the self-scoring the thesis forbids. Do not import the reviewer as an evaluation authority.

## WAM layer(s) informed

Method-level, cross-layer. The node schema and staged search inform how to structure an advisory-WAM improvement loop; the audit (57% overlap passing review) is the strongest single warning that the success signal must be an external verifier, not a reviewer.
