## Primary-source facts

- **Setting**: Tool-Integrated Reasoning (TIR). An LLM policy `pi: s_k -> (r_{k+1}, T_{k+1})` interleaves natural-language reasoning `r`, tool-call sets `T`, and observations `o`. Trained with RL (GRPO) instead of SFT, because SFT overfits patterns and limits optimal tool-use strategy.
- **System prompt / output format**: the model must emit `<think>`...`</think>`, then optional `<tool_call>` (JSON objects each with a `name` and a `parameters` dict, possibly multiple), then optional `<response>`. This is structurally the repo's Actor Turn: reasoning + a schema-bound tool call with logical parameters.
- **Reward design (rule-based, the paper's central claim)**: `R_final = R_format + R_correct`.
    - **Format reward** in {0,1}: 1 iff all required fields appear in the correct order.
    - **Correctness reward** in [-3,3], from optimal matching of predicted calls `P` to ground-truth calls `G`, decomposed into:
        - Tool-Name Matching: Jaccard `|N_G ∩ N_P| / |N_G ∪ N_P|`.
        - Parameter-Name Matching: per-call Jaccard over parameter keys.
        - Parameter-Content Matching: per-key exact-match indicator.
    - Normalized to [-3,3] so correctness dominates format (`R_final` in [-3,4]).
- **Key design points**: rewards are RULE-BASED (computed against ground-truth calls), explicitly NOT a single binary; separating structural compliance from fine-grained semantic correctness. GRPO with group-normalized advantage, KL penalty omitted to let the model adapt to the format freely. The authors' own margin notes flag reward-hacking risk (e.g. padding `N_P` to inflate name-match), motivating the Jaccard-with-union denominator.
- **Result (paper-stated)**: ToolRL improves base/instruct models substantially over SFT and prior RL baselines on tool-use benchmarks (generalizes to free-form final-outcome settings despite step-wise training).

## Interpretation (inference, labeled)

- ToolRL is the lane's bridge to the repo's EXACT surface: the unit being optimized is a schema-bound, parameterized tool call, scored by a deterministic, decomposed, rule-based reward against a reference. This is "RLVR for tool selection + parameter filling."
- It is the strongest existing evidence that a tool-calling actor's behavior can be improved against a VERIFIABLE, structured reward rather than a learned judge. The repo's verifier is the structured, rule-based scorer; ToolRL shows what improving against such a scorer looks like.
- The format/correctness decomposition is a direct analog of the repo's two acceptance gates: schema/structure validity (format) and verified-truth of the chosen action's parameters/effects (correctness).

## Mechanically useful vs research contribution

- **Mechanically useful**:
    - **Decomposed rule-based reward as the score**: if the repo ever scores actor-policy candidates (or prompt edits) in an improvement loop, ToolRL is the template for a structured, verifier-derived, gameable-resistant reward: separate format-validity from per-parameter correctness, score against the runtime's ground-truth outcome, never collapse to one opaque scalar.
    - **Tool-name / param-name / param-content as separable credit signals** maps onto the repo's gate stack: tool selected correctly, schema satisfied, parameters verified true. Each is a separable signal for diagnosing and improving an actor.
    - The reward-hacking margin notes (Jaccard union denominator to stop padding) are a concrete reminder to design scores that resist trivial inflation.
- **What to avoid / overclaim**:
    - ToolRL trains MODEL WEIGHTS by RL. This repo's hard rule is the LLM proposes and the runtime owns truth; the repo does not fine-tune the actor in the live social loop. Borrow ToolRL's REWARD STRUCTURE (how to score a tool call against ground truth), not its weight-update mechanism, unless an explicit offline policy-training lane is opened.
    - ToolRL's correctness reward needs ground-truth tool calls `G`. The repo's verifier supplies an analog (verified effect), but for OPEN-ENDED social actions there is no ground-truth `G` to match against; the bound from the rest of the lane applies.
    - Reward engineering for tool use is support, not a research contribution to claim.
