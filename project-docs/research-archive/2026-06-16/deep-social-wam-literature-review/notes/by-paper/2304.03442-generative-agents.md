## Primary-source facts (what the paper states/shows)

- Research question: can an LLM-backed agent architecture produce *believable* individual and emergent social behavior in a Sims-like sandbox (Smallville, 25 agents)?
- Architecture = three mechanisms over a natural-language **memory stream**:
  1. **Memory + Retrieval**: memory stream = list of memory objects (NL description + creation ts + last-access ts). Retrieval score = weighted sum of *recency* (exponential decay, factor 0.995 over sandbox hours), *importance* (LLM rates poignancy 1–10 at creation), *relevance* (cosine similarity of embeddings to a query memory); all α set to 1. Top-ranked memories that fit context are injected.
  2. **Reflection**: when summed importance of recent events exceeds threshold 150 (≈2–3×/day), the agent prompts the LLM for salient questions over the last 100 memories, retrieves evidence, and synthesizes higher-level insights that **cite the memory objects used** (e.g. "Klaus is dedicated to research (because of 1,2,8,15)"). Reflections are stored as memories; reflection trees form.
  3. **Planning + Reacting**: top-down recursive plans (day → hour → 5–15 min chunks), each entry = location + start + duration, stored in memory. On perception, decide whether to react, regenerate the plan from the reaction point; inter-agent dialogue generated by conditioning each utterance on summarized memory about the other agent.
- Input/output: input = NL environment + retrieved NL memories; output = NL action/plan/utterance. Everything is natural-language strings; there is no typed external state and no transition model.
- Evaluation: dependent variable is **believability**. A controlled "interview" probes self-knowledge, memory, plans, reactions, reflection (human ranking); ablations removing observation/planning/reflection each degrade believability. The Valentine's-party emergent result (invitations spread, dates arranged, coordinated arrival) is reported as emergent social behavior judged for believability, not for verified state.

## Interpretation (my inference, labeled)

- Architectural root of LLM social agents (cited by SOTOPIA, Concordia, Lifelong SOTOPIA, belief-behavior, PEBA). Its memory/reflection/planning triad is the "memory continuity" the repo needs, but implemented as *unconstrained natural language* — exactly what the repo forbids as runtime authority.
- The reflection "(because of 1,5,3)" citation is the single most transferable idea: evidence-linked memory. The repo's rule that each memory/relationship update cite a concrete runtime artifact is a world-grounded hardening of it.
- Canonical example of "measures plausible behavior, not verified consequence": believability is the metric; the party "happens" only inside the simulation's own narrative, with no external world able to refute it.

## Mechanically useful vs research contribution

- Mechanically useful: retrieval scoring (recency × importance × relevance) as a cheap memory-ranking recipe for actor memory / PlanBead surfacing; reflection-with-citations → reflections must cite transcript/inventory/container/verifier ids, not other prose; recursive plan decomposition as a CycleGoal granularity reference (advisory only).
- Avoid / overclaim: do NOT let NL memory be executable authority (SPEC §2.2). GenAgents' plans/reflections *are* the agent's truth; the repo's must be Mineflayer-backed. Do not cite emergent-behavior anecdotes as evidence of capability.

## WAM layer(s) informed

- Social WAM + Institutional WAM (memory continuity, relationship formation, information diffusion). Informs the *memory* substrate, not a prediction substrate: GenAgents has no forward consequence model, it generates plausible next text.
