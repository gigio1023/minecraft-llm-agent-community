# Lifelong SOTOPIA: Evaluating Social Intelligence of Language Agents Over Lifelong Social Interactions

- title: LIFELONG SOTOPIA: Evaluating Social Intelligence of Language Agents Over Lifelong Social Interactions
- authors: (Sotopia lab; extends SOTOPIA)
- year: 2025
- arxiv_id: 2506.12666
- urls: paper https://arxiv.org/abs/2506.12666 ; code https://github.com/sotopia-lab/sotopia
- source read: LaTeX (papers/latex/2506.12666/3-framework.tex, 5-results.tex)

## Primary-source facts

- Research question: do LLM agents stay socially competent over **multi-episode** ("lifelong") interaction, where context accumulates across linked episodes?
- Method: **episode chaining** — for a character pair, chain ~40 SOTOPIA episodes sampled by relationship type; characters carry memory of all prior interactions, so context grows linearly. Some chained episodes are interconnected (e.g. once you donated to a charity you may be less willing again). Two memory implementations: (a) entire prior interaction as context; (b) advanced memory module (200–300 word per-episode summary focused on overview + negotiation techniques + new info about the other character).
- Evaluation: two dimensions — Believability [0–10] (used as a *consistency* signal) and Goal Completion [0–10] (used as a *social-intelligence* signal). GPT-4 evaluator.
- Added instrument: **BelievabilityExtended** — because GPT-4 *overestimated* believability at long context, they built an 8-item failure checklist (each failure = −5 to believability): (1) sentence repetition, (2) imitating the other's traits, (3) dialogue off-goal, (4) not leaving after goal resolution, (5) repeating exact private goals verbatim, (6) stalling/idle, (7) not responding to the other / ignoring questions, (8) abrupt/unrelated episode start (confusing current with past episodes). GPT-4's per-checkpoint accuracy was human-validated (50 pos/50 neg each).
- Findings:
  - With full-history memory, **both believability and goal completion decline over the chain** (GPT-4o worst). "Providing additional information has a detrimental effect"; agents confuse their own identity/goals with past or partner's.
  - The advanced summary memory module **fixes the decline on the (randomly chained) easy episodes** (GPT-4o, Gemini-1.5 near-perfect).
  - But on **5 hand-crafted hard scenarios that explicitly require recalling past episodes, all LLMs' goal completion drops sharply while humans maintain.** Believability stays stable (character consistency) but using memory to achieve goals fails.
  - Humans stay consistent and high on both, and qualitatively reuse past interactions (adopt others' negotiation tactics, recall preferences).

## Interpretation (my inference, labeled)

- This is the lane's primary evidence on **memory continuity** — the repo's named central metric. The result is precise and sobering: naive long-context memory *degrades* social behavior; summarization fixes it only when the test doesn't actually demand recall; on tests that *force* cross-episode recall, LLMs fail and humans don't. So memory-continuity must be tested with scenarios that genuinely require a prior fact, not incidental history (the repo's `failed_promise_v1` and obligation-closure scenarios are exactly that kind).
- The GPT-4-evaluator-overestimates-at-long-context finding is a second-order validity warning: the same LLM-judge that SOTOPIA validated becomes unreliable as context grows, which is precisely the regime the repo operates in (long multi-cycle runs). This is a concrete reason the repo should not rely on an LLM judge for long-run social scoring and should anchor to world artifacts.

## Mechanically useful vs research contribution

- Mechanically useful:
  - The **8-item BelievabilityExtended checklist** is a directly adoptable list of *concrete dialogue/behavior failures* to detect from transcripts (especially "not leaving after goal resolution," "stalling," "confusing current with past episode") — these are cheap, rule-checkable signals that complement world-state verification.
  - Episode chaining + the "hand-craft scenarios that require recall" method is the right way to build the repo's memory-continuity benchmark (avoid incidental history; force a needed prior fact).
- Avoid / overclaim:
  - Do not use an LLM judge for long-run social scores without the checklist correction; the paper shows it over-rates at long context.
  - Do not claim memory continuity from easy chained episodes where approaching each independently also scores well.
