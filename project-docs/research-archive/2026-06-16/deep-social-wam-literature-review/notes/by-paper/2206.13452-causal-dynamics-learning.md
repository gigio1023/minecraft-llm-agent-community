## Research question
Most model-based RL (MBRL) learns a DENSE dynamics model: every next-step state variable is predicted from ALL current state variables plus the action. Dense models pick up spurious correlations and generalize poorly to unseen states. Can a learned dynamics model instead keep ONLY the necessary (causal) dependencies, and can a reward-independent (task-independent) state abstraction be derived from that causal structure so it transfers across many tasks?

## Primary-source facts (what the paper states/shows)

Model / formalism:
- Setting: a factored Markov Process M = (S, A, P) where the state is a vector of d_S typed state variables {S^i}. This is a STRUCTURED, variable-factored state, not pixels.
- Causal graphical model (Pearl): a DAG over the variables {s^{1:dS}_t, a_t, s^{1:dS}_{t+1}}; edge i->j means s^i_t is a "direct cause" of s^j_{t+1}. The transition factorizes as the product of per-variable conditionals given each variable's parents PA_i (intro.tex eq for p_X).
- The core trichotomy (the part most relevant to this repo). Given the causal graph, state variables split into three disjoint types (approach.tex Def 1-3):
  1. Controllable (s^C): descendants of the action a_t (variables the agent can change), e.g. doors A and B in the running example.
  2. Action-relevant (s^R): ancestors of controllable variables that are not themselves controllable, e.g. an obstacle that can block a door's motion. They are not controllable but they gate the effect of actions on controllable variables.
  3. Action-irrelevant (s^I): everything else, neither a descendant of the action nor an ancestor of a controllable variable, e.g. a wall clock. They influence nothing the agent acts on.
- The transition then factors into three blocks (approach.tex): P(s_{t+1}|s_t,a_t) = p(s^C_{t+1}|s^C_t, s^R_t, a_t) . p(s^R_{t+1}|s^R_t) . p(s^I_{t+1}|s^R_t, s^I_t). The action only enters the controllable block.
- State abstraction phi: DROP the action-irrelevant variables, phi(s_t) = (s^C_t, s^R_t). The abstract dynamics is still causal and keeps the same generalization benefit. Crucially this abstraction is derived from DYNAMICS ALONE, with no reward function, so it is "task-independent": it retains every variable the agent could ever control, hence applies to any "actively-accomplishable" task (one whose reward depends only on controllable + action-relevant variables).

How the causal edges are learned:
- Theorem 1 (under assumptions A1-A5: causal Markov + faithfulness; full observability + Markov; self-edge s^i_t->s^i_{t+1} exists; no simultaneous/backward edges; per-variable independent transitions): a single Conditional Independence Test (CIT) decides each edge. If s^i_t is NOT independent of s^j_{t+1} given {a_t, all other current state vars}, then the edge s^i_t -> s^j_{t+1} exists. Same test with a_t in place of s^i_t decides the action edge.
- The CIT is operationalized by Conditional Mutual Information (CMI): CMI^{ij} = E[ log( p(s^j_{t+1} | a_t, s_t) / p(s^j_{t+1} | a_t, s_t minus s^i_t) ) ]. If CMI^{ij} >= epsilon (a threshold), variable i is necessary to predict j, so the edge exists.
- Architecture trick (approach.tex, Fig "architecture"): naively this needs (d_S)^2 predictors; instead one model per output variable with a MASKING scheme. Each input (action and every state variable) is mapped to a d_f feature; a binary mask sets selected features to -inf; an element-wise max pools the features; a head predicts s^j_{t+1}. Masking nothing gives p(.|a_t,s_t); masking feature i gives p(.|a_t, s_t minus s^i_t); masking all non-parents gives the final causal predictor p(.|PA). This reduces the cost to d_S models. Training maximizes the sum of the three log-likelihoods (full, leave-one-out, parents-only).
- Active data collection: an RL agent (PPO over skills) collects transitions with an intrinsic reward equal to the prediction-gap between the dense and the causal predictor (tanh-scaled). It seeks transitions where the causal graph is still wrong, "PredDiff" exploration.
- Planning/downstream: dynamics is frozen; only a reward predictor over the abstract state is learned; the Cross-Entropy Method (CEM) plans over the learned causal dynamics + reward.

Evaluation protocol + metrics:
- Two environments: (1) a Chemical environment (10 colored objects, action recolors one object, color changes propagate along a ground-truth causal graph; tested with full / chain / collider graph structures), and (2) a robosuite table-top Manipulation environment (movable cube, unmovable cube, randomly-moving cube, plus 3 random distractor markers to inject spurious correlations).
- Metrics: causal-graph edge accuracy vs ground truth; k-step (1-5) prediction accuracy on in-distribution (ID) and out-of-distribution (OOD) states (OOD created by setting irrelevant variables, e.g. marker positions, to unseen values); downstream task learning curves and OOD episode reward on Match/Reach/Lift/Stack.
- Headline numbers:
  - Causal graph accuracy: CDL 100.0/100.0/99.1% on Chemical collider/chain/full and 90.2% on Manipulation, beating the regularization baseline "Reg" (99.4/99.7/97.7/84.4%). (experiments.tex Table "causal_graph_accuracy".)
  - Generalization: on ID states all methods are similar; on OOD states dense baselines (Modular/GNN/Monolithic) "drop 60 ~ 90%" in some complex environments while CDL retains near-ID accuracy (intro.tex; experiments.tex prediction results).
  - Active collection: PredDiff exploration 90.2% graph accuracy vs Uniform 89.1% vs Curiosity 88.9%; the paper argues even ~1% matters because it is ~5-6 wrong edges in a 23x24 dependency graph.
  - Downstream: CDL learns Lift/Stack most sample-efficiently and has the best OOD episode reward.

Limitations (theirs):
- Conclusions + assumptions: "this paper focuses on low-dimensional state spaces; an important future direction is to extend CDL to high-dimensional space, like images." Assumption A5 (per-variable independent transitions) "does not necessarily hold for rich observation spaces (e.g., images)"; A2/A5 may require first learning a disentangled low-dim representation.
- Task-independence is bounded: the abstraction is valid only for "actively-accomplishable" tasks (reward depends only on controllable + action-relevant variables). A task that rewards an action-irrelevant variable (e.g. "+1 if door A opens at 1pm") is out of scope. The authors argue such rewards are "relatively uncommon."
- Needs a CMI threshold epsilon per environment (chosen by hand); needs the faithfulness assumption to hold.

## Interpretation (my reading, labeled inference)
- CDL is the single closest formal template in this lane for a STRUCTURED-STATE Minecraft WAM. Its state is exactly the project's shape: a vector of typed variables, not pixels. Its three-way variable split is almost a one-to-one analogue of what an advisory WAM needs to reason about:
  - controllable = variables the actor's action directly changes (own inventory count, own position, a block I place);
  - action-relevant = variables that gate whether my action succeeds but that I do not own (durability of the tool I am about to use, whether a container is already claimed, whether a station is reachable);
  - action-irrelevant = world variables irrelevant to the candidate action (time-of-day for a pure-crafting action, a far-away actor's inventory).
- The central RESEARCH claim CDL backs for this repo is the generalization one: a model that encodes "what actually causes what" stays accurate on states it never trained on, whereas a dense model that learned spurious correlations collapses (60-90% drop). For a social-material WAM this is the difference between "I predict lending the pickaxe creates a return obligation" generalizing to a new pair of actors vs. only working for actor pairs seen in training.
- CDL's CMI masking architecture is mechanically reusable as a way to LEARN which typed state variables matter for predicting a given delta, instead of hand-authoring the dependency. But the repo's runtime already owns ground-truth transitions cheaply (the verifier emits the exact (o,a,o') triplet as typed JSON), so the project could SKIP CDL's expensive active-exploration step and instead recover the causal graph offline from logged verifier transitions.

## Mechanically useful vs research contribution (for this repo)
- Mechanically useful (engineering to borrow):
  - The controllable / action-relevant / action-irrelevant partition as a typing discipline for the WAM's predicted-delta schema. It tells the advisory model which variables an action can change, which merely gate it, and which to ignore, mirroring the repo's `current_state` typed-fact split.
  - The CMI-masking idea as an offline analysis over logged verifier triplets to discover (not hand-author) which state fields are causal parents of which delta.
  - The factored transition as a structure for per-delta prediction heads (predict inventory delta, possession delta, obligation delta separately, each from only its causal parents).
- Research contribution (would be a novel claim if done here, not a freebie):
  - Showing that a causal (sparse, parent-only) structured-state WAM generalizes to unseen actor configurations / item sets / social pairings better than a dense predictor, in Minecraft. That is CDL's claim transported to a SOCIAL state space, which CDL never tested.
  - Extending the trichotomy to the social/material layers (a "controllable social variable" like a relationship the actor can repair vs an "action-relevant" one like another actor's standing obligation). CDL only studied physical robot/chemical variables.
- What to avoid / overclaim guard:
  - Do NOT claim CDL "proves" a Minecraft social WAM generalizes; CDL's guarantees rest on faithfulness + full observability + per-variable independence, all of which are shakier for social state (hidden intent, partner-dependent effects, correlated social variables). Cite CDL as a template and a hypothesis source, not as evidence about social dynamics.
  - CDL is reward-free abstraction; it does not select actions. It is a World Model (p(o'|o,a)) with a planner bolted on, NOT a WAM in the survey's joint sense. Keep that distinction explicit: CDL informs the PREDICTION half, and only advisorily.

## WAM layer(s) informed
- Physical WAM (primary): the controllable/action-relevant split is exactly physical-consequence typing (this block change depends on the action + tool durability, not on the clock).
- Material/economic and Social WAM (by analogy, as a research extension): the same factorization is the natural typing for possession/obligation deltas, but transporting it there is unproven (interpretation).
- Ties to evaluation: CDL's ID-vs-OOD prediction-accuracy protocol is a concrete, structured-state way to report the advisory WAM's prediction quality as its own axis, complementing the survey's coupled-metric gap (see `wam-training-evaluation-and-open-problems.md`).
