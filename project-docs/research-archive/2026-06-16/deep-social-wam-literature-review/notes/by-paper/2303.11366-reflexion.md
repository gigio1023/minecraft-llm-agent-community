## Primary-source facts (what the paper states/shows)

- Reinforces an LLM agent NOT by updating weights but by **verbal reinforcement**: after a trial,
  the agent writes a natural-language self-reflection into an episodic memory buffer (bounded to
  1-3 entries), which conditions the next trial. Three modules: Actor (generates actions),
  Evaluator (scores the trajectory), Self-Reflection (turns the score into a verbal lesson).
- **The Evaluator (the reward source) is the key variable, and it is pluggable**:
    - reasoning (HotPotQA): **exact-match grading against the environment/ground truth** (external,
      verifiable);
    - decision-making (AlfWorld): a hand-written **heuristic** (e.g. repeat-action / step-count
      thresholds) plus optionally an LLM classifier;
    - programming (HumanEval, MBPP, LeetcodeHardGym): **self-generated unit tests** (the model
      writes its own tests, then runs them).
- **Results**: AlfWorld +22% over ReAct (130/134 tasks), HotPotQA +20%, HumanEval pass@1 91% vs
  GPT-4 80%. Refinement-only (no reflection) and "blind trial-and-error" debugging do not improve
  on hard tasks; the verbal-reflection step is what helps.
- **The self-evaluation failure mode, measured**: on programming, self-generated unit tests can be
  flaky. The paper reports false-positive rates (tests pass but the solution is wrong): 1.4% on
  HumanEval Python vs 16.3% on MBPP Python, and the high MBPP false-positive rate is why Reflexion
  UNDERPERFORMS the baseline on MBPP. A false-positive label makes the agent prematurely report a
  wrong solution as done.
- Stated limitations: Reflexion "relies on the power of the LLM's self-evaluation capabilities (or
  heuristics)" and has "no formal guarantee for success"; it is a policy optimization that can hit
  non-optimal local minima.

## Interpretation (flagged as inference)

- Reflexion is the hinge case for the bright line because its reward source spans the line within
  one paper. Where the Evaluator is external (exact-match, compiler, unit tests that actually
  run), the loop is verifier-grounded and admissible. Where the Evaluator is the LLM judging
  itself, it inherits the self-evaluation failure modes. The MBPP false-positive result is a small,
  concrete instance of self-evaluation marking wrong work as done: the same shape as the repo's
  progress laundering, at the granularity of one task.
- For the repo, Reflexion is the closest published analog of the actor's CycleJudgment loop:
  observe outcome, write a verbal lesson, carry it in memory, act better next cycle. The
  transferable, safe version replaces the self-evaluated outcome label with the runtime verifier's
  label. Reflexion's own evidence (it works best with exact-match / executed tests, worst with
  flaky self-tests) supports keeping the verifier, not the actor, as the scorer.
- Reflexion is also the bridge between memory (wave-3) and learning signal (this lane): a judgment
  about a past attempt, stored as memory, changes future behavior. The wave-3 file noted this;
  this note adds that the judgment's RELIABILITY (external vs self) is what decides whether the
  loop improves or degrades.

## Mechanically useful vs research contribution

- **Mechanically useful**: the Actor / Evaluator / Self-Reflection decomposition is a clean
  template for an advisory, no-weight-update improvement loop on this repo: the runtime verifier is
  the Evaluator; a reflection step writes a verifier-grounded lesson into actor-workspace memory;
  the actor reads it next cycle. The false-positive measurement is a reusable guardrail metric: if
  the scorer is ever a self-test, measure P(scorer says pass | actually fail) and treat a high
  value as disqualifying.
- **Research contribution it is NOT**: Reflexion's contribution is verbal RL with episodic memory.
  The repo borrows the loop shape and the explicit caution about self-evaluation; the verbal-RL
  mechanism is support, not a research claim.
