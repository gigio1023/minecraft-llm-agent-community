지금은 구현 속도보다 방향 고정이 더 중요합니다. 이 페이지는 SPEC.md 구현 후 기대 상태, 프로젝트의 진짜 목표, 그리고 가치 판단과 구현 우선순위를 함께 묻습니다.

이번 인터뷰의 목적

  • 이 프로젝트가 연구용인지, 프로브용인지, 사회 시뮬레이션용인지 구분
  • SPEC.md 구현 완료 후 무엇을 "성공"으로 볼지 정의
  • 지금 당장 버릴 욕심과 남길 욕심을 분리

구현 후 기대할 수 있는 것

  • 작고 이해 가능한 bounded runtime
  • 단일 봇 기준 관찰, 실행, 검증, transcript 경계 정리
  • 숨은 LLM 의존성 없는 deterministic baseline
  • 재연결과 persistence를 붙일 준비가 된 구조

구현 후 아직 기대하면 안 되는 것

  • 사람 같은 NPC 사회 완성
  • 멀티봇 협업의 안정적 완성도
  • 장기 실행 전체 복구 자동화
  • 풍부한 social drama

프로젝트의 잠정 핵심 목표

  • headless Minecraft에서 작동하는 작은 tool-loop probe
  • eval 없는 runtime-owned boundary
  • 실험과 리팩터링이 쉬운 연구용 기반
