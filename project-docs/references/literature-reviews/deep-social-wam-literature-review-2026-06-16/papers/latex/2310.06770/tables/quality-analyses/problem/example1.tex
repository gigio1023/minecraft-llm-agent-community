When displaying a DatasetGroupBy in an interactive Python session, the first line of output contains a trailing whitespace. The first example in the documentation demonstrate this:

\lstset{linewidth=\linewidth}
\lstset{basicstyle=\small\ttfamily}

\begin{lstlisting}[language=Python]
>>> import xarray as xr, numpy as np
>>> ds = xr.Dataset(
...     {"foo": (("x", "y"), np.random.rand(4, 3))},
...     coords={"x": [10, 20, 30, 40], "letters": ("x", list("abba"))},
... )
>>> ds.groupby("letters")
DatasetGroupBy, grouped over 'letters' 
2 groups with labels 'a', 'b'.
\end{lstlisting}

There is a trailing whitespace in the first line of output which is "DatasetGroupBy, grouped over `letters' ". This can be seen more clearly by converting the object to a string (note the whitespace before \pythoninline{\n}):

\begin{lstlisting}[language=Python]
>>> str(ds.groupby("letters"))
"DatasetGroupBy, grouped over 'letters' \n2 groups with labels 'a', 'b'."
\end{lstlisting}
