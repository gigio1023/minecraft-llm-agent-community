\section{Prompts}
\label{sec:prompts}

We include all the prompts used in the experiments in Figures \ref{fig:prompt:acronym:init}-\ref{fig:prompt:sentiment:refine}:
\begin{itemize}
\item \textbf{Acronym Generation:} Figures \ref{fig:prompt:acronym:init}-\ref{fig:prompt:acronym:refine}
\item \textbf{Code Optimization:} Figures \ref{fig:prompt:pie:init}-\ref{fig:prompt:pie:refine}
\item \textbf{Code Readability Improvement:} Figures \ref{fig:prompt:readability:fb}-\ref{fig:prompt:readability:refine}
\item \textbf{Constrained Generation:} Figures \ref{fig:prompt:commongen:init}-\ref{fig:prompt:commongen:refine}
\item \textbf{Dialogue Response Generation:} Figures \ref{fig:prompt:response:init}-\ref{fig:prompt:response:refine}
\item \textbf{Math Reasoning:} Figures \ref{fig:prompt:gsm:init}-\ref{fig:prompt:gsm:refine}
\item \textbf{Sentiment Reversal:} Figures \ref{fig:prompt:sentiment:init}-\ref{fig:prompt:sentiment:refine}


\end{itemize}

Recall that the Base LLM requires a generation prompt $p_{gen}$ with input-output pairs $\langle x_i, y_i \rangle$, the feedback module requires a feedback prompt $p_{fb}$ with input-output-feedback triples $\langle x_i, y_i, fb_i \rangle$, and the refinement module (\itermod) requires a refinement prompt $p_{refine}$ with input-output-feedback-refined quadruples $\langle x_i, y_i, fb_i, y_{i+1} \rangle$.


\begin{itemize}
    \item \textbf{\sentxfer} We create positive and negative variants of a single review from the training set and manually write a description for converting the negative variant to positive and vice versa. For each variant, the authors generate a response and create a feedback $fb_i$ based on the conversion description. 
\item 
\textbf{\dialgen} 
We sample six examples as $\langle x_i, y_i \rangle$ for the few-shot prompt for the Base LLM. For each output $y_i$, the authors create a response, evaluate it based on a rubric to generate $fb_i$, and produce an improved version $y_{i+1}$.

\item \textbf{\acrogen} We provide the Base LLM with a total of 15 (title, acronym) examples. 
Then, for one title ($x_i$) we generate an acronym ($y_i$) using \chatgpt.
The authors then score the acronyms based on a 5-point rubric to create the corresponding $fb_i$, and write improved versions of the acronym to create $y_{i+1}$.
3 such examples are used for \itermod and \fbmod.

\item \textbf{\codeopt} We use the slow ($x_i$) and fast ($y_i$) versions of programs released by \citet{pie} for Base LLM.
We use their provided explanations \citep{pie} for \fbmod and \itermod.

\item \textbf{\gsm} The prompts for the Base LLM are sourced from PaL~\citep{gao2022pal} as $\langle x_i, y_i \rangle$. 
We select two examples from the training set on which \codex fails when prompted with PaL-styled prompts, and manually write the correct solution ($y_{i+1}$) and reasoning ($fb_i$) for \itermod and \fbmod.

\item \textbf{\commongenhard} We provide ten examples to the Base LLM as $\langle x_i, y_i \rangle$.
We sample six examples from the training set of \commongen and create variants with missing concepts or incoherent outputs. The missing concepts and the reason for incoherence form $fb$.

\end{itemize}
