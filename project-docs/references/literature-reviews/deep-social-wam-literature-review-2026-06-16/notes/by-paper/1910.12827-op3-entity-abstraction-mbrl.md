# OP3: Entity Abstraction in Visual Model-Based Reinforcement Learning

- title: Entity Abstraction in Visual Model-Based Reinforcement Learning (OP3 = Object-centric
  Perception, Prediction, and Planning)
- authors: Rishi Veerapaneni, John D. Co-Reyes, Michael Chang, Michael Janner, Chelsea Finn,
  Jiajun Wu, Joshua B. Tenenbaum, Sergey Levine
- year: 2019
- venue: CoRL 2019
- arxiv_id: 1910.12827
- urls: paper https://arxiv.org/abs/1910.12827 , code https://github.com/jcoreyes/OP3
- source_availability: latex (papers/latex/1910.12827/, content in src/)

## Primary-source facts (what the paper states)

- Research question: does modeling a scene as ENTITIES + their LOCAL interactions (rather than
  globally) help generalize to physical tasks in a combinatorial space not seen in training?
- OP3 is described as the first fully PROBABILISTIC, entity-centric, dynamic latent-variable
  framework for model-based RL that acquires entity representations from raw vision WITHOUT
  supervision, and uses them to predict and plan.
- ENTITY ABSTRACTION: symmetric processing of each entity representation by the same
  locally-scoped function. This is what lets OP3 scale to different numbers/configurations of
  objects than training.
- Variable binding (grounding entity slots to actual objects) is framed as an INFERENCE problem;
  an interactive inference algorithm uses temporal continuity and interactive feedback to bind
  object properties to entity variables.
- Action space / domain: block-stacking manipulation; the model is used for planning.
- Results: on block-stacking, OP3 generalizes to novel block configurations and MORE objects than
  seen in training, outperforming an oracle model that assumes object supervision, and achieving
  2-3x better accuracy than a state-of-the-art video-prediction model lacking entity abstraction.
- Discussion (their own framing): the hard part of purely symbolic abstract representations is
  updating them with raw data; OP3 frames symbolic entities as random variables in a latent
  dynamics model and refines their posterior over time, suggesting a bridge between abstract
  symbolic variables and the noisy continuous physical world.

## Interpretation (my reading, labeled inference)

- OP3 is the strongest single-paper argument for the project's core bet at the perception/state
  level: entity-factored, locally-scoped processing generalizes combinatorially and beats both a
  supervised oracle and a global video model. "More objects than training" is the property a
  social world needs (variable population of actors/items).
- The "symbolic entity as a refined random variable" idea is the conceptual hinge for a
  neuro-symbolic social predictor: typed entities (actors, items, claims) can be both symbolic
  (for checking/auditing) and continuously updated from evidence.

## Mechanically useful vs research contribution

- Mechanically useful: the entity-abstraction principle (same locally-scoped function applied
  symmetrically per entity) as the inductive bias for a predictor over a VARIABLE set of
  Minecraft actors/items; entity-count generalization as the feasibility argument for open worlds.
- Avoid / overclaim: OP3 learns entities from pixels and plans with the model; the project's
  entities are already typed by the runtime and the predictor must stay advisory (no planning
  authority). OP3 is non-social (block stacking); borrow the abstraction, not a social claim.

## WAM layer(s) informed

- Physical primarily. Entity-count generalization is the cross-cutting property that matters for
  Material/Social layers (variable numbers of actors, items, claims).
