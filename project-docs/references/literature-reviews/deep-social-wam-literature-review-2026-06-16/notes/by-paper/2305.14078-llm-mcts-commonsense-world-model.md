## Primary-source facts (LaTeX-verified)

- **The two roles of an LLM, named**: L-Policy = query the LLM directly for the next action and execute it (the common pattern). L-Model = use the LLM as a commonsense WORLD MODEL. LLM-MCTS uses the LLM for BOTH: the LLM builds a commonsense prior belief over states (the world model), MCTS samples from that belief to estimate action values, and the LLM-as-policy is used ONLY as a search heuristic (PUCT prior), not committed to directly. Quote: "While L-Policy commits to the actions chosen by the LLM for execution, LLM-MCTS uses these choices only as a search heuristic."
- **World model = commonsense prior belief over states**: for object-search / rearrangement tasks the LLM predicts likely object locations (e.g. a fridge is in the kitchen), forming an initial belief; this belief is updated by real actions and observations; MCTS samples states from it in simulation.
- **The MDL decision rule (the lane-distinct contribution)**: a minimum-description-length argument for WHEN to use the LLM as a world model rather than as a policy. The L-Model has a smaller description length than the L-Policy (a transition/affordance model is a more compact hypothesis than a full task policy), so by MDL it generalizes better. Stated principle: "if the description length of the world model is substantially smaller than that of the policy, using LLM as a world model for model-based planning is likely better than using LLM solely as a policy." This is a principled answer to "should the LLM be the dynamics model here?"

## Numbers as stated

- Tasks: multiplication, multi-hop travel planning, object rearrangement (VirtualHome-style households). LLM-MCTS "outperforms both MCTS alone and policies induced by LLMs (GPT2 and GPT3.5) by a wide margin, for complex, novel tasks" (abstract). Exact per-task tables not transcribed here; the load-bearing claim is the MDL principle plus the wide-margin gain over both pure-MCTS and pure-L-Policy.

## Reproducibility status

Self-reported (NeurIPS 2023). The MDL claim is a theoretical-analysis argument plus empirical support, not an environment-verified delta in the repo's sense. Treat the MDL rule as a well-motivated heuristic, not a proof transferable to social state.

## Relevance to this repo

- **Mechanical import (high, conceptual)**: LLM-MCTS gives the repo a DECISION RULE for the cheapest-first predictor question. The repo's advisory WAM is exactly an L-Model (a transition/affordance predictor), and the MDL argument says using the LLM as such a compact dynamics model should generalize better than using it as a full policy. It also gives the safe usage pattern the repo needs: the LLM-policy is only a search heuristic, the world model provides the belief, and (in the repo's case) the verifier owns the ground-truth update, paralleling LLM-MCTS updating belief from real observations.
- **Why the MDL framing matters for layers**: at Physical/Material the transition rules (a stone pickaxe is needed for iron ore; lending moves possession and creates an obligation) are short, compact hypotheses, so MDL favors the world-model use there. At the Social layer the "dynamics" (how a partner reacts) are not obviously more compact than a policy, so MDL does not clearly favor the world-model framing, which is an honest boundary on transporting this paper upward.
- **The grounding parallel**: LLM-MCTS updates its commonsense belief with each real action and observation. The repo's verifier is the real-observation channel; the difference is that LLM-MCTS's belief is over hidden object locations (passive), whereas the repo's verifier gives an exact typed delta (active ground truth), which is stronger.
- **Research contribution vs support**: the MDL rule is the borrowable idea. The contribution is testing whether, for typed social-material transitions, the world-model (L-Model) description length really is shorter than a policy, which would justify the advisory-WAM framing at those layers.

## WAM layer(s) informed

Physical and Material (compact transition rules favor the world-model use under MDL). Social/Institutional is where the MDL argument weakens (partner dynamics are not clearly more compact than a policy); record as a boundary, not a result.
