# DIAYN: Diversity is All You Need - Learning Skills without a Reward Function (Eysenbach, Gupta, Ibarz, Levine, 2018)

The canonical unsupervised skill-discovery method: learn a set of distinguishable skills with no task
reward by maximizing the mutual information between a skill label and the states it visits. Source:
LaTeX deep-read at `papers/latex/1802.06070/iclr_diayn.tex`.

## Plain-language: what it introduced and why it mattered

DIAYN answers "where do the low-level skills come from?" without any task reward. It samples a discrete
*skill* label z, runs a skill-conditioned policy, and trains everything so that **different skills visit
different states** (a discriminator must be able to guess which skill produced a state) while each skill
stays as random as possible otherwise (a maximum-entropy policy). The result is the *unsupervised
emergence* of diverse behaviors (walking, jumping, different gaits) purely from the drive to be
distinguishable. These skills can then be used as the low-level primitives of a hierarchy, or as a
pre-training initialization for a downstream task.

Why it mattered: it is the cleanest and most-cited instance of **mutual-information skill discovery**,
the sub-thread of HRL that tries to *learn* a skill repertoire bottom-up rather than hand-design it. It
showed an information objective alone yields useful, composable skills, and that those skills help on
sparse-reward, long-horizon tasks when stacked under a higher-level chooser.

## Definitions (first use)

- **Skill (here)**: a latent-conditioned policy pi(a | s, z) for a fixed skill code z; varying z gives
  different behaviors.
- **Mutual information (MI) objective**: maximize the information the visited states carry about the
  skill z (so skills are distinguishable from their state-visitation), implemented by training a
  discriminator q(z | s) to predict z from states and rewarding the policy when it is predictable.
- **Maximum-entropy policy**: a policy regularized to stay as random as possible subject to its
  objective, here so each skill explores broadly rather than collapsing to one trajectory.
- **Unsupervised skill discovery**: learning a set of behaviors with no task/extrinsic reward, judged
  only by an intrinsic criterion (diversity/distinguishability).

## Primary-source facts

- DIAYN "learns skills by maximizing an information theoretic objective using a maximum entropy policy"
  and yields "the unsupervised emergence of diverse skills, such as walking and jumping" with no reward
  (`iclr_diayn.tex` abstract).
- On RL benchmarks it "is able to learn a skill that solves the benchmark task despite never receiving
  the true task reward."
- Pretrained skills "can provide a good parameter initialization for downstream tasks, and can be
  composed hierarchically to solve complex, sparse reward tasks" (`iclr_diayn.tex`).
- Motivation explicitly lists HRL: "For long horizon tasks, skills discovered without reward can serve
  as primitives for hierarchical RL, effectively shortening the episode length" (`iclr_diayn.tex` intro).

## Interpretation (labeled inference)

- DIAYN is the headliner of the *skill-discovery* sub-thread that Director's related-work groups with
  VIC (Variational Intrinsic Control, 1611.07507) and DADS (Dynamics-Aware Discovery of Skills,
  1907.01657). The honest summary across that sub-thread: MI skill discovery reliably produces *diverse*
  skills, but "diverse" is not the same as "useful for a specific task," and the discovered skills are
  notoriously sensitive to the state representation and the MI estimator. This is the second pillar
  (with option collapse) of "skill/subgoal discovery is unsolved."
- DADS is the most relevant variant to a *world-model* lane because it explicitly biases discovery
  toward **skills whose outcomes are easy to predict** (it co-learns skill dynamics), tightening the
  link between skill discovery and model-based planning. For this repo, that is the more interesting
  idea than DIAYN's pure diversity: a useful subgoal is one whose consequence is predictable and
  verifiable.

## Mechanically useful vs research contribution

- **Mechanically useful**: conceptually, the criterion "a good skill is one whose effect is
  distinguishable and (per DADS) predictable" is a useful selection principle for which *typed subgoals*
  an advisory WAM should bother modeling, prefer subgoals whose post-state is cleanly verifiable. This
  is a design heuristic, not code to import.
- **Research contribution (NOT to adopt)**: unsupervised RL skill discovery by MI maximization is a
  research method that requires training policies and discriminators by RL; the repo's skills are
  LLM-proposed and runtime-verified, so DIAYN is cited as the skill-discovery reference and as evidence
  of the discovery problem's difficulty, not adopted as a mechanism.

## Relevance to this repo (one line)

The canonical "learn a skill repertoire with no reward" method; it supplies a selection principle (model
subgoals whose effects are distinguishable and predictable, per DADS) and is the second piece of evidence
that automatic skill discovery is unsolved.

## Which WAM layer(s) it informs
