# LAPA: Latent Action Pretraining from Videos

- title: Latent Action Pretraining From Videos
- authors: Ye, Jang, Jeong, Kwon, Kim, Yu, et al. (KAIST / Microsoft Research / others)
- year: 2024 (ICLR 2025)
- arxiv_id: 2410.11758
- urls: paper https://arxiv.org/abs/2410.11758 ; project https://latentactionpretraining.github.io
- source_availability: latex (`papers/latex/2410.11758/iclr2025_conference.tex`)
- reproducibility_status: reproducible (code + checkpoints promised/open-sourced)
- WAM layer(s): Physical (transition dynamics); method informs the "no labels at all" extreme.

## Primary-source facts

- Research question: can a VLA be pretrained from internet video **without any
  ground-truth action labels** (not even a small labeled seed)?
- Method: three stages. (1) **Latent Action Quantization**, a VQ-VAE objective
  learns discrete latent actions *between raw image frames*, fully unsupervised
  (analogous to BPE tokenization of "atomic actions", no predefined action priors
  like end-effector or joint positions). (2) **Latent Pretraining**, a VLM is
  trained to predict the latent actions from observation + task description
  (behavioral cloning in latent space). (3) **Fine-tune** on a small action-labeled
  robot set to map latent actions to real end-effector delta actions.
- Result: outperforms baselines that pretrain without ground-truth labels, and
  beats OpenVLA (trained WITH ground-truth actions) by +6.22% while achieving
  **>30x greater pretraining efficiency**. Positive transfer even when pretrained
  only on human-manipulation video.
- Notable: the VQ-VAE **decoder can serve as a world model**, predict future
  frames conditioned on current observation + predicted latent action, i.e. a
  closed-loop neural simulator. This is a cascaded-WAM `p(a|o',o,l) times p(o'|o,l)`
  construction. (Explicit in their contributions list.)
- Related latent-action lineage cited: GENIE (Bruce 2024) maps user inputs to
  latent space; Edwards 2018 / Schmidt 2024 use latent actions for video games.

## Interpretation (inference)

- LAPA invents latent actions **because pixel video lacks an action vocabulary**.
  The VQ-VAE is doing, in an unsupervised pixel space, what this repo's schema
  already does explicitly: define a bounded, discrete action vocabulary (Action
  Card function tools + `author_mineflayer_action`; primitives like `move_to`,
  `mine_block`, `craft_item`, `deposit_shared`).
- Therefore the "hard, clever" part of LAPA (learning an action codebook from
  appearance) is a non-problem here. This repo's challenge is the inverse and
  easier: it has typed actions and typed deltas; it needs to *predict* the delta,
  not *discover* the action space.

## Mechanically useful vs research contribution

- Mechanically useful: the cascaded framing (predict `o'` then predict `a|o',o,l`)
  maps cleanly onto a structured WAM that predicts `predicted_delta` then scores a
  candidate action against it. Also: latent-action pretraining is the right
  *fallback* if this repo ever wants to exploit raw pixel runs without verifier
  labels, but it should not be the first move.
- Would be overclaim: presenting VQ-VAE latent actions as needed here. With a typed
  action space and verifier deltas, introducing latent actions adds entanglement
  risk (the very "visual entanglement / shortcut learning" failure LAPA, CLAP,
  ConLA, and VLA-JEPA all spend effort fighting) for no benefit.
