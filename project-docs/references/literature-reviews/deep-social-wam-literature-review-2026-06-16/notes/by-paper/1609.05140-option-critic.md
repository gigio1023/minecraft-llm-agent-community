# The Option-Critic Architecture (Bacon, Harb, Precup, 2016)

The first end-to-end method to learn options (temporally-extended skills with their own policies and
termination conditions) jointly with the policy over them, from reward alone. Source: LaTeX deep-read
at `papers/latex/1609.05140/bacon.tex`.

## Plain-language: what it introduced and why it mattered

The **options framework** (Sutton, Precup, Singh 1999) is the formal language for "skills": an option
is a triple of an initiation set (where it can start), an internal policy (what to do while it runs),
and a termination condition (when to stop). Before Option-Critic, people mostly hand-built options or
discovered them with heuristics (find bottleneck states, then learn a policy to reach them). Option-
Critic showed you can learn the *whole option*, its internal policy and its termination probability,
end-to-end by gradient descent, at the same time as the policy that chooses among options, using only
the task reward and no subgoals.

Why it mattered: it derived **policy gradient theorems for options** (the "intra-option policy gradient"
and the "termination gradient"), turning option discovery from a heuristic, two-stage pipeline into a
single differentiable optimization. It is the canonical "learn options end-to-end" reference and the
formal anchor for what a "skill" is in HRL.

## Definitions (first use)

- **Option**: a temporally-extended action = (initiation set I, internal policy pi_omega, termination
  function beta). Once chosen, the option's policy runs until it terminates per beta. A one-step option
  is just a primitive action, so options strictly generalize actions.
- **Semi-MDP (SMDP)**: the decision process you get when actions can take variable amounts of time
  (because options run for several steps); the options framework shows a Markov decision process (MDP)
  plus a fixed set of options is an SMDP, which is why options are "between MDPs and semi-MDPs."
- **Policy over options**: the higher-level policy pi_Omega that picks which option to run next.
- **Intra-option policy gradient**: the gradient of the expected return with respect to an option's
  internal policy parameters.
- **Termination gradient**: the gradient with respect to an option's termination-probability
  parameters; it relates to the "advantage" of continuing the current option versus switching.

## Primary-source facts

- Derives policy gradient theorems for options and proposes the option-critic architecture that learns
  "both the internal policies and the termination conditions of options, in tandem with the policy over
  options, and without the need to provide any additional rewards or subgoals" (`bacon.tex` abstract).
- "To our knowledge, this is the first end-to-end approach for learning options" (`bacon.tex`).
- Works in both discrete (four-rooms) and continuous settings, and scales to ATARI with a prototypical
  intra-option Q-learning implementation.
- Reported behavior: when the environment changes, the option-critic agent "recovers faster" than a
  flat baseline, illustrating the transfer benefit options are supposed to provide.

## Interpretation (labeled inference)

- Option-Critic is the formal definition this lane's theme file leans on for the word "skill": a skill
  = an option = (initiation, policy, termination). That triple is the precise vocabulary the project's
  informal "action skill" and "CycleGoal" reuse, and naming it pins the project's hierarchy to an
  academic concept rather than a local invention.
- The well-known failure mode (corroborated by `papers/latex/2406.00483/main.tex`, the limits-of-
  hierarchical-world-models paper) is **option collapse / degeneracy**: without extra regularization,
  learned options tend to degenerate so that either one option is used for everything (no temporal
  abstraction) or options never terminate (the termination function collapses). This is the single
  clearest evidence that "skill/subgoal discovery is unsolved," which the theme file must state plainly.

## Mechanically useful vs research contribution

- **Mechanically useful**: the *vocabulary* (initiation set, internal policy, termination condition,
  policy over options, SMDP) is the precise language the repo should use to describe its action skills
  and subgoals, so the hierarchy reads as an instance of a known framework. The initiation set maps to

  completion check.
- **Research contribution (NOT to adopt)**: learning options by policy gradient from reward is an RL
  research result; the repo does not learn options by RL (the LLM proposes; the runtime verifies). It
  is cited as the definition of a skill, not as a training procedure.
