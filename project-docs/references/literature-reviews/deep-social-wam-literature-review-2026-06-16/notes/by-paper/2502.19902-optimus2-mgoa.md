# Optimus-2 / MGOA dataset

- title: Optimus-2: Multimodal Minecraft Agent with Goal-Observation-Action Conditioned Policy
- year: 2025 (CVPR 2025)
- arxiv_id: 2502.19902
- urls: paper https://arxiv.org/abs/2502.19902 ; code https://github.com/lizaijing/Optimus-2 ;
  data https://huggingface.co/datasets/iLearn-Lab/Optimus-2-MGOA
- source_availability: latex (`papers/latex/2502.19902/`, 41 tex) + HF dataset card
- reproducibility_status: reproducible (code + MGOA dataset public, MIT)
- WAM layer(s): Physical (goal-conditioned action policy in Minecraft).

## Primary-source facts

- MGOA = "Multimodal Goal-Observation-Action" dataset (`Supplementary/MGOA_data.tex`):
  built from ~25,000 gameplay videos covering **8 atomic tasks**, Log, Seed, Dirt,
  Stone (cobblestone), Iron, Gold, Diamond, Redstone.
- HF card: license MIT, size category 10M-100M, `webdataset` format, modality text
  (plus the implied frame/observation payload inside the webdataset shards).
- Conditioning is Goal (task text) + Observation (pixel frames) -> Action
  (keyboard/mouse), i.e. a pixel-grounded VLA-style policy dataset.

## Interpretation (inference)

- "Goal-Observation-Action" is structurally the same triple this repo records per
  turn (CycleGoal + observation/source-evidence + selected tool/params). The
  difference is modality: MGOA's observation is pixels and its action is low-level
  mouse/keyboard; the repo's observation is typed state and its action is a typed
  Action Card / primitive with logical params.
- The 8 atomic tasks are all material-acquisition (gather/craft), there is **no
  social, possession, obligation, or multi-actor content**. MGOA cannot inform the
  Social or Institutional WAM layers at all.

## Mechanically useful vs research contribution

- Mechanically useful: as a *contrast baseline only* (per lane brief). It shows the
  most recent (2025) Minecraft agent dataset is still single-actor, pixel,
  material-acquisition. It is not a starting consequence predictor for a structured
  social-material WAM (modality + scope mismatch).
- Overclaim risk: treating MGOA as reusable weights/data for this project. It is
  not, different observation space, no social events, no possession ledger.
