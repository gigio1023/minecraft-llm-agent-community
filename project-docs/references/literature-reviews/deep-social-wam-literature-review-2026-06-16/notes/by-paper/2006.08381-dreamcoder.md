## Primary-source facts (LaTeX-verified)

- **What it is**: a system that learns to solve problems by writing programs, and in doing so GROWS its own domain-specific library of reusable program subroutines plus a neural network that guides the search. It is NOT an LLM method (predates the LLM-evolution wave); it is the canonical library-learning / program-induction system.
- **Wake-sleep loop, three phases**:
    - Wake: solve tasks by searching for programs (in the current library/DSL), guided by the neural recognition model.
    - Abstraction sleep: grow the library by "compressing out" reused code from waking solutions. The objective is Bayesian/MDL: minimize description length of the library plus the description lengths of refactorings of solved programs. Crucially it compresses out reused SEMANTIC patterns (via refactoring), not just syntactic substrings.
    - Dream sleep: train the neural recognition model on "fantasies" (programs sampled from the library) and replays (waking solutions), so search gets faster as the library grows.
- **Refactoring is the hard part**: a program has infinitely many refactorings, so they bound lambda-calculus evaluation steps (set to 3) and use a version-space-algebra + equivalence-graph data structure with a dynamic program. Example: representing ~10^14 refactorings (rediscovering `map` from primitives + the Y-combinator) as a 10^6-node version space computed in minutes.
- **Results (abstract-stated)**: solves classic inductive-programming tasks and creative tasks (drawing, scene building). Rediscovers the basics of modern functional programming starting from 1960s Lisp; reconstructs vector algebra; and infers classical physical laws including Newton's and Coulomb's laws. Concepts are built compositionally from earlier-learned ones, yielding interpretable multi-layer symbolic libraries that transfer to new tasks.

## Interpretation (flagged as inference)

- DreamCoder is the "named subroutine retention" pattern the lane seed flags: the loop does not just solve tasks, it distills reusable, named, inspectable subroutines into a growing library and reuses them. This is the library-learning ancestor of this repo's skill library (`author_mineflayer_action` candidates promoted, retired, superseded).
- Its discovery signal is solving the GIVEN tasks (program correctness on input/output examples), an exact checker, plus an MDL/compression objective for which abstractions to keep. So even DreamCoder, the most "open-ended" library learner here, rests on an exact per-task correctness signal; the open-endedness is in WHICH abstractions to retain, not in scoring success.
- Compression-as-abstraction is a concrete, principled rule for "which discovered skill is worth keeping": keep the abstraction that most shortens (compresses) the set of verified solutions. This is more disciplined than "keep skills the LLM liked".

## Mechanically useful vs research contribution

- **Mechanically useful**: the library-growth discipline (after solving tasks, compress out reused patterns into named subroutines; keep those that minimize total description length; let a learned model speed up future search). For this repo: a principled promotion rule for `author_mineflayer_action` skills (promote a skill when it compresses/explains many verified action sequences), and a learned retrieval/guidance model over the skill library.
- **Not a contribution to copy**: DreamCoder's contribution is the wake-sleep Bayesian program-learning algorithm and its science demonstrations. The repo borrows the library-learning + compression-based-retention idea, not the algorithm wholesale.
- **Bound**: the retention objective (MDL) needs a set of VERIFIED solutions to compress. It applies cleanly to Physical/Material skills (verifier-confirmed action sequences). There is no compression-of-verified-solutions signal for "good social conduct" because the social solutions are not cleanly verifiable.
