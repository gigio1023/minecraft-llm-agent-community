- **title**: The Curse of Recursion: Training on Generated Data Makes Models Forget. arXiv v1 title was "Model Dementia: Generated Data Makes Models Forget"; the Nature 2024 camera-ready is "AI Models Collapse When Trained on Recursively Generated Data". Same paper.

## Primary-source facts (LaTeX-verified)

- **Definition.** Model collapse is "a degenerative process affecting generations of learned generative models, where generated data end up polluting the training set of the next generation of models; being trained on polluted data, they then mis-perceive reality." Two phases: **early model collapse** (the model loses information about the tails of the distribution) and **late model collapse** (the model entangles modes and converges to a distribution with little resemblance to the original, often with very small variance).
- **Distinct from catastrophic forgetting.** The paper is explicit: this is not catastrophic forgetting. Models do not forget previously seen data; they progressively misinterpret what they believe is real by reinforcing their own beliefs across generations.
- **Two causes (NOT three in this version).** Primary = **statistical approximation error** (finite sample size; non-zero probability that low-probability information is lost at each resampling step; vanishes only as samples -> infinity). Secondary = **functional approximation error** (function approximators are not perfectly expressive; e.g. fitting a 2-component Gaussian with one Gaussian; in the absence of statistical error this acts only at the first generation). (The Nature 2024 version adds a third, "learning error"; this arXiv version names two.)
- **The formal data-mixing process ("Learning with Generational Data").** Resample generation i+1 from `p_{i+1} = alpha_i * p_theta_{i+1} + beta_i * p_i + gamma_i * p_0`, with alpha+beta+gamma=1: alpha = fraction from the new model, beta = from the previous generation's data, gamma = from the original real distribution. The math sets alpha=gamma=0 (only the latest model's outputs); numerics use more realistic mixes. This `gamma_i` term is precisely the "keep original real data" lever later papers exploit.
- **Inevitability proof (discrete case).** With perfect functional approximation, the process X^i -> F_theta -> p_{i+1} -> X^{i+1} is a Markov chain whose only absorbing states are delta functions; with probability 1 it converges to a constant state, losing all information of the original distribution. A state with probability q <= 1/M is expected to be sampled < 1 time, so its information is lost. Tails go first; the limit is a delta. The authors note floating-point discreteness makes this argument general.
- **Gaussian case (with both errors).** They derive a lower bound on the Wasserstein-2 distance from the true distribution that grows over generations, using the Gelbrich/Gaussian universal W2 lower bound. The variance of the estimate diverges across generations.
- **LLM experiment (fine-tuning, not from scratch).** They fine-tune **OPT-125M** on **wikitext2**, generate a same-size synthetic dataset via 5-way beam search, and refit across generations. They explicitly did NOT train an LLM from scratch (cost / CO2). Findings: no data preserved -> perplexity degrades over generations (e.g. ~20 to ~28 points on the original task); generated text develops long tails (sequences the original model would never produce = accumulated errors) and collapses toward high-probability sequences. **Preserving 10% of the original real data each generation leads to "only minor degradation."** Penalizing repetition makes collapse worse, not better.
- **Authors' recommended mitigation.** "First mover advantage": access to the original (genuine human) data distribution must be preserved; data about real human interactions becomes increasingly valuable; provenance of web data is the open problem.

## Interpretation (flagged as inference)

- This is the founding statement of the strongest limit on naive self-improvement: a loop that trains on its own un-anchored outputs converges to a degenerate distribution, and the proof (in the idealized case) makes it inevitable, not merely likely. The mechanism is information loss by sampling, which no amount of loop engineering removes.
- The 10%-real-data result is the key crack in the "inevitable" framing and directly motivates the accumulating-data paper (2404.01413): keeping a fixed real anchor (`gamma_i > 0`) changes the dynamics. The repo's runtime verifier is a stronger anchor than "10% real data" because it is an external GROUND-TRUTH check on every transition, not just a retained data fraction.
- The fine-tuning-only caveat matters for honesty: the dramatic "models become useless" result is cleanest in from-scratch generative settings (GMM/VAE) and in theory; the LLM experiment shows degradation, not total collapse, and a small real-data fraction nearly arrests it.

## Mechanically useful vs research contribution

- **Mechanically useful**: the data-mixing formalism (alpha/beta/gamma) is a clean way to reason about any repo self-improvement loop's data diet; the diagnostic that "tails disappear first" gives an early-warning signal (watch rare/low-frequency transition types in the verifier log for under-representation); the 10%-real-anchor result justifies always mixing fresh externally-verified transitions into any training set.
- **Not a contribution to copy**: model collapse is a phenomenon to AVOID, not a method. The repo's research claim is unrelated; this paper is the boundary condition on a verifier-grounded loop.
- **Honest bound it supplies**: a loop trained on its own un-verified outputs degrades; therefore the repo loop must always be fed fresh, externally-verified data (the runtime verifier), never the actor's own generations scored by itself.
