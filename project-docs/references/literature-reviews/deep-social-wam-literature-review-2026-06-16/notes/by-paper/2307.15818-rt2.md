# RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control

- **title**: RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control
- **authors**: Brohan, Brown, Carbajal, Chebotar, ... Zitkovich (Google DeepMind)
- **year**: 2023
- **venue/source**: arXiv 2307.15818; CoRL 2023; project robotics-transformer2.github.io
- **arxiv_id**: 2307.15818
- **urls**: paper https://arxiv.org/abs/2307.15818 ; LaTeX at `papers/latex/2307.15818/`
- **source availability**: latex (full, `main.tex` + `appendix.tex`)
- **WAM layer(s)**: Physical (control), but its real point is *semantic transfer*. Reactive `p(a|o,l)`, no `o'` prediction, so not a WAM.

Part of the VLA-canon set missing from wave-1. RT-2 is where the term
**"vision-language-action model" (VLA)** is coined and where the central VLA bet
(reuse a web-pretrained VLM as the action backbone) is established.

## Primary-source facts

- **Research question**: can a large vision-language model (VLM) pretrained on
  web-scale image-text data be integrated *directly* into low-level robot control, so
  that web semantic knowledge transfers into action and produces emergent reasoning,
  rather than the VLM only doing high-level planning above a separate controller?
- **Core method**: represent **robot actions as text tokens** and co-train the VLM on
  robot trajectories alongside its original web vision-language tasks. Action vector
  -> a string the VLM emits as ordinary text, e.g.
  `"terminate Δpos_x Δpos_y Δpos_z Δrot_x Δrot_y Δrot_z gripper"` ->
  `"1 128 91 241 5 101 127"`. The model produces actions the same way it produces
  language; at inference, the text is de-tokenized back into robot commands.
- **Action space / tokenization**: 6-DoF end-effector displacement + gripper + a
  discrete terminate command; continuous dims discretized into **256 uniform bins**
  (inherited from RT-1). To make a *pretrained* VLM emit these, action bins are mapped
  onto existing vocabulary tokens: PaLI-X has unique tokens for integers <1000 (used
  directly); PaLM-E overwrites its 256 least-frequent tokens (a form of "symbol
  tuning"). Output is constrained to valid action tokens when prompted for a robot
  action.
- **Backbones**: two web-pretrained VLMs adapted into VLAs: **PaLI-X** and **PaLM-E**,
  in sizes from billions up to **55B parameters** (RT-2-PaLI-X-55B). No new action-only
  layers; weights fully shared across language and action.
- **Co-Fine-Tuning** (the key recipe detail): fine-tune on robot data *mixed with the
  original web data* rather than robot-only. This is what preserves and transfers the
  semantic knowledge; robot-only fine-tuning loses it.
- **Dataset**: RT-1's robot dataset (130k demos, 700+ tasks) for the robot side, plus
  the VLM's original web vision-language data in the co-fine-tuning mix.
- **Evaluation**: 6k+ real-world trials. Large gains in generalization to novel
  objects, backgrounds, and semantically varied instructions over RT-1. **Emergent
  capabilities** from web knowledge: place an object on a number/icon never in the
  robot data; pick "the object that could be a hammer" (a rock); with chain-of-thought
  prompting, multi-stage semantic inference. Physical *skills* stay within the robot-
  data distribution, but the model deploys them in new, semantically-cued ways.
- **Inference**: 55B model runs at 1-3 Hz via a multi-TPU cloud service queried over
  the network; the 5B version at ~5 Hz. "Largest model ever used for direct closed-
  loop robotic control" at the time.

## Interpretation (inference)

- RT-2's contribution over RT-1 is **not** a better action representation (it reuses
  256-bin tokens); it is the demonstration that **a frozen-knowledge web VLM, co-fine-
  tuned, makes robot policies semantically general**. The web pretraining is the source
  of generalization, not the robot data. This is the founding argument of the VLA
  paradigm and the reason every later VLA starts from a VLM backbone.
- Still a pure VLA (`p(a|o,l)`). RT-2 does not predict any future observation `o'`. Its
  "reasoning" is language reasoning about the *current* scene, not forward simulation.
  So it is squarely the "act" lineage a WAM augments with a "predict" lineage.

## Mechanically useful vs research contribution

- **Useful for THIS repo**: the structural lesson "reuse a large pretrained
  language/vision-language model's semantic knowledge instead of training a controller
  from scratch" is exactly the repo's bet (an LLM Actor Turn over typed tools). The
  repo already gets RT-2's *semantic* benefit from a frontier LLM without RT-2's motor-
  token machinery, because the repo's actions are typed tool calls, not 256-bin text.
- **Avoid / overclaim**: do not import "actions as text tokens" as a representation
  for the repo (the repo's schema-bound params are stronger and verifiable). Do not
  treat RT-2's emergent-reasoning anecdotes as evidence of social or material reasoning;
  they are object-level semantic generalization in single-agent manipulation.
