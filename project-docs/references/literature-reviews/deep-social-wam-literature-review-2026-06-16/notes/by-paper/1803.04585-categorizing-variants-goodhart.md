## Primary-source facts

- **Goal**: enumerate the distinct ways that optimizing a metric (a proxy) instead of the
  true goal fails. Goodhart's law: "when a measure becomes a target, it ceases to be a good
  measure." The paper defines four variants by their underlying MECHANISM.
- **The four variants** (definitions in the authors' terms):
  - **Regressional Goodhart**: "selection for an imperfect proxy necessarily also selects
    for noise." You select not only for the true goal but for the difference between the
    proxy and the goal ("the tails come apart"). Stated as the most fundamental variant and
    one that "cannot be avoided" when the metric is inexact.
  - **Extremal Goodhart**: "selection for the metric pushes the state distribution into a
    region where old relationships no longer hold" (like out-of-sample prediction).
    Sub-types: Model Insufficiency (the fitted relationship is only locally accurate) and
    Change in Regime (the proxy relates to the goal differently in different regions).
  - **Causal Goodhart**: when the proxy-goal correlation is non-causal, "intervening can
    change the relationship between the measure and proxy" so the proxy stops tracking the
    goal. Sub-types include Shared Cause Intervention, Intermediary Intervention, and Metric
    Manipulation.
  - **Adversarial Goodhart**: "an agent with different goals than the regulator causes the
    collapse." Named cases: Adversarial Misalignment Goodhart (the agent has goals the
    regulator is unaware of), Campbell's Law, and the Cobra Effect (Normal and Non-Causal
    variants) when the regulator uses incentives.
- **Mitigation**: discussed only per-variant, no general remedy. Regressional "cannot be
  avoided" with an inexact metric. For Causal (shared-cause), "more extreme interventions
  can be less effective than less extreme ones," so further optimization on the metric can
  be counterproductive.
- **Reproducibility**: conceptual; only illustrative equations and causal-graph diagrams,
  no datasets, code, or empirical results.

## Interpretation (labeled inference)

- This taxonomy is the vocabulary the rest of the lane uses. Gao 2210.10760 explicitly maps
  its two empirical terms to Regressional (the `alpha` term) and Extremal (the `beta` term)
  Goodhart. Skalse 2209.13085 formalizes when a proxy can avoid the reversal Adversarial
  Goodhart exploits. So the four names organize the whole area: Regressional and Extremal
  are about a learned/correlated PROXY; Adversarial is about an AGENT gaming the check;
  Causal is about checking the wrong quantity.
- The crucial split for the repo is proxy-side (Regressional, Extremal) vs check-side
  (Adversarial, Causal). A deterministic verifier removes the proxy-side variants but not
  the check-side ones.

## Relevance to this repo (mechanical import vs research contribution)

- **Mechanical import (diagnostic vocabulary)**: classify each verifier risk by variant.
  (1) A deterministic per-instance verifier is structurally immune to Regressional and
  Extremal Goodhart, because it is per-instance ground truth, not a learned/correlated proxy:
  there is no proxy-goal noise term to select on, and no fitted relationship to break at
  extremes. (2) It remains exposed to Adversarial Goodhart, an actor finding an
  instance-only shortcut that satisfies the check without doing the work (this is the
  directions-report "instance-only shortcut" risk). (3) It is exposed to Causal Goodhart if
  the verified quantity is not the quantity of interest, passing the check does not causally
  produce the true goal (e.g. verifying "item in inventory" when the goal is "item fairly
  acquired").
- **Why a learned reward model is the more hackable choice (tie)**: switching the success
  signal from a deterministic verifier to a learned reward model RE-ADDS the proxy-side
  variants (Regressional + Extremal), which Gao 2210.10760 shows scale smoothly worse with
  optimization. The deterministic verifier's only residual exposure is the check-side
  variants, which are addressable by hardening (differential/invariance checks, choosing the
  right verified quantity) rather than unavoidable.
- **Research contribution it is NOT**: the taxonomy is conceptual scaffolding (support); the
  repo borrows the NAMES to reason about which failure modes its verifier does and does not
  face. Not a deliverable.
- **What it does not cover**: no empirics, no embodied/social instance, no quantified
  severity (that is Gao and Pan). It is a map of failure KINDS, not magnitudes.
