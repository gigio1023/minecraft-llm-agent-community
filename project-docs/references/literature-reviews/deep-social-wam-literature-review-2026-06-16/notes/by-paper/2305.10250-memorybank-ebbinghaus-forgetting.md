## Primary-source facts (what the paper states/shows)

- Goal: give LLMs a long-term memory mechanism for sustained interaction (personal companion, psychological counseling). Demonstrated via an AI companion chatbot, SiliconFriend, built over ChatGPT, ChatGLM, and BELLE.
- Three pillars.
  1. Memory storage. Records multi-turn conversations chronologically with timestamps. Builds a hierarchical event summary: condense daily dialogues into a daily event summary, then synthesize into a global summary. Builds dynamic personality understanding: per-day personality insights aggregated into a global user-personality portrait. (Distillation is via fixed summarization prompts to the LLM.)
  2. Memory retrieval. Dual-tower dense retrieval (Dense Passage Retrieval style). Each conversation turn and event summary is a memory piece pre-encoded to a vector; the whole store is indexed with FAISS; the current conversation context is the query. Embedding model is interchangeable (MiniLM for English, Text2vec for Chinese).
  3. Memory updating (the distinctive part). Inspired by the Ebbinghaus Forgetting Curve, modeled as exponential decay R = e^(-t/S), where R is retention, t is time since learning, S is memory strength. S is a discrete value initialized to 1 on first mention; when a memory item is recalled during conversation, S increments by 1 and t resets to 0, so a recalled memory is forgotten with lower probability. The paper states this is an exploratory, highly simplified model. (A commented-out variant in the source used 3 discrete strength levels with the curve starting at day k1 = 7 and k2 = 30.)
- SiliconFriend pipeline: optional LoRA tuning on 38k psychological dialogues (open-source backbones only), then MemoryBank integration. On each turn, retrieved relevant memory + global user portrait + global event summary are organized into the prompt.
- Evaluation (from abstract/framing): qualitative analysis with real user dialogs, and quantitative analysis with simulated dialogs where ChatGPT plays users with diverse characteristics generating long-term dialog contexts. Reported capability: empathetic response, recall of relevant memories, user-personality understanding for long-term companionship.

## Reproducibility status

- Self-reported, with public code. The forgetting mechanism is a closed-form heuristic (not learned). The companion evaluation is qualitative + simulated-dialog quantitative. Not environment-verified by me; details quoted from the source.

## Interpretation (my inference, labeled)

- MemoryBank is the canonical instance of explicit, principled forgetting and consolidation in LLM memory: it is the most-cited mechanization of "memory decays unless reinforced by recall." This is exactly the consolidation/forgetting angle the lane asks for, and the recall-strengthens-retention rule (S += 1, t := 0 on recall) is a clean, cheap primitive.
- The hierarchical distillation (turn -> daily summary -> global summary) is the same many-events-to-abstraction pattern as Generative Agents' reflection, but without the citation. For the repo, the abstraction is fine, the gap is that MemoryBank's distilled summaries and personality portraits are free prose with no pointer to the verified events that justify them.
- The forgetting curve is double-edged for an evidence-grounded system. Recency-weighting is reasonable for surfacing, but the repo must NOT let probabilistic forgetting delete the immutable evidence (the source-of-truth event). Forgetting should be a surfacing/ranking policy over a distilled tier, with the underlying verified artifact retained (this is exactly TierMem's immutable-Tier-2 vs decaying-Tier-1 split, 2602.17913).

## Mechanically useful vs research contribution

- Mechanically useful: the Ebbinghaus recall-reinforced decay (R = e^(-t/S), S increments on recall) as a retrieval-ranking / surfacing weight for cross-cycle memory; hierarchical event-summary distillation; dual-tower FAISS retrieval as the standard recall substrate; per-user/per-agent dynamic portrait as a model for a relationship/belief summary tier.
- Research contribution it is NOT for this repo: MemoryBank is a personalization memory system for companion chat. It has no typed material/social state, no transition prediction, no verifier. Its memories assert without citing evidence. The forgetting model is explicitly "highly simplified" and not validated as cognitively accurate. It is a support primitive (ranking and consolidation), not the predictive object.

## WAM layer(s) informed

- Cross-cycle continuity / forgetting primitive, with most natural fit at the Social layer (the dynamic user portrait = a relationship/belief summary that evolves). But the portrait must, in the repo, be a ledger over verified events, not a free-prose personality summary.
