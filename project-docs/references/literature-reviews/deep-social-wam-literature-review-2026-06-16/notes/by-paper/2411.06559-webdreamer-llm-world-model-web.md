## Primary-source facts (LaTeX-verified)

- **The thesis, verbatim sense**: "Pretrained on web-scale data, LLMs have implicitly acquired both structural knowledge of websites and common sense needed to predict the outcomes of proposed actions," so an LLM can simulate transitions in a complex environment without a trained dynamics model. This is the closest published statement of the repo's "cheapest first predictor is an LLM predicting the next structured state."
- **Mechanism = MPC with an LLM as both sim and score**: a learned simulation function `sim(o, a)` predicts the outcome of an action as a NATURAL-LANGUAGE description of the next state; a scoring function `score(tau)` evaluates a simulated trajectory; the agent executes `a* = argmax_a score(sim(o, a))` over a finite horizon H. Both `sim` and `score` are the SAME LLM (GPT-4o) prompted differently.
- **Scoring is a 3-scale progress judgment**: GPT-4o scores each simulated trajectory as complete (1.0), on track (0.5), or incorrect (0); the action score is averaged over multiple simulated trajectories and scorings. Candidate actions are first top-k sampled, then LLM-self-refined to prune unavailable ones.
- **Why model-based and not tree search**: live websites have "state-changing and irreversible actions" (submit an order, create an account), so backtracking (the cornerstone of tree search) is infeasible. Simulating in the LLM's head avoids irreversible real actions. This is the same safety argument the repo makes for an advisory (never-acting) predictor.
- **Ablation isolating the world model**: replacing simulation with direct action reranking (ask the scorer to evaluate each candidate action without simulating its outcome) gives "some improvement over the reactive baseline, but the gain is small and still falls well behind WebDreamer." Quote: "the LLM-based world model simulation plays a crucial role." So the predicted DYNAMICS, not just a better action ranker, carry the gain.

## Numbers as stated

- VisualWebArena (233 human-verified tasks): +34.1% relative success over the reactive agent, trailing tree search only slightly, while 4-5x more efficient than tree search.
- Online-Mind2Web: +42.3% relative over reactive. Mind2Web-Live: +23.8% relative.
- Fine-tuning Qwen2-VL-7B into Dreamer-7B with synthesized in-domain data: +4.7% absolute on VWA, +4.0% Online-Mind2Web, +4.8% Mind2Web-Live, beating vanilla Qwen2-VL-72B and comparable to GPT-4o on the two online benchmarks.
- Automatic eval pipeline has 85% agreement with human judgment (VWA).

## Reproducibility status

Self-reported, code and trained world-model weights (Dreamer-7B/72B) released. Online benchmarks make exact reproduction environment-dependent. The world-model fidelity is measured only indirectly through downstream task success, not as per-step transition accuracy.

## Named failure modes (load-bearing for this lane)

- **Action-proposal hallucination within simulation**: "the action proposal within simulation is biased toward generating seemingly relevant actions for task completion, even when these actions are not available based on the predicted outcome." The LLM imagines doing things its own predicted state does not support.
- **Error accumulation over horizon**: "simulating multiple steps ahead is challenging due to error accumulation," consistent with other LLM-simulation work. Longer H does not monotonically help.

## Relevance to this repo

- **Mechanical import (highest in this lane)**: WebDreamer is the operational blueprint for an advisory, never-acting LLM dynamics model. `sim(o,a)` = the repo's prompt-based WAM predicting the next structured state; `score` is a progress judgment. The repo's advantage over WebDreamer is decisive: WebDreamer must use a SECOND LLM call (`score`) as its judge because a web environment gives no cheap ground truth, whereas the repo's deterministic verifier auto-labels the true next-state delta at near-$0. So the repo can replace WebDreamer's LLM-`score` with the verifier, which removes the self-scoring weakness entirely.
- **Calibration / failure detection**: WebDreamer's two named failures (imagining unavailable actions, error accumulation) are exactly what the repo's verifier catches: a predicted possession move or claim change that the runtime does not confirm is a measurable prediction error per step, not a silent compounding one. The repo's per-step grounding is the fix WebDreamer leaves to "future work."
- **The ablation is the argument for the repo**: WebDreamer shows the predicted dynamics (not just reranking) carry the gain, which justifies investing in a structured-state predictor rather than only a better action scorer.
- **Research contribution vs support**: building the verifier is support. The contribution is showing an LLM `sim` over TYPED social-material state, scored by a deterministic verifier rather than a second LLM, is a sharper instance of WebDreamer's paradigm than WebDreamer itself.

## WAM layer(s) informed

Physical and Material directly (predict-then-score the next state of a structured environment). Social is the open extension: WebDreamer never simulates another agent's reaction; the advisory pattern transfers but the prediction target (a partner's response) is far harder and ungrounded mid-rollout.
