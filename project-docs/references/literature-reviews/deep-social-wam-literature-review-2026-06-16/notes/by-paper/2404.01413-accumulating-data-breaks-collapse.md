## Primary-source facts (LaTeX-verified)

- **The reframing.** Prior model-collapse work (2305.17493, 2307.01850, and Dohmatob et al. 2402.07712) assumed each generation REPLACES the previous data with synthetic. The more realistic web dynamic is that synthetic data ACCUMULATES alongside real data. The paper asks what accumulation does to collapse, and chooses a "maximally pessimistic" accumulate setting (synthetic dumped uncontrolled).
- **Main empirical result.** Replacing real data with each generation's synthetic data does tend toward collapse (confirmed). But ACCUMULATING successive synthetic generations alongside the original real data AVOIDS model collapse, across model sizes, architectures, and hyperparameters. Confirmed on causal transformers (language modeling), diffusion models (molecule conformation), and VAEs (images).
- **Theory (linear regression, extending Dohmatob et al.).** In the REPLACE setting, test squared error grows LINEARLY with the number of iterations n (unbounded), at a rate set by a noise-to-signal ratio `sigma^2 * d / (T - d - 1)`. In the ACCUMULATE setting, the test error has a FINITE upper bound independent of n: model collapse no longer occurs.
- **The mechanism (why accumulation works).** When data accumulate, iteration i contributes only a fraction 1/i of the training set, so the noise injected by the i-th generation of synthetic data has its effect on the MSE shrunk proportional to 1/i^2 (squared error). Since the series sum 1/i^2 converges (it is bounded), the test error cannot grow indefinitely. Replacement discards prior data, so each iteration's fresh noise compounds undamped.
- **A "halfway" caveat.** Replacing the previous dataset with a PURE synthetic dataset that grows in size (size i*T at iteration i) does NOT prevent divergence: the error then scales as O(log n), still unbounded. Merely using more synthetic data is not enough; the REAL data must be retained in the mix.
- **Terminology nuance (from the discussion).** The authors note "model collapse" has been used for at least four distinct phenomena: (0) unbounded test-error blowup (their definition), (1) modal collapse (collapse to a few modes), (2) collapse to uniformity, (3) amplification of artifacts. Claims across papers are not directly comparable without specifying which.

## Interpretation (flagged as inference)

- This paper is the principled answer to "is the curse of recursion fatal?": NO, provided you keep accumulating real (or externally-anchored) data rather than replacing it. It does not contradict 2305.17493 / 2307.01850; it identifies the data-handling assumption (replace vs accumulate) that flips the outcome. In the 2305.17493 formalism, this is `gamma_i > 0` and never discarding the original.
- For the repo thesis this is the single most encouraging limits-paper: a self-improvement loop does NOT inevitably collapse if its data diet keeps including externally-grounded data. The repo's runtime verifier produces a fresh stream of externally-checked transitions every cycle, which is a stronger anchor than "retain the original real corpus" because it re-grounds each NEW transition against physical truth, not just preserves old data.
- Honest caveat: the clean finite-bound proof is for linear regression and confirmed empirically for generative pretraining; it bounds DISTRIBUTIONAL collapse (test error), not reward-hacking or safety drift in an RL/agent loop. Accumulation prevents the statistical-sampling collapse of the model-collapse line; it does not by itself prevent the proposer-equals-scorer collapse that H5 documents (2505.21444) or the safety drift the trilemma paper (2602.09877) documents. Those need an external verifier on the SUCCESS SIGNAL, not just on the data.

## Mechanically useful vs research contribution

- **Mechanically useful**: the accumulate-don't-replace rule is a direct loop-design constraint for the repo. Any self-training of an advisory WAM should append fresh verifier-labeled transitions to an accumulating buffer (keep old + add new), never train on a sliding window of only the latest self-generated data. The 1/i^2 intuition gives a quantitative reason the buffer stabilizes.
- **Not a contribution to copy**: this is a mitigation principle, not a method the repo would claim as research.
- **Honest bound it supplies**: distributional collapse is avoidable with data accumulation, but accumulation is necessary-not-sufficient. It fixes the data-source limit, not the success-signal limit. Pair it with verifier-grounded scoring.
