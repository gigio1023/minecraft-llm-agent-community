- title: Video PreTraining (VPT): Learning to Act by Watching Unlabeled Online Videos
- authors: Baker, Akkaya, Zhokhov, Huizinga, Tang, Ecoffet, Houghton, Sampedro, Clune (OpenAI)
- year: 2022
- venue: NeurIPS 2022
- arxiv_id: 2206.11795
- urls: paper https://arxiv.org/abs/2206.11795 ; data/weights https://github.com/openai/Video-Pre-Training
- source_availability: latex (18 tex files, `papers/latex/2206.11795/`)
- reproducibility_status: reproducible (weights, IDM, contractor data spec, code released)
- WAM layer(s): Physical (action-conditioned dynamics); methodologically informs all layers via the auto-label idea.

## Primary-source facts

- Research question: can the internet-scale pretraining paradigm be extended to
  sequential decision domains (Minecraft) where public video has **no action labels**?
- Method (`2_method.tex`): two-stage. (1) Train a **non-causal Inverse Dynamics
  Model** `p_IDM(a_t | o_1..T)` on a small set of labeled contractor data; the IDM
  may use future frames `o_{t'>t}`, which makes it far more data-efficient than
  behavioral cloning. (2) Run the IDM to pseudo-label a large unlabeled web-video
  corpus, then train a behavioral-cloning foundation policy on those labels.
- Action space: native human interface, keyboard presses + continuous mouse at
  20 Hz. Observation: raw pixels (640x360 video downsized from 1280x760 recorder).
- Dataset shape: contractors ran a custom MCP-Reborn recorder that wrote, in 5-min
  chunks, (a) the video file, (b) an **actions JSONL** (one JSON per game tick),
  and (c) a Minecraft state file. (`a_1_contractor_data.tex`)
- Data cost (the key cost anchor, `a_1_contractor_data.tex`): contractors paid
  **$20/hour**. ~4,500 hours used cost ~$90,000; ~$160k total over the project.
  IDM data-scaling (`3_4_data_scaling.tex`): crafting needs ≥10 h of IDM data,
  rises quickly to **100 h**, then plateaus. The paper's headline efficiency claim:
  "we could likely obtain most of our results with an IDM trained using only
  **$2000 worth of data**" (≈100 hours).
- Result: first computer agents to craft diamond tools (≈24,000 env actions);
  human-level on several tasks after BC + RL finetuning.

## Interpretation (inference)

- The entire $90k-$160k cost exists for one reason: **Minecraft video on the web
  has no action labels**, so VPT had to buy a small labeled seed and amortize it
  through an IDM. The IDM is a *learned auto-labeler*.
- The non-causal IDM trick (use future frames to infer the action that happened)
  is exactly what a verifier does symbolically: it observes the before/after and
  certifies which action occurred and whether it succeeded.

## Mechanically useful vs research contribution

- Mechanically useful for THIS repo: the *paradigm* "auto-label a large unlabeled
  corpus from a cheap labeling function." This repo's Mineflayer runtime already
  emits the action (the validated tool call + typed params) AND the verified delta
  (`last_tool_result.beforePosition/afterPosition/distanceMoved`,
  `runtime_hooks[].progress_classification`) per turn. So the repo gets the IDM's
  output (`(o_t, a_t, o_{t+1})` triplets) **for free**, no IDM training, no
  pixel decoding, no $90k.
- Would be overclaim: claiming VPT-scale generalist behavior. VPT's value is pixel
  behavioral priors from 70k+ web hours; this repo neither has nor needs that for a
  structured social-material WAM. Do not frame "we have a verifier" as a model
  contribution, it is the cheap label source, the support infrastructure.
