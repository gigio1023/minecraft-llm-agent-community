# JARVIS-VLA - Post-Training VLMs to Play Visual Games with Keyboard & Mouse (ActVLP)

- **authors**: Li, Zhang, Cai et al. (Team CraftJarvis)
- **year**: 2025 - **venue**: arXiv (NeurIPS 2025 track)
- **arxiv_id**: 2503.16365
- **urls**: https://arxiv.org/abs/2503.16365 ; code https://github.com/CraftJarvis/JarvisVLA ; weights https://huggingface.co/CraftJarvis/JarvisVLA-Qwen2-VL-7B ; data https://huggingface.co/datasets/CraftJarvis/minecraft-vla-sft (216 parquet shards, ~106 GB, 1M-10M rows, MIT)

## Primary-source facts

- **Research question**: can a general VLM be post-trained into a Minecraft VLA
  that follows instructions across **1,000+ atomic tasks** (craft, smelt, cook,
  mine, kill) using the **human keyboard+mouse** interface?
- **Architecture**: LLaVA-style - ViT visual encoder + 2-layer MLP projector +
  autoregressive LLM (Qwen2-VL / LLaVA-Next base). **Non-Markovian**: a history
  of observation images is placed in the prompt for partial observability. An
  **action decoder** repurposes the **51 least-used tokenizer tokens** (22 for
  mouse, 29 for keyboard) à la RT-2; continuous actions discretized to bins;
  **action chunking** for temporal coherence.
- **Training - Vision-Language Post-Training (ActVLP), 3 stages**:
  1. text-only world-knowledge SFT (ViT frozen);
  2. multimodal vision-language alignment + spatial grounding SFT (full unfreeze);
  3. action post-training via imitation learning on trajectories (vision frozen,
     tokenizer extended with action tokens, LLM fully fine-tuned).
- **Datasets**: ~277K knowledge QA + 35K caption/VQA keyframes + 404K spatial-
  grounding points (non-trajectory); **7.4M trajectory frames** (OpenAI
  contractor + YouTube + VPT/JARVIS-1 rollouts) + 6.4M synthesized GUI craft/smelt
  entries; ~10B action-post-training tokens.
- **Findings**: SOTA on MCU atomic tasks vs prior Minecraft agents; scaling the
  **non-trajectory** vision-language data improves downstream action performance
  (a scaling-law claim). Open-sourced code, models, datasets.

## Interpretation (inference)

- JARVIS-VLA is a **monolithic VLA**: `p(a | o_history, language)`, pixels in,
  keyboard+mouse out, no predictive `o'` ⇒ not a WAM. Its contribution is the
  *training recipe* (understand-before-act post-training), not a consequence
  model.
- It is the strongest evidence that "understanding the world in language first"
  helps action - which, read sideways, supports this repo's bet that a
  **structured-state** understanding layer (not pixels) can carry the social/
  material reasoning, with the body handled separately.

## Mechanically useful vs research contribution

- **Mechanically useful**: (a) the staged understand->ground->act curriculum is a
  template if this repo ever fine-tunes a small local model on its own
  transcripts; (b) `minecraft-vla-sft` is a concrete public corpus for a visual
  lane; (c) action-token-via-vocab is an engineering trick, not needed here
  (Mineflayer actions are function calls, not tokens).
- **What would be overclaim**: claiming the keyboard+mouse VLA as the social
  substrate - it has no notion of possession, obligation, or other actors; its
  metric is atomic-task success on MCU (a competence gate).

## WAM layer(s) informed

Physical-WAM (instructable executor + training recipe). Reproducible: weights +
code + data public.
