## What it introduced and why it mattered (plain language)

DIAMOND (DIffusion As a Model Of eNvironment Dreams) is the canonical "diffusion world
model." Its claim, right in the title: the small visual details that discrete-token
world models (like IRIS) compress away can be exactly the details an agent needs (a
distant pedestrian, a tiny enemy, a ball). So instead of tokenizing frames, DIAMOND
keeps the world model in image space and predicts the next frame with a diffusion
model (the same generative method that powers modern image generators). It then trains
the agent with reinforcement learning inside that diffusion world model. This mattered
because it set a new best (mean human-normalized score 1.46) among agents trained
entirely within a world model on Atari 100k, and gave concrete evidence for "visual
fidelity of the predicted next state can change downstream control performance."

Newcomer terms:
- Diffusion model: a generative model that learns to reverse a gradual noising process;
  to sample, it starts from noise and denoises step by step. Good at sharp, multi-modal
  image distributions, hard to mode-collapse.
- Compounding / accumulating error: when a model predicts one step from its own previous
  prediction, errors snowball over a long rollout. Discrete tokens were partly motivated
  as a way to limit this.
- EDM: a specific diffusion formulation (Karras et al. 2022) with a particular noise
  schedule and network preconditioning, which DIAMOND adapts.

## Primary-source facts

- Architecture: a diffusion-based next-frame predictor built on the EDM formulation
  (drift `f = 0`, `g(tau) = sqrt(2 sigma' sigma)`), parameterized with Karras
  preconditioners, a standard 2D U-Net for the vector field. Conditioning: a buffer of
  `L` past observations concatenated channel-wise to the noisy next frame; actions
  injected via adaptive group-normalization in the U-Net residual blocks. Separate
  CNN-LSTM reward/termination model. Sampling: Euler method (found sufficient vs higher-
  order or stochastic samplers).
- Behavior learning: actor-critic (shared CNN-LSTM, policy + value heads), policy via
  REINFORCE with value baseline, value via lambda-returns; trained entirely in
  imagination, with periodic real-env data collection to update the world model (the
  IRIS/Dreamer loop).
- Input/output: pixels in, pixels out (image-space prediction, no discretization).
  Action space: discrete Atari actions.
- Evaluation protocol + metrics: Atari 100k (26 games). Mean HNS 1.46, IQM 0.64;
  superhuman on 11 games; new best among agents trained entirely within a world model.
  Notably strong where small details matter (Asterix, Breakout, Road Runner). Compute:
  ~12GB VRAM, ~2.9 days/game on one RTX 4090 (1.03 GPU-years total over 5 seeds x 26).
- Generality demo: trains the same diffusion WM on 87 hours of static Counter-Strike:
  Global Offensive play to produce a playable "neural game engine" for the Dust II map.
- Stated limitations (theirs): mainly discrete-control evaluation (continuous left
  open); frame-stacking is a minimal memory mechanism (an autoregressive transformer
  over time would scale memory better); reward/termination not yet integrated into the
  diffusion model itself.

## Interpretation (inference, labeled)

- DIAMOND is the cleanest empirical version of the debate's pro-fidelity argument inside
  a control setting: it shows that what the world model preserves in its predicted next
  state (not just how the policy is trained) measurably changes performance. That is
  evidence that the representation choice is not cosmetic.
- But the same paper exposes the cost: ~1 GPU-year for 26 tiny Atari games, frame-
  stacking as the memory hack, no long-horizon transformer. This is the practical face
  of "pixel prediction is expensive," which the JEPA camp (2506.09985) uses to argue for
  predicting in representation space instead.

## Mechanically useful vs research contribution

- Mechanically useful: the lesson "decide what the model must preserve based on what the
  decision depends on" transfers directly to a structured-state social WAM: a social WAM
  must preserve the typed facts a social claim depends on (possession, durability,
  obligation), the analog of DIAMOND keeping the distant pedestrian. DIAMOND also
  demonstrates a WM doubling as an inspectable interactive simulator (drop-in env), which
  is a pattern for a human-reviewable Minecraft visual sidecar (optional, not authority).
- Avoid / overclaim: do not read DIAMOND as "pixels are the right target for this repo."
  Its lesson is about preserving decision-relevant detail; in a Mineflayer runtime the
  decision-relevant detail is typed state, not RGB. Do not adopt RL-in-imagination as the
  runtime controller (conflicts with LLM-proposes / runtime-owns-truth / WAM-advisory).
