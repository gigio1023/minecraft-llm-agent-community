# WorldLLM: Improving LLMs' world modeling using curiosity-driven theory-making

- **title**: WorldLLM: Improving LLMs' world modeling using curiosity-driven theory-making
- **authors**: Guillaume Levy, Cedric Colas, Pierre-Yves Oudeyer, Thomas Carta, Clement Romac (Inria Flowers AI and CogSci, Univ. Bordeaux; MIT; Hugging Face)
- **year**: 2025 (v3 Jan 2026)
- **venue/source**: arXiv preprint (ICLR 2025 template); Inria HAL copy
- **arxiv_id**: 2506.06725
- **urls**: paper https://arxiv.org/abs/2506.06725 ; html https://arxiv.org/html/2506.06725v3
- **source availability**: latex (tarball extracted, `papers/latex/2506.06725/main_v2.tex`)

## Primary-source facts (what the paper states)

- Goal: autonomously improve an LLM's ability to predict environment dynamics, WITHOUT gradient fine-tuning, by giving the LLM natural-language hypotheses in its prompt and refining those hypotheses from self-collected evidence.
- Formal object: the LLM world model is `P^LLM(s_{t+1} | s_t, a_t, H)`, where `H` is a set of natural-language hypothesized theories placed in the prompt. The thing being optimized is `H` (a few sentences of theory), not the LLM weights. Objective (Eq.): maximize the forward model's likelihood over all transitions while minimizing |H| (smallest set of hypotheses).
- Three interacting modules (the autoresearch loop made literal):
    1. **Statistician** = the LLM forward model `P^St(s'|s,a,H)`, scores how well current hypotheses predict collected transitions (via the log-likelihood the LLM assigns to the true next state).
    2. **Scientist** = a second LLM as a Bayesian proposal distribution; uses the **Metropolis algorithm** to propose and accept/reject candidate hypothesis sets given collected transitions (accept if the candidate raises the Statistician's log-likelihood on the data). Uniform prior over hypothesis space.
    3. **Experimenter** = a curiosity-driven RL policy that collects transitions with **low log-likelihood** under the current Statistician+hypotheses (the data most likely to expose where the theory is wrong).
- The loop alternates: Experimenter collects evidence -> Scientist refines hypotheses -> repeat (T iterations). Stated inspiration: "how humans, from children to scientists, actively update their internal world model by performing experiments."
- Environment is **reward-LESS** (reward always 0): there is no task to solve. Curiosity exists purely to gather model-improving data. Tested in Playground-Text (textual object-manipulation game; grow Plants, Small Herbivores, Big Herbivores via a tech tree). LLM = Phi-3-mini for both Statistician and Scientist; Experimenter is a small MLP policy trained with PPO over a symbolic state.
- Three curiosity rewards studied: **RL-LogP** (intrinsic reward = prediction error `-log P^St(s'|s,a,H)`, the ICM/RND-style signal), **RL-ALP** (absolute Learning Progress, |loglik change between hypothesis updates|), **RL-ALPEXP** (ALP per hand-defined transition-type partition).
- Key results:
    - Giving the Statistician hypotheses ALWAYS beats no-hypotheses, for every transition type and every data-collection strategy, with no gradient updates.
    - **RL-LogP** (prediction-error curiosity) explores the whole transition space and ends up near the O-Hardest data distribution; it produces hypotheses on par with or better than the oracle data collectors, and the curiosity policy solves the full tech tree as a byproduct.
    - **RL-ALP** FAILS: it collects mostly trivial "Standing" (movement) transitions and produces hypotheses that only explain those. Diversity of collected transitions is what the Scientist needs.
    - vs fine-tuning (F-LogP, SFT on the same collected transitions): fine-tuning reaches perfect (1.0) in-distribution top-3 accuracy, but in a generalization environment where the observation SYNTAX is changed, the natural-language-hypothesis approach is argued to generalize better (the produced theories carry abstract rules; F-LogP overfits surface form). The authors note the current Scientist often embeds concrete examples in hypotheses, which limits generalization, and propose favoring shorter hypotheses to force abstraction.

## Interpretation (flagged as inference)

- This is the single most on-target precedent for the lane: an LLM agent that runs its own experiments (chooses which transitions to collect) for the explicit purpose of improving its OWN predictive world model, where the model stays advisory (it predicts next states; it does not act). The Scientist/Statistician/Experimenter triple is, structurally, the ENPIRE reset-rollout-verify-refine loop applied to the world model rather than a robot policy, and applied via prompt-level theories instead of code edits.
- The "advisory and verifier-grounded" thesis is directly supported: hypotheses are accepted only when they raise the Statistician's likelihood on REAL collected transitions (the environment is the scorer of fit), not when the Scientist merely asserts them. The Scientist proposes; the data disposes.
- The RL-ALP failure is a concrete, transferable warning: a naive intrinsic-motivation signal can collapse onto the cheapest-to-reach transitions (here, walking around) and never probe the interesting dynamics. This is the abstract version of the noisy-TV problem and a caution for any curiosity-driven exploration the repo adds.

## Mechanically useful vs research contribution

- **Mechanically useful**: the three-module decomposition is a ready blueprint for autonomously improving the repo's advisory WAM at near-$0. Map: Statistician = the WAM predictor (typed-delta predictor over Minecraft+social state); Scientist = an LLM that proposes natural-language correction rules ("placing a crafting table on water fails"; "a borrowed tool returned late lowers the lender's future willingness"); Experimenter = a policy/heuristic that steers the actor toward transitions the WAM predicts poorly, scored by the runtime verifier's real `(state, action, next-state)`. Prompt-level hypotheses (no gradient training) fit the repo's no-paid-LLM-training, advisory posture. The interpretability of natural-language theories matches the repo's "evidence, not hidden preselection" rule.
- **What to avoid / overclaim**: Playground-Text is a tiny, deterministic, fully-observable object game; the result that "complex transitions are naturally harder to predict, so prediction-error curiosity orders exploration well" is explicitly flagged by the authors as unlikely to hold in complex environments. Do not claim WorldLLM-style curiosity will cleanly order Minecraft social exploration. And the curiosity Experimenter here optimizes for low-likelihood transitions regardless of social cost; in a social world that objective would reward anti-social probing (take an item to see what happens), which is the named bound for this lane (see theme file).

## Limitations (theirs + spotted)

- Theirs: structurally-simple environment; lightweight non-LLM Experimenter; one LLM model (Phi-3-mini, weak as a Scientist at abstract hypotheses); Metropolis updates only on most-recent transitions cause forgetting under non-stationary collection (suggest replay buffer / particle filters); local-optima risk.
- Spotted: no notion of intervention COST or safety; the curiosity objective is value-free. The hypothesis space is unconstrained natural language, so the WM's "improvement" is only as trustworthy as the likelihood-on-collected-data signal, which can be gamed if the Experimenter collects a biased transition set (exactly the RL-ALP failure).
