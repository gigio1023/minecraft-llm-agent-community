# GNS: Learning to Simulate Complex Physics with Graph Networks

- title: Learning to Simulate Complex Physics with Graph Networks (Graph Network-based Simulators, GNS)
- authors: Alvaro Sanchez-Gonzalez, Jonathan Godwin, Tobias Pfaff, Rex Ying, Jure Leskovec, Peter W. Battaglia
- year: 2020
- venue: ICML 2020
- arxiv_id: 2002.09405
- urls: paper https://arxiv.org/abs/2002.09405 , code
  https://github.com/deepmind/deepmind-research/tree/master/learning_to_simulate
- source_availability: latex (papers/latex/2002.09405/main.tex)

## Primary-source facts (what the paper states)

- Research question: can one general learned simulator predict the dynamics of many physical
  materials (fluids, rigid solids, deformables, and their interactions) directly from data,
  and generalize to far more particles and longer horizons than seen in training?
- Model architecture: a learnable simulator `s_theta` that iterates a one-step dynamics model
  `d_theta` under a fixed Euler-integrator update (`X_{k+1} = Update(X_k, d_theta)`). `d_theta`
  uses an "encode-process-decode" Graph Network scheme:
  1. Encoder: state X (a set of N particles) -> latent graph G0. Each particle is a NODE
     (`v_i = eps_v(x_i)`); directed EDGES are added between particles within a connectivity
     radius R, with edge embeddings `e_ij = eps_e(r_ij)` from pairwise properties (relative
     displacement and its magnitude in the relative variant).
  2. Processor: M rounds of learned message passing (`G^{m+1} = GN^{m+1}(G^m)`), with residual
     connections; uses interaction-network-style GN blocks (no global update in the main model).
  3. Decoder: extracts per-node dynamics info `y_i = delta_v(v_i^M)`, interpreted as per-particle
     acceleration, fed to the Euler integrator.
- Input / output: input per-particle state = position, last C=5 velocities, and static material
  features (water / sand / goop / rigid / boundary). Output = per-particle acceleration. State is
  STRUCTURED (typed particles + relations), NOT pixels.
- Action space: none in the usual RL sense. GNS is a pure learned SIMULATOR / world model
  `p(o' | o)`; "actions" are external forces / globals appended as node features. (This matters
  for the WAM taxonomy: GNS is a WM, not a WAM; it has no coupled action generation.)
- Observation / state representation: particle graph. Relative-position encoder variant imposes
  spatial-translation invariance; edges recomputed by nearest-neighbor search each step.
- Training objective: supervised L2 on per-particle acceleration from single-step pairs
  `(x^{t}_i, x^{t+1}_i)`. Crucial trick: corrupt training inputs with random-walk noise
  (`sigma_v = 0.0003`) so the model is robust to its own accumulated rollout error (the inputs
  it sees at test time are its own noisy predictions). All inputs/targets normalized online.
- Evaluation protocol + metrics: one-step MSE and full-rollout MSE on held-out trajectories;
  also distributional metrics (optimal transport via Sinkhorn, MMD). Generalization tested by
  changing particle count, object shape/number, initial conditions, and trajectory length.
- Datasets: ~17 domains (Water-3D, Sand-3D, Goop-3D, BoxBath, FluidShake, WaterRamps, etc.),
  typically 1000 train / 100 val / 100 test trajectories, 300-2000 steps each. Generated by
  established simulators (SPH SPlisHSPlasH, PBD FleX, MPM Taichi-MPM).
- Key results: a single architecture handles all materials; long-range performance is governed
  mainly by the NUMBER OF MESSAGE-PASSING STEPS and by the noise-injection training trick.
  Generalizes from 2.5k training particles to 28k at test time (an order of magnitude more) and
  to inflow / unseen-object / much-larger-domain conditions. Simpler and more accurate than the
  prior specialized methods (DPI, Continuous Convolution).
- Stated limitations / future work: efficiency vs traditional simulators (needs better
  parallelization); could extend to meshes (-> MeshGraphNets) and to Hamiltonian / symmetry
  priors; differentiability could be used for inverse problems.

## Interpretation (my reading, labeled inference)

- GNS is the clearest evidence that "dynamics = learned message passing over entities and their
  relations" scales, generalizes combinatorially, and is data-efficient relative to global /
  pixel models. The number-of-message-passing-steps finding operationalizes a key intuition: the
  reach of a prediction scales with how many relational hops the model propagates. In a social
  setting, a consequence like "if Alice lends Bob the only pickaxe, Carol's mining plan is also
  blocked" is a multi-hop relational inference of exactly this kind.
- GNS is a WM, not a WAM (per the survey at `<ROOT>/papers/latex/2605.12090/`): it predicts
  `o'` but does not select actions. For this project that is the correct division of labor: the
  LLM proposes the action; a GNS-like relational predictor forecasts the structured delta.
- The relation-edge construction (connectivity radius, recomputed each step) is a physical
  analogue of "which entities can affect which" - in Minecraft this is reachability / proximity /
  containment / claim relations, which are already typed in the runtime rather than computed by
  radius.

## Mechanically useful vs research contribution

- Mechanically useful for this repo:
  - The encode-process-decode GN template as the architecture for a structured-state predictor:
    nodes = actors/items/containers/stations/places; edges = typed relations (holds, controls,
    owes, inside, near, can-reach); message passing yields per-node and per-edge deltas.
  - The rollout-stability discipline (noise injection, residual connections, controlling
    message-passing depth) as practical guidance if any multi-step prediction is attempted.
  - The MSE / distributional-metric eval as a template for measuring transition-prediction error
    against logged ground-truth deltas.
- What this repo should AVOID / would be overclaim:
  - GNS predicts continuous physical accelerations; do not claim it models possession, obligation,
    or trust. It supplies the message-passing-over-relations representation, not social content.
  - Do not adopt GNS as runtime authority. The repo rule keeps the predictor advisory; GNS is a
    forward simulator, and using its rollout to gate execution would violate "runtime owns truth".

## WAM layer(s) informed

- Physical (the demonstrated domain). The relational message-passing machinery is the
  cross-cutting representation that, by interpretation, extends to Material / Social /
  Institutional layers when the nodes/edges are possession and obligation relations rather than
  particles and contact forces.
