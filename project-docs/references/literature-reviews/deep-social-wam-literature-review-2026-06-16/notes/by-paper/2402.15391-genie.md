## What it introduced and why it mattered (plain language)

Genie is the model that learned to make worlds you can play, from internet videos with
no action labels at all. Its trick (the "latent action model") is the most important
single idea in this lane: train a model to invent its own small set of action codes by
watching what changes between consecutive video frames, then condition a video predictor
on those codes so a user can "press buttons" frame by frame. This unlocked
"action-controllable" world models from raw, unlabeled video, the cheapest and most
abundant data there is. At 11B parameters DeepMind called it a "foundation world model."
It matters here because the project must reason about what counts as an action when no
action labels exist, and Genie's answer (infer latent actions from observed change) is
directly analogous to inferring social-material deltas from observed Minecraft state
transitions.

Newcomer terms:
- Latent action model (LAM): a module that infers a discrete "action code" between each
  pair of frames, unsupervised, by requiring that the code be enough (with the past) to
  reconstruct the next frame. The codebook is tiny so codes stay human-meaningful.
- Inverse-dynamics flavor: LAM is close to an inverse-dynamics model (infer the action
  that explains a transition), but with a learned discrete latent instead of a known
  action label.
- Foundation world model: a single large world model that generalizes across many
  prompts/domains, the way a foundation language model generalizes across tasks.
- MaskGIT: a masked-token image/video generator that fills in masked tokens in parallel
  (used as Genie's dynamics decoder).

## Primary-source facts

- Three components, all built on a memory-efficient spatiotemporal (ST) transformer:
  (1) Latent Action Model (LAM): encoder takes frames `x_1:t` and the next frame `x_t+1`,
  outputs continuous latent actions; a decoder reconstructs `x_t+1` from past + latent
  action; trained VQ-VAE-style with a small codebook (`|A| = 8`). The LAM is discarded at
  inference and replaced by the user's chosen action code. (2) Video tokenizer: a VQ-VAE
  with ST-transformer (ST-ViViT) compressing video to discrete tokens. (3) Dynamics model:
  a decoder-only MaskGIT transformer predicting next-frame tokens from past tokens +
  latent actions (latent actions added as additive embeddings, which improved
  controllability).
- Training: two phases (tokenizer first, then co-train LAM from pixels + dynamics on
  tokens), next-token cross-entropy with Bernoulli token masking (rate 0.5-1.0). Final
  model 10.7B params (10.1B dynamics), 942B tokens, 256 TPUv5p.
- Data: ~30k hours filtered from 55M public 2D-platformer internet video clips, 160x90 at
  10 FPS, action-free and text-free. A separate model trained on action-free RT-1 robot
  videos to show generality.
- Metrics: video fidelity via FVD; controllability via a custom delta-PSNR comparing
  generations under ground-truth-inferred vs random latent actions. Scaling studied 40M ->
  2.7B (dynamics), graceful scaling in model and batch size.
- Notable claimed behaviors: consistent latent-action semantics across different prompts
  ("akin to learning the buttons on a new controller"); OOD prompting from text-to-image
  outputs, hand sketches, real photos; emulating parallax and "physical properties of
  objects such as bags of chips"; using frozen LAM to label expert videos and train an
  imitation policy in unseen RL environments.
- Stated limitations (theirs; appendix/discussion): ~1 FPS interaction; limited memory
  (16-frame context); can hallucinate / lose coherence; quality bounded by the small
  action vocabulary and platformer-domain data.

## Interpretation (inference, labeled)

- The single most reusable abstraction in this lane is Genie's classification table:
  World Models = train on Video + Actions, frame-level control; Video Models = Video +
  Text, video-level control; Genie = Video only, frame-level control. This is the cleanest
  way to teach a newcomer the difference between a world model, a video model, and Genie's
  latent-action variant, and it lines up with the WAM survey's own VLA/WM/Video-Policy
  taxonomy (2605.12090).
- "Learns physical properties (bags of chips), emulates parallax" are emergent-visual
  claims from a generative model. They are real and impressive as video behavior, but they
  are the exact claims the debate says must be tested, not assumed to be causal/physical
  understanding. Read them as "the generated video is locally physically plausible," not
  "the model has a correct physics engine."

## Mechanically useful vs research contribution

- Mechanically useful: the latent-action principle ("infer a small discrete action vocabulary
  from observed transitions, unsupervised") is a strong template for a Minecraft setting if
  action labels were ever missing, and more usefully as a framing for the inverse problem the
  social WAM faces: given a logged state transition, infer which typed action (lend/return/
  place) explains the social-material delta. The tiny-codebook-for-interpretability choice is
  a good design instinct.
- Avoid / overclaim: Genie is a pixel video model. The repo already has explicit, schema-bound
  Mineflayer actions, so it does NOT need to discover latent actions; inventing a latent-action
  layer on top of typed actions would add hidden authority and conflict with "runtime owns the
  action space." Use Genie as a conceptual lens, not an architecture to graft on.
