## Primary-source facts (what the survey states)

- Scope: the first systematic review dedicated to memory mechanisms of LLM-based agents (memory is the component that supports long-term, complex agent-environment interaction and self-evolution).
- Three guiding questions: what is memory, why is it needed, how to design and evaluate it.
- Why memory (three perspectives): cognitive psychology (memory is the agent's cognitive basis), self-evolution (memory enables the agent to improve), agent applications.
- Formal framing. In agent-environment interaction there are three phases: perceive-and-store, process-stored-information, act-on-processed-memory. The memory module implements these via three operations:
  - Memory writing W (store an observation/action),
  - Memory management P (process stored information: e.g. reflection, summarization),
  - Memory reading R (read relevant memory for the next action).
  - Generative Agents is given as an example: R uses similarity + time interval + importance, P is the reflection process.
- Memory sources, three categories: inside-trial information (history within the same trial / task attempt), cross-trial information (history across different trials and tasks), and external knowledge (static, outside the loop). The first two are dynamically generated; the third is static.
- Memory forms and memory operations are the other two design dimensions surveyed (memory forms = textual vs parametric; operations = the W/P/R above).
- The survey also covers evaluation of memory and many memory-using agent applications, and lists limitations and future directions.

## Interpretation (my inference, labeled)

- This is the foundational, most-cited taxonomy for the lane's "memory architectures" angle, and its inside-trial / cross-trial / external-knowledge source split is the cleanest way to name what a multi-cycle actor needs: cross-trial memory is exactly "carry an obligation/lesson across cycles," which intra-trial (single-cycle) memory cannot do.
- The W/P/R operation decomposition (write, manage, read) is the same write-manage-read loop the neighbor theme cites from 2603.07670; this survey is its earlier source. For the repo, the management step P (reflection/summarization) is where the evidence-linking rule must bite: P must not fabricate a distilled fact, it must summarize verified events and keep pointers to them.
- A later survey (2505.00675, also deep-read this lane) explicitly notes this survey covers only high-level writing/management/reading and misses operations like indexing; so this taxonomy is foundational but coarser than the six-operation one.

## Mechanically useful vs research contribution

- Mechanically useful: the source taxonomy (inside-trial / cross-trial / external) as the vocabulary for designing actor memory scope; the W/P/R operation split as the minimal interface for any memory module; the survey's catalog of retrieval scoring (similarity + recency + importance) confirming the Generative-Agents recipe as the field standard.
- Research contribution it is NOT for this repo: a survey provides vocabulary and a map, not a mechanism. It does not solve evidence-grounding or embodied memory; it lists them as open problems. It is orienting material, not an adoptable component.
