# FAST: Efficient Action Tokenization for Vision-Language-Action Models

- **title**: FAST: Efficient Action Tokenization for Vision-Language-Action Models
- **authors**: Pertsch, Stachowicz, Ichter, Driess, Nair, Vuong, Mees, Finn, Levine (Physical Intelligence / UC Berkeley / Stanford)
- **year**: 2025 (arXiv Jan 2025)
- **venue/source**: arXiv 2501.09747; project pi.website/research/fast
- **arxiv_id**: 2501.09747
- **urls**: paper https://arxiv.org/abs/2501.09747 ; tokenizer https://huggingface.co/physical-intelligence/fast (Apache-2.0) ; LaTeX at `papers/latex/2501.09747/`
- **source availability**: latex (full, `sections/04_method.tex` etc.)

Part of the VLA-canon set missing from wave-1. FAST is not a policy; it diagnoses why
the canon's default action tokenization (RT-1/RT-2/OpenVLA's per-dimension 256-bin
binning) breaks on high-frequency data, and replaces it with a frequency-space scheme.

## Primary-source facts

- **Research question**: autoregressive VLAs tokenize continuous actions before
  predicting them. The standard scheme (per-dimension, per-timestep binning into 256
  bins) "performs poorly when learning dexterous skills from high-frequency robot data."
  Why, and what is a better tokenization?
- **The diagnosis (the teaching point)**: autoregressive training signal per token is
  proportional to the **marginal information** of token `T_i` given `T_{1:i-1}`. With
  naive per-timestep binning, as control frequency rises the per-step change shrinks, so
  consecutive tokens become highly correlated and this marginal information **approaches
  zero**. The model learns to just **copy the previous action**, and convergence stalls.
  Concrete instance cited: **OpenVLA fit low-frequency BridgeV2 / RT-1 data well but
  struggled on the higher-frequency DROID dataset.** A didactic spline-interpolation
  experiment shows naive binning MSE rising steeply with sampling rate.
- **The method (FAST = Frequency-space Action Sequence Tokenization)**: compress the
  redundant action signal into fewer high-information tokens.
  1. **Quantile-normalize** each action dimension (1st-99th percentile -> [-1, 1]),
     robust to outliers and helpful across embodiments.
  2. **Discrete Cosine Transform (DCT)** per action dimension (the same frequency
     transform used in JPEG); low-frequency coefficients capture overall shape, high
     frequencies capture sharp jumps.
  3. **Quantize** coefficients (scale-and-round; one hyperparameter trades lossiness vs
     compression), yielding a sparse coefficient matrix.
  4. **Byte-Pair Encoding (BPE)** over the flattened (low-frequency-first / column-first)
     coefficient stream losslessly squashes zeros and merges frequent patterns into a
     fixed-size action vocabulary that drops into a VLM's existing vocabulary.
  - All steps invertible (fast decoding). Two hyperparameters (DCT scale, BPE vocab
    size); both insensitive (defaults: scale 10, BPE vocab 1024). Beats learned
    VQ-VAE tokenizers while being far simpler to tune.
- **FAST+ (universal tokenizer)**: train the BPE step on a large cross-embodiment
  corpus of **~1 million 1-second action chunks** (single-arm, bimanual, mobile; joint
  and end-effector control; various frequencies). The result is a **black-box tokenizer**
  usable on any robot's 1-second action chunk, competitive with per-dataset tokenizers.
  Released as a HuggingFace `AutoProcessor` (3 lines to tokenize; `.fit()` to train a
  new one).
- **Result**: trains autoregressive VLAs on high-frequency, dexterous data where naive
  binning "fails completely"; with pi-0, scales to **10k hours** and **matches diffusion
  VLA performance while reducing training time up to 5x**.

## Interpretation (inference)

- FAST is the capstone of the canon's **action-representation** axis. The arc:
  RT-1/RT-2/OpenVLA = per-dimension 256-bin tokens (simple, but low marginal information
  at high frequency); Octo/pi-0 = continuous diffusion/flow heads (sidestep tokenization
  but are not autoregressive); FAST = keep autoregression but **compress actions in
  frequency space** so each token carries real information. It is the clearest statement
  that *how you discretize the action target* materially changes what an autoregressive
  policy can learn.

## Mechanically useful vs research contribution

- **Useful for THIS repo (conceptual)**: the core lesson is **"a token should carry
  meaningful marginal information."** The repo's typed Action Cards already satisfy this
  by construction (each tool selection is a high-information, discrete, semantically
  meaningful choice, not a near-redundant per-timestep bin), which is an independent
  validation of the repo's high-altitude action surface over low-level motor tokens. If
  the repo ever logs or models sequences of typed actions, the "compress redundancy,
  predict high-information units, low-frequency/coarse structure first" principle is a
  good prior.
- **Avoid / overclaim**: FAST is a motor-action tokenizer for high-frequency continuous
  control; the repo has no continuous high-frequency action stream, so FAST+ itself is
  not directly applicable. Borrow the *principle* (marginal-information-per-token), not
  the DCT machinery. Do not frame action tokenization as a research contribution for the
  repo.
