# Tree of Thoughts: Deliberate Problem Solving with Large Language Models

- **title**: Tree of Thoughts: Deliberate Problem Solving with Large Language Models
- **authors**: Shunyu Yao, Dian Yu, Jeffrey Zhao, Izhak Shafran, Thomas L. Griffiths, Yuan Cao, Karthik Narasimhan (Princeton, Google DeepMind)
- **year**: 2023 (arXiv 2023-05-17)
- **venue/source**: NeurIPS 2023
- **arxiv_id**: 2305.10601
- **urls**: paper https://arxiv.org/abs/2305.10601 ; code https://github.com/princeton-nlp/tree-of-thought-llm
- **source availability**: latex (single main.tex, deep-read)

## Primary-source facts (LaTeX-verified)

- **The framing**: generalize Chain-of-Thought into a search over a tree where each node is a STATE `s = [x, z_1..i]` (the input plus the partial sequence of thoughts so far). A "thought" is a coherent language unit (an intermediate step). Motivated explicitly by dual-process theory (System 1 fast token-level generation vs System 2 deliberate search).
- **Four design choices a ToT instance must answer**: (1) how to DECOMPOSE a problem into thought steps; (2) how to GENERATE candidate thoughts from a state; (3) how to heuristically EVALUATE states; (4) which SEARCH algorithm (BFS or DFS) with lookahead and backtracking.
- **The novel part, in the authors' words**: "This implementation of search heuristics via LM self-evaluation and deliberation is novel, as previous search heuristics are either programmed or learned." The LLM evaluates its own partial states (e.g. "sure / maybe / impossible" for Game of 24) to steer the search.
- **Key distinction from RAP/WebDreamer for this lane**: ToT's "state" is a partial SOLUTION string and the LLM SELF-EVALUATES the promise of a state; it does NOT predict environment dynamics `p(s'|s,a)`. So ToT is the purest "LLM is both the proposer and the scorer" archetype, with no external grounding at all. This is the structure the repo's hard rule forbids for scoring (the proposer must never be the scorer).

## Numbers as stated

- Game of 24: GPT-4 with CoT prompting solves 4%; ToT achieves 74%.
- Also evaluated on Creative Writing and 5x5 Mini Crosswords (ToT superior on both; Game of 24 is the headline).

## Reproducibility status

Self-reported, code and all prompts released; widely reproduced. The "evaluation" is the LLM's own self-judgment of partial states, validated only by final task success. No external transition ground truth.

## Relevance to this repo

- **Mechanical import (medium)**: ToT supplies the search scaffolding (propose thoughts, evaluate states, BFS/DFS with backtracking) that any LLM-as-world-model search reuses. Its decompose/generate/evaluate/search checklist is a clean design template.
- **The cautionary core for the repo**: ToT is the canonical self-evaluation-as-heuristic system, and it is exactly what the repo must NOT do for scoring. ToT's value signal is the same LLM judging its own thoughts; this is the progress-laundering pattern (proposer == scorer) that the directions report says must be structurally blocked. ToT works on closed puzzles (Game of 24) where the final answer is checkable, which masks the self-scoring weakness; in an open social-material world there is no clean final check, so a ToT-style self-evaluated search would launder progress unless every node's predicted state is re-scored by the runtime verifier.
- **Useful contrast**: ToT shows search over an LLM helps even WITHOUT a dynamics model (pure self-evaluation). The repo can keep ToT's search but must move the EVALUATOR out of the LLM and into the verifier. That single change converts ToT from "LLM grades itself" to "LLM proposes, runtime grades," which is the admissible advisory form.
- **Research contribution vs support**: ToT is upstream scaffolding, not a result to claim. The contribution is replacing LM self-evaluation with verifier-grounded state scoring.

## WAM layer(s) informed

Method-level, cross-layer (search scaffolding). It is the clearest negative example for the Social/Institutional layers: a self-evaluated tree search has no external check, so its scores cannot be trusted as progress without the verifier.
