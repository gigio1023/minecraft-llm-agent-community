## Primary-source facts (what the survey states)

- Thesis: prior memory surveys emphasize application-level use and overlook the atomic operations that govern memory dynamics. This survey formalizes both representations and operations.
- Memory representations, two top types:
  - Parametric memory: knowledge implicit in model weights.
  - Contextual memory: explicit external data, split into Contextual Unstructured (text, images, audio, video, embeddings, KV-cache short-term, memory buffers / retrieval DBs / episodic stores long-term) and Contextual Structured (knowledge graphs, relational tables, experiences, ontologies, queryable schemata).
- Temporal split: long-term memory (multi-turn dialogue, external observations) vs short-term (KV-cache, current context).
- Six core operations, grouped into three categories:
  - Encoding: Consolidation (integrate m short-term experiences into persistent memory), Indexing (build auxiliary access codes / temporal and relational structure for retrieval).
  - Evolving: Updating (reactivate and modify existing memory with new knowledge; parametric = locate-and-edit, contextual = summarize/prune/refine), Forgetting (selectively suppress outdated/irrelevant/harmful content; parametric = unlearning, contextual = time-based deletion or semantic filtering).
  - Adapting: Retrieval (score memory fragments with a similarity function, keep those above threshold tau), Condensation/Compression (reduce size while preserving key information).
- Functional types (cognitive lens): Episodic (temporally anchored experiences, dialogue histories, event sequences; often an external module), Semantic (facts/concepts, as knowledge graphs/tables or parametric), Procedural (learned skills/action sequences; explicit skill repositories or implicit via training), Working memory (dynamic control layer over the context window that selects and integrates short-term context + activated long-term knowledge).
- Four high-impact research topics mapped to the taxonomy: long-term memory, long-context memory, parametric memory modification (knowledge editing), multi-source memory.
- Explicitly notes that the earlier Zhang et al. 2024 survey (2404.13501) covers only writing/management/reading and misses operations like indexing.

## Interpretation (my inference, labeled)

- This is the most precise vocabulary for the lane's "episodic vs semantic vs working memory" and "consolidation / forgetting / retrieval scoring" angles. The parametric-vs-contextual split is the exact axis that separates PEAM (2605.27762, parametric internalization) from the retrieval systems (MemGPT, MemoryBank, MemoryOS, TierMem, all contextual).
- For the repo, the contextual-structured category is the home of the design the repo wants: the social/material ledger is structured contextual memory (typed, queryable, evidence-linked), not unstructured prose and not parametric. The repo's hard line ("a social variable is a ledger value updated from verified events") is, in this taxonomy, a commitment to contextual-structured memory with consolidation gated by a verifier.
- The six-operation decomposition is a useful checklist for the repo's memory design: every operation (consolidate, index, update, forget, retrieve, condense) must, for a social/material value, point back to the verified event(s) that justify it, otherwise the operation laundions unverified prose into durable state.

## Mechanically useful vs research contribution

- Mechanically useful: the representation axis (parametric vs contextual-structured vs contextual-unstructured) as a design decision the repo has already made (structured, evidence-linked); the six-operation checklist as a coverage map for an actor-memory implementation; the functional types (episodic for cycle events, semantic for distilled norms, procedural for skills, working for the in-context window) as naming for the actor workspace layers.
- Research contribution it is NOT for this repo: a survey gives structure, not a mechanism or a result. It does not solve verifier-grounded memory; it catalogs the operations such a system must implement.
