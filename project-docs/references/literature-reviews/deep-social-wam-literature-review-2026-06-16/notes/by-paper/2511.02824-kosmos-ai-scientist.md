## Primary-source facts (from main.tex)

- Kosmos runs up to 12 hours on an open-ended objective + a dataset, doing parallel cycles of data analysis, literature search, and hypothesis generation, then synthesizes 3-4 scientific reports (abstract, line 86).
- **Structured world model** is the core advance (lines 150, 344): a shared, continuously-updated structure that coordinates two general-purpose Edison agents (a data-analysis agent and a literature-search agent) across parallel rollouts. Each cycle runs up to ten tasks, updates the world model with task-output summaries, then queries the world model to propose the next cycle's tasks. This context-management strategy lets Kosmos run ~8x more iterations than prior systems and stay coherent over 200 agent rollouts.
- Scale per run: ~42,000 lines of code across ~166 data-analysis rollouts, ~1,500 papers read across ~36 literature rollouts (line 141). Every statement in a report cites a Jupyter notebook (code) or a primary-literature source, so claims are traceable.
- **Accuracy** (line 152, expert evaluation of 102 statements from 3 reports, evaluators NOT shown the supporting code/citation): overall **79.4%** accurate, broken down as **85.5% of data-analysis statements reproducible, 82.1% of literature-review statements validated, 57.9% of synthesis (interpretation) statements accurate**.
- Time-equivalence (lines 154-156): authors estimate ~4.1 expert-months per run by task tally; collaborating academic groups independently estimated a 20-cycle run equaled ~6.14 months of their research; reported valuable-findings count scales roughly linearly with cycles (tested to 20).
- Seven highlighted discoveries across metabolomics, materials science, connectomics, statistical genetics, proteomics, transcriptomics; three independently reproduce findings from manuscripts not accessed at runtime; some are novel.
- **Limitations** (lines 346-358): evaluators noted Kosmos "tends to make excessively strong claims and can sometimes veer in unexpected trajectories"; it "invent[s] unorthodox quantitative metrics... statistically sound but conceptually obscure"; the 57% interpretation accuracy is attributed to "its propensity to conflate statistically significant results with scientifically valuable ones." Critically: "there exists no automated method to reliably evaluate if a claim is accurate, novel, and significant" (line 356). Identifying valuable discoveries is a human, domain-expert, time-intensive task. Stochastic: independent runs may not converge.

## Interpretation (inference, flagged)

- This is the most direct AI-scientist-to-WAM bridge in the lane, but the bridge must be drawn carefully. Kosmos's "structured world model" is a SHARED-STATE / CONTEXT-MANAGEMENT structure (a blackboard that aggregates agent outputs and proposes next tasks), NOT a predictive transition model `p(o'|o,a)`. By the survey's WAM definition it lacks Forward Predictive Modeling and Coupled Action Generation. Calling it a "world model" is a naming choice about coordination, not a learned dynamics model. Flag this whenever citing Kosmos as a "world model" precedent.
- The accuracy breakdown is the lane's cleanest empirical signal for the original query and the autoresearch thesis: statements that are externally verifiable (reproduce the analysis: 85.5%; check the citation: 82.1%) are reliable, while interpretive/synthesis statements (57.9%) are not. This maps directly onto the repo's layer dependency: physical/material predictions (verifiable against the runtime) can be trusted far more than social/institutional interpretations (contested, no clean verifier).
- "No automated method to reliably evaluate if a claim is accurate, novel, and significant" is independent corroboration that an LLM cannot be its own success signal for the interpretive layer; a human or an external verifier is required.

## Mechanically useful vs research contribution

- **Mechanically useful**: the claim-to-source binding (every statement linked to a notebook or citation) is a concrete provenance pattern the repo can mirror, binding every WAM prediction to the verifier evidence that confirms or refutes it. The shared structured state for coordinating parallel rollouts is a usable pattern for running many headless Minecraft instances coherently.
- **Research contribution to NOT overclaim**: Kosmos's contribution is autonomous data-driven discovery at scale with traceability; do not cite it as evidence that an LLM "world model" can predict consequences. Its world model does not predict `o'`. The repo should not borrow the label to imply predictive capability it does not have.

## WAM layer(s) informed

Method-level. The accuracy-by-statement-type result informs the layer hierarchy directly (verifiable layers reliable, interpretive layers not). The structured-state coordination informs fleet/parallel ops, not the predictive WAM itself.
