# Project Docs Index

Search token: `PROJECT_DOCS_INDEX`.

This is the internal documentation root. Public Docusaurus pages live under
`docs/public-docs/`; root authority docs such as `SPEC.md` and `AGENTS.md` stay
at the repository root.

## Start Here

- `project-docs/orientation/overview.md` - internal project overview.
- `project-docs/orientation/documentation-map.md` - where documents belong.
- `project-docs/orientation/agent-search-index.md` - search-token routing map.
- `project-docs/orientation/terminology.md` - normative vocabulary.

## Current Authority

- `project-docs/research/current-spine/` - current research plan and gates.
- `project-docs/specification/` - long-term specs and governance boundaries.
- `project-docs/runtime/` - runtime architecture and implementation contracts.
- `project-docs/operations/` - setup, handoffs, audits, run reports, and future
  work.

## Evidence And Reference Stores

- `project-docs/experiments/` - curated reports and raw run artifacts.
- `project-docs/knowledge/` - stable Minecraft and Mineflayer background.
- `project-docs/references/` - literature reviews, sweeps, external-project
  notes, and preserved source material.
- `project-docs/archive/` - superseded plans and historical public-doc material.

Do not delete reference or archive material during cleanup. If it is noisy, add
an index or move it to the correct non-active directory.
