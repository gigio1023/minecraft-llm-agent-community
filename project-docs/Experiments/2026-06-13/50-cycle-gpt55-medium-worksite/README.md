## Findings

P1 - Benchmark harness missing target early-stop.

Evidence: the declared behavior target was satisfied by cycle 12, but the run
continued until provider quota failed at cycle 29.

Fix target: add benchmark target predicates and progress events so the runner
can stop when the target is reached, or intentionally continue only when the
case is marked as an endurance/recovery benchmark.

P1 - OpenAI `gpt-5.5` long run had no local budget guard.

Evidence: provider usage summary marked the run as `unbudgeted`; the run ended
with provider `insufficient_quota`.

Fix target: add repo-local OpenAI long-run budgets before repeating expensive
50-cycle or multi-model runs.

P2 - Generated action timeout mismatch.

Evidence: generated candidates requested longer behavior such as
`timeoutMs: 40000`, but `run_mineflayer_program` timed out at 12000ms. Several
generated action trials therefore failed by timeout.

Fix target: align generated action candidate timeout policy with runtime
execution timeout, or reject generated candidates that require more time than
the active runtime can grant.

P2 - Generated action authoring must be scored separately.

Evidence: after the target was reached, cycles 19, 21, 24, 25, 26, and 28 used
`author_mineflayer_action`. Those calls dominated latency and token spikes.

Fix target: separate base Actor Turn goal pursuit from generated action-skill
evolution cost in benchmark reports.
